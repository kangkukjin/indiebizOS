# IndieBiz OS 사용자 가이드

이 문서는 IndieBiz OS를 실제로 사용하려는 사람을 위한 가이드입니다.

---

## 왜 이렇게 설계되었는가

### AI는 미디어이자 동료다

현재 대부분의 AI 서비스는 "하나의 전능한 AI"와 대화하는 구조입니다. ChatGPT든 Claude든, 하나의 대화창에서 투자 상담을 하다가 PC 문제를 물어보고, 건강 질문을 하죠. 결과적으로 컨텍스트가 섞이고, 기록이 뒤엉키고, AI도 혼란스러워집니다.

그리고 이런 "범용 AI"는 모든 사람에게 맞는 답을 하려다 보니 **구체성이 떨어집니다**. 레스토랑 추천을 받아도, 전시회 추천을 받아도, 누구에게나 무난한 답만 돌아오죠.

IndieBiz OS는 다르게 접근합니다:
- **프로젝트**라는 폐쇄된 공간에서 관련된 일만 처리
- **에이전트**에게 특정 역할과 페르소나를 부여
- 인간이 자기 일을 원하는 대로 나누고 구조화

**페르소나의 힘**: 같은 질문이라도 에이전트의 페르소나에 따라 답이 달라집니다. "서울에서 좋은 레스토랑 추천해줘"라고 물었을 때:
- "30대 여성 드라마 작가" 페르소나 → 분위기 있는 곳, 스토리가 있는 공간
- "50대 과학자" 페르소나 → 조용히 대화할 수 있는 곳, 음식의 퀄리티 중심
- "미국의 철학자" 페르소나 → 낯선 시선으로 본 서울의 독특한 장소

범용 AI의 평균적인 답 대신, 특정 관점에서 나온 구체적이고 개성 있는 답을 받을 수 있습니다.

그리고 로컬에서 실행되기 때문에 파일을 직접 수정하고, PC에 연결된 하드웨어(안드로이드폰, ESP32 보드 등)까지 제어할 수 있습니다.

### 힘은 연결에서 온다

AI 하나만으로는 대단한 일을 못 합니다. 도구가 있어야 실제로 행동할 수 있죠.

IndieBiz OS는 세 가지 연결을 제공합니다:
- **도구와의 연결**: 사용자가 직접 도구를 만들어 무한히 확장
- **에이전트 간 연결**: 위임 체인으로 협업 (단일/순차/병렬 위임)
- **외부 세계와의 연결**: IndieNet으로 다른 사용자들과 소통

MCP 서버와 달리 에이전트별로 도구를 할당하므로 도구 수의 제한이 없습니다. 투자 에이전트에게 50개, 부동산 에이전트에게 40개의 도구를 줘도 서로 간섭하지 않습니다.

### 에이전트가 스스로 협업한다

**위임 체인**은 에이전트들이 팀처럼 일하게 합니다:
- **단일 위임**: 투자 에이전트 → 정보센터 에이전트 (데이터 수집 요청)
- **순차 위임**: 기획 → 제작 → 홍보 (릴레이처럼 단계별 처리)
- **병렬 위임**: 컨텐츠 에이전트가 투자, 부동산, 창업 에이전트에게 동시 요청

위임받은 에이전트는 작업 완료 후 자동으로 결과를 보고합니다. 복잡한 작업도 당신은 결과만 받으면 됩니다.

### 빈 PC처럼 시작한다

IndieBiz OS는 처음 설치하면 **비어 있습니다**. 이건 의도된 설계입니다.

일반 AI 서비스가 "이미 완성된 앱"이라면, IndieBiz OS는 "새로 산 PC"입니다:
- 어떤 도구 패키지를 설치할지 사용자가 결정
- 어떤 프로젝트를 만들지 사용자가 결정
- 어떤 에이전트를 어떻게 구성할지 사용자가 결정

시간이 지나면서 "나만의 AI 환경"이 됩니다. 아래 예제들이 그 과정을 보여줍니다.

## 1. 시작하기

### 목적부터 — 공유창고, 그리고 왜 AI 에이전트인가

시작 전에 목적 하나를 쥐고 가면 나머지 전부가 의미를 갖습니다. IndieBiz OS의 한 축은 **공유창고**입니다 — 폴더에 파일을 던져 넣으면 세상에 발행되는 창고. 동영상을 넣으면 유튜브가, 글을 넣으면 블로그가, 상품 목록을 넣으면 쇼핑몰이 하던 일을 창고 하나가 합니다. 플랫폼이 종류별로 따로 있었던 건 인간용 매대가 각각 필요했기 때문이고, 읽는 쪽이 AI라면 매대 없이 창고면 됩니다.

그리고 그 창고가 다른 한 축인 **AI 에이전트 시스템**을 요구합니다. 창고는 채워져야 살고(에이전트가 보고서·신문·카탈로그를 매일 만들어 자동 발행 — 생산이 곧 발행), 이웃 창고의 raw 파일 더미를 뒤져 이해하는 것도 내 AI의 몫입니다. 블로그가 글쓰기를 의미 있게 했듯, 뭐든 내놓을 수 있는 창고는 뭐든 만들 수 있는 AI를 의미 있게 합니다. AI 에이전트 시스템은 세상에 많지만, IndieBiz OS가 다른 점은 배포·연결 채널과 한 몸이라는 것입니다.

이 목적을 기준으로, 시작에 필요한 것은 명확히 정해져 있습니다. **필수 기둥 둘**(AI + 공개 주소), **준필수 하나**(폰), **자동 하나**(Nostr), 그리고 **선택 항목들**(통신 채널). 이 장은 그 순서대로 갑니다.

### 핵심 기능 ↔ 필요한 세팅 (지도)

| 쓰게 되는 것 | 무엇인가 | 필요한 세팅 |
|---|---|---|
| AI 에이전트 시스템 | 자율주행·조종실·앱 3표면 — 내 PC에서 도는 나만의 AI | 기둥① AI API 키 |
| 시스템의 설치와 수리 | 시스템을 지어주고, 고장 나면 고쳐주는 외부의 손 | 기둥① 외부 하네스 |
| 공유창고 | 파일을 던져 넣으면 세상에 발행되는 창고 + 이웃 창고 피드 | 기둥② Cloudflare+도메인 |
| 원격 런처·원격 NAS·포털·가족신문·게시판 | 밖에서 브라우저로 쓰는 모든 공개 표면 | 기둥② Cloudflare+도메인 |
| 폰에서 쓰기 | 어떤 폰이든 브라우저로(설치 0) · 안드로이드는 폰 자체가 두 번째 노드 | 기둥③ (안드로이드만) 앱 설치 |
| 다른 사용자와 메시징 | Nostr 통신 | 자동 (키 생성 = 계정) |
| 이메일·텔레그램 등 | 추가 통신 채널 | 선택 (원할 때) |

기둥①만 있어도 에이전트 시스템은 돕니다. 하지만 기둥②가 없으면 공개 표면 전부 — 원격 접속과 공유창고 — 를 쓸 수 없어 **시스템이 반쪽이 됩니다**. 만드는 것은 되는데, 배포하고 연결하는 것이 안 되기 때문입니다.

**비용 구조는 단순합니다**: 변동비 = AI API 사용량 하나뿐. 고정비 = 도메인 연 1회(1~2만 원). 나머지(Cloudflare·Nostr·소프트웨어 자체)는 전부 무료입니다.

### 설치

1. [GitHub Releases](https://github.com/kangkukjin/indiebizOS/releases)에서 최신 버전 다운로드
   - macOS: `.dmg` 파일
   - Windows: `.exe` 설치 파일
2. 다운로드한 파일로 설치
3. IndieBiz 앱 아이콘 더블클릭

앱이 시작되면 백엔드 서버가 자동으로 함께 실행됩니다.

더 좋은 방법은 **Claude Desktop(또는 Claude Code) 같은 AI 하네스에게 설치를 통째로 맡기는 것**입니다 — "https://github.com/kangkukjin/indiebizOS 를 내 PC에 설치해줘" 한 줄이면, 클론·의존성·초기 설정을 대화로 진행해 줍니다. 왜 이 방법을 권장하는지는 바로 아래 기둥①에서 설명합니다.

**개발자용 (소스에서 실행)**

```bash
cd backend && python api.py        # 터미널 1
cd frontend && npm run electron:dev # 터미널 2
```

### 기둥 ① — AI API 키 + 외부 하네스 (필수)

**AI API 키가 최소 1개 필요합니다.** 이게 유일한 변동비이자 첫 번째 관문입니다. 이것만 넘으면 나머지 설정은 시스템 AI에게 물어보거나 시키면 됩니다.

**2026년 현재 추천: Google Gemini API (무료 시작)**

1. https://aistudio.google.com 에서 API 키 발급
2. 앱 내 설정 → API 키 탭에 입력
3. 모델 선택:
   - `gemini-3-flash-preview`: 빠르고 가벼움 (일반 용도 추천)
   - `gemini-3-pro-preview`: 더 강력함 (복잡한 작업용)

발급 방법을 모르겠다면 아무 AI에게 "Gemini API 키 발급 방법 알려줘"라고 물어보세요. 키를 넣으면 시스템 AI가 깨어나고, 그 다음부터는 대화로 도구 설치·프로젝트 생성·설정 변경을 할 수 있습니다.

**외부 하네스(Claude Desktop, Claude Code 등)도 사실상 필수입니다.** 대안이 아니라 기둥의 절반입니다. 이유는 두 가지입니다:

- **설치 대장장이**: IndieBiz OS는 `npm install`로 끝나는 정적인 앱이 아니라 당신에게 맞춰 계속 벼려지는 시스템입니다. 설치한 하네스가 곧 그 대장장이가 됩니다.
- **복구 경로**: 시스템 AI도 자기 시스템을 고칠 수 있지만, **백엔드 자신이 고장 난 상황에서는 원리적으로 불가능합니다** — 의식을 잃은 환자가 자가 수술을 못 하는 것과 같습니다. 그때 외부 하네스가 시스템 폴더를 열고 고칩니다. 시동이 안 걸릴 때 견인해 갈 정비소가 있어야 차를 안심하고 탈 수 있습니다.

**도구용 API 키(날씨·부동산·통계 등)는 선택이고 나중 일입니다.** 필요해지는 순간 시스템 AI가 어떤 키가 필요한지, 어디서 발급받는지 안내합니다 (설정 → API 키 탭에 키별 발급 안내와 실사용 테스트가 있습니다).

### 기둥 ② — Cloudflare 계정 + 본인 도메인 (공개 얼굴)

이게 없어도 에이전트 시스템은 돌지만, 다음이 전부 잠깁니다: **원격 런처**(밖에서 폰 브라우저로 집 시스템 조작), **원격 NAS**, **공유창고**(파일 발행+이웃 피드), **개인 포털**, **가족신문**, **게시판**. 요컨대 시스템의 공개 얼굴 전체입니다.

**왜 Cloudflare인가**: 필요한 네 부품 — 터널(집 PC를 포트포워딩 없이 공개 인터넷에 연결), Worker(엣지 로직), R2(캐시 저장소), DNS — 를 무료로 한 계정에 묶어주는 곳이 현재 Cloudflare뿐입니다. 대역폭·서버비가 들지 않는 구조라 상시 비용 없이 운영됩니다.

**도메인은 필수입니다**: 터널에 안정된 공개 주소를 붙이려면 본인 도메인이 Cloudflare에 등록되어 있어야 합니다. 연 1~2만 원이 이 시스템 전체의 유일한 고정비입니다. 그리고 이건 단순 비용이 아닙니다 — 유튜브 채널명은 유튜브 것이지만, **도메인은 당신이 소유하는 주소**라서 나중에 어떤 사업자로 옮겨도 들고 갑니다. 공유창고에서는 주소가 곧 당신의 정체이기도 합니다.

**절차 개요** (상세는 하네스나 시스템 AI에게 시키면 됩니다):

1. Cloudflare 가입 (무료)
2. 도메인 구매(Cloudflare Registrar, 원가 판매) 또는 보유 도메인의 네임서버를 Cloudflare로 이전
3. 터널(cloudflared) 생성 — 집 PC와 Cloudflare를 연결
4. Worker 배포 — 공개 표면 서빙
5. 공유창고 폴더 지정 — 이제 파일을 넣으면 발행됩니다

가장 쉬운 방법은 하네스에게 "Cloudflare 터널과 공유창고를 설정해줘"라고 맡기고, 계정 가입과 도메인 결제만 직접 하는 것입니다.

### 기둥 ③ — 스마트폰 (준필수)

폰은 두 단계 사다리입니다:

- **1단 — 아무 폰, 브라우저 (설치 0)**: 기둥②가 서면 자동으로 열립니다. 아이폰이든 안드로이드든 브라우저 주소 하나로 원격 런처(집 시스템 리모컨)·원격 NAS·공유창고에 접속합니다. 별도 설치가 전혀 없습니다.
- **2단 — 안드로이드 앱 (폰이 두 번째 노드가 됨)**: 폰 자체에 IndieBiz OS가 돌아서, 리모컨이 아니라 독립된 몸이 됩니다 — 위치·마이크·카메라 감각, 폰 하드웨어 조작, 클립보드로 카카오톡 직결, 집 PC가 꺼져 있어도 동작. 현재 설치는 USB 연결 후 스크립트 실행이 필요해 다소 번거로우니, 하네스에게 "폰에 IndieBiz 설치해줘"라고 맡기는 것을 권합니다.

1단만으로도 일상 사용에는 충분합니다. 2단은 폰을 적극적으로 쓰게 되면 그때 올라가면 됩니다.

### 자동으로 되는 것 — Nostr 신원

Nostr는 가입 절차가 없는 통신 프로토콜입니다 — 키를 만들면 그게 곧 계정입니다. IndieBiz OS가 키를 자동 생성하며, 그 공개키(npub)가 당신의 연락처가 되어 공유창고 발행물에 자동으로 붙습니다. 당신이 할 일은 없습니다.

### 선택 — 통신 채널 늘리기

더 많은 채널을 원할 때만 설정합니다. 시스템 AI에게 맡기되, 난이도를 알고 시작하면 좋습니다:

- **텔레그램 (봇)**: 쉬움 — BotFather와 채팅 한 번으로 토큰 발급
- **매트릭스**: 쉬움 — 홈서버 계정 가입
- **이메일 (Gmail)**: 어려움 — Google Cloud 프로젝트 생성, OAuth 동의 화면 구성, 클라이언트 ID 발급, 앱 게시까지 여러 단계가 필요합니다. 원하면 하네스와 함께 차근차근 진행하세요.
- **카카오톡**: 개인 메시징 API가 없어 채널로는 불가 — 대신 안드로이드 폰 앱(기둥③ 2단)이 클립보드 경로로 우회합니다.

### 시작 체크리스트

1. **설치** — Claude Desktop에 "설치해줘" (또는 Releases에서 다운로드)
2. **AI API 키 입력** — 시스템 AI가 깨어남 (여기까지가 최소 동작)
3. **Cloudflare + 도메인** — 공개 얼굴이 켜짐 (강력 권장 — 이게 없으면 반쪽)
4. **폰** — 브라우저로 원격 런처 접속, 안드로이드면 앱 설치
5. **나머지 전부** — 시스템 AI와 대화하면서: 프로젝트 만들기, 도구 설치, 채널 추가

## 2. 핵심 개념

### 구조

```
IndieBiz OS
├── 프로젝트 (투자, 부동산, 의료, ...)
│   └── 에이전트 (각각 역할과 도구를 가짐)
└── 도구 패키지 (설치/제거 가능한 기능 모음)
```

### 프로젝트

독립된 작업 공간입니다. 프로젝트끼리 대화 기록이나 맥락이 섞이지 않습니다.

예시:
- **투자**: 주식 분석, 재무제표 조회, 시장 뉴스
- **부동산**: 실거래가 조회, 지역 분석
- **의료**: 건강 상담, 복용약 관리

### 에이전트

프로젝트 안에서 실제로 대화하는 AI입니다. 각 에이전트는:
- **역할(Role)**: 어떤 전문가인지, 어떻게 대화하는지
- **도구(Tools)**: 어떤 기능을 사용할 수 있는지
- **AI 모델**: 어떤 프로바이더/모델을 쓸지

한 프로젝트에 여러 에이전트를 둘 수 있습니다. 의료 프로젝트에 내과, 피부과, 가정의학과 에이전트처럼.

### 도구 패키지

에이전트가 사용할 수 있는 기능 모음입니다. 앱 내에서 설치/제거할 수 있습니다.

예시:
- **investment**: 한국/미국 주가, 재무제표, 공시 조회
- **real-estate**: 아파트/주택 실거래가
- **web**: 웹 검색, 크롤링
- **media_producer**: HTML 기반 슬라이드(12종 테마), 영상 제작, AI 이미지 생성

## 3. 기본 사용법

### 프로젝트 만들기

1. 메인 화면(런처)에서 빈 공간을 우클릭
2. "새 프로젝트" 선택
3. 이름 입력 (예: "투자", "부동산")

프로젝트 아이콘을 더블클릭하면 프로젝트 화면으로 들어갑니다.

### 에이전트 설정하기

프로젝트 화면 좌측에 에이전트 목록이 있습니다.

**새 에이전트 추가:**
1. 에이전트 목록 하단의 "+" 버튼
2. 이름과 역할 설명 입력

**역할 설명 예시:**
```
당신은 20년 경력의 내과 전문의입니다.
환자의 걱정을 먼저 인정한 후 질문합니다.
의학 용어는 일상 언어로 설명합니다.
```

**AI 모델 선택:**
에이전트별로 다른 AI를 쓸 수 있습니다. 설정에서 프로바이더(Anthropic, OpenAI, Google, Ollama)와 모델을 선택합니다.

### AI의 장기 메모리: 메모 활용하기

IndieBiz OS의 각 AI에는 "메모" 기능이 있습니다. 이것은 AI의 장기 메모리 역할을 합니다. 대화가 아무리 길어져도 메모에 적힌 내용은 항상 AI에게 전달됩니다.

**메모의 종류:**
- **시스템 메모**: 시스템 AI가 기억할 내용 (설정 화면에서 편집)
- **노트**: 프로젝트 에이전트가 기억할 내용 (에이전트 설정에서 편집)
- **근무지침**: 자동응답 AI가 기억할 내용 (비즈니스 설정에서 편집)

**메모에 적으면 좋은 것:**
- 사용자의 선호나 상황 (예: "나는 당뇨가 있다", "서울 강남에 산다")
- 반복적으로 언급하기 싫은 맥락 (예: "React 프로젝트다", "타겟 고객은 20대 여성이다")
- AI가 지켜야 할 규칙 (예: "항상 한국어로 응답", "가격 정보는 직접 확인 후 답변")

**주의: 짧을수록 좋다**

메모는 매 대화마다 AI에게 전달됩니다. 메모가 길면:
- 토큰 낭비 → 비용 증가, 응답 속도 저하
- AI 혼란 → 너무 많은 지시는 오히려 품질 저하
- 핵심 희석 → 중요한 내용이 묻힘

**권장:** 메모는 5~10줄 이내로, 정말 중요한 것만 적으세요. "있으면 좋겠다" 수준의 내용은 필요할 때 대화로 전달하는 게 낫습니다.

### 도구 할당하기

에이전트가 어떤 도구를 사용할 수 있는지 설정합니다.

1. 에이전트 설정 화면에서 "도구" 탭
2. 사용할 도구 체크

**원칙:** 에이전트는 할당된 도구만 봅니다. 투자 에이전트에게 투자 도구만 주면, 불필요한 도구로 혼란스러워하지 않습니다.

### 도구 패키지 설치/제거

상단 메뉴에서 "도구 상자"를 클릭하면 패키지 관리 화면이 열립니다.

- **설치됨**: 현재 사용 가능한 패키지
- **미설치**: 설치 가능한 패키지

패키지를 설치하면 해당 도구들을 에이전트에게 할당할 수 있습니다.

### 도구 패키지 공유 (Nostr)

내가 만든 도구 패키지를 다른 IndieBiz 사용자들과 공유할 수 있습니다.

**패키지 공개하기:**
1. 도구 상자에서 설치된 패키지의 "Nostr에 공개" 버튼 클릭
2. 설치 방법이 AI에 의해 자동 생성됨 (수정 가능)
3. 선택적으로 사인(서명) 추가
4. 공개 버튼 클릭

공개된 패키지는 `#indiebizOS-package` 해시태그로 Nostr 네트워크에 게시됩니다.

**다른 사람의 패키지 검색/설치:**
1. 도구 상자에서 "도구 검색" 버튼 클릭
2. Nostr 네트워크에서 공개된 패키지 검색
3. 패키지 선택 후 상세 정보 확인
4. "설치하기" 클릭 시 시스템 AI가 검토 후 설치

시스템 AI는 설치 전 보안, 품질, 호환성을 검토합니다.

## 4. 스위치

반복하는 작업을 저장해두고 원클릭으로 실행하는 기능입니다.

### 스위치 만들기

1. 프로젝트 화면 상단의 "스위치" 버튼
2. "새 스위치" 클릭
3. 이름과 프롬프트 입력

**예시:**
- 이름: "오늘의 뉴스"
- 프롬프트: "AI, 블록체인, 스타트업 관련 뉴스를 검색해서 5줄로 요약해줘"

### 스위치 실행

스위치 목록에서 실행 버튼을 누르면 해당 프롬프트가 지정된 에이전트에게 전달됩니다. 결과는 outputs 폴더에 저장할 수도 있습니다.

## 5. IndieNet (P2P 네트워크)

Nostr 프로토콜 기반의 탈중앙 네트워크입니다.

### 설정

1. 상단 메뉴에서 "나의 정보"
2. Nostr 키 생성 또는 기존 키 입력

### 기능

- **게시판**: 다른 IndieBiz 사용자들과 공개 게시물 공유
- **암호화 DM**: 1:1 비공개 메시지

### 비즈니스 네트워크

- **이웃 관리**: 비즈니스 파트너 등록
- **자동응답 V3**: Tool Use 기반 단일 AI 호출로 판단/검색/발송 통합
  - 메시지 수신 → AI가 의도 판단 → 비즈니스 DB 검색 → 즉시 응답 발송
  - 스팸/광고는 자동으로 필터링 (응답 안 함)
  - Gmail과 Nostr 모두 동일한 로직으로 처리

### 다중채팅방

여러 에이전트를 한 채팅방에 초대하여 복합적인 주제를 함께 논의할 수 있습니다.

- **별도 창**: 각 채팅방이 독립 창에서 열림
- **에이전트 소환**: 모든 프로젝트에서 에이전트를 대화로 초대
- **@지목**: @이름 문법으로 특정 에이전트 호출
- **도구 할당**: 각 에이전트에 특정 도구 할당

**예시**: "부동산 갭투자 vs 주식 투자, 세금까지 고려하면 뭐가 나을까?"
→ 투자, 부동산, 세무 에이전트가 각자 관점에서 의견을 제시합니다.

## 6. 설계 철학

IndieBiz OS는 "만능 AI 하나"가 아닌 "전문가 팀 + 인간 오케스트레이션"을 지향합니다.

- 인간이 프로젝트 구조를 설계합니다
- 각 에이전트는 자기 분야의 도구만 가집니다
- 인간이 프로젝트들 사이를 연결하고 판단합니다
- 패턴이 보이면 스위치로 자동화합니다

모든 데이터는 로컬에 저장됩니다. 클라우드에 의존하지 않습니다.

## 9. 예제: 나만의 환경 구축하기

빈 IndieBiz OS를 어떻게 나만의 시스템으로 만들어가는지 몇 가지 예를 보여드립니다.

### 예제 1: 투자 환경 구축

**목표**: 한국/미국 주식을 분석하고 시장 뉴스를 추적하는 환경

**1단계 - 도구 설치**
- 패키지 관리에서 `investment` 패키지 설치
- 필요하다면 `web` 패키지도 설치 (뉴스 검색용)

**2단계 - 프로젝트 생성**
- "투자"라는 이름의 프로젝트 생성

**3단계 - 에이전트 구성**
- 에이전트 이름: "투자분석가"
- 역할: "당신은 10년 경력의 애널리스트입니다. 데이터에 기반해 분석하고, 리스크를 먼저 언급합니다."
- 도구 할당: 주가조회, 재무제표, 공시조회, 웹검색

**4단계 - 스위치 설정 (선택)**
- "일일 시황": "오늘 코스피/나스닥 동향과 주요 뉴스 요약해줘"
- "관심종목 체크": "삼성전자, 애플, 테슬라의 현재가와 최근 공시 알려줘"

이제 투자 프로젝트에 들어가면 투자 전용 AI가 투자 도구만으로 응답합니다.

### 예제 2: 의료 상담 환경 구축

**목표**: 건강 질문에 답하고 복용약을 관리하는 환경

**1단계 - 도구 설치**
- `medical` 패키지 설치 (약물 정보, 상호작용 조회)

**2단계 - 프로젝트 생성**
- "의료"라는 이름의 프로젝트 생성

**3단계 - 여러 에이전트 구성**
- **내과**: "당신은 내과 전문의입니다. 환자의 걱정을 먼저 인정하고, 의학용어는 쉽게 설명합니다."
- **가정의학과**: "당신은 가정의학과 전문의입니다. 예방의학과 생활습관 개선에 초점을 맞춥니다."
- **복약관리**: "당신은 약사입니다. 복용약 간 상호작용을 체크하고 복용 스케줄을 관리합니다."

각 에이전트에게 필요한 도구만 할당합니다. 복약관리 에이전트에게는 약물상호작용 조회 도구를, 내과 에이전트에게는 증상 검색 도구를.

### 예제 3: 개발 보조 환경 구축

**목표**: 코드 리뷰, 문서 작성, 버그 추적을 돕는 환경

**1단계 - 도구 설치**
- `file` 패키지 (파일 읽기/쓰기)
- `web` 패키지 (문서 검색)
- 필요하다면 직접 GitHub API 도구를 만들어 추가

**2단계 - 프로젝트 생성**
- "개발"이라는 이름의 프로젝트 생성

**3단계 - 에이전트 구성**
- **코드리뷰어**: "당신은 시니어 개발자입니다. 코드의 버그, 성능 문제, 가독성을 체크합니다."
- **문서작성자**: "당신은 테크니컬 라이터입니다. 복잡한 코드를 명확한 문서로 정리합니다."

### 예제 4: 콘텐츠 제작 환경 구축

**목표**: 슬라이드와 홍보 영상을 자동 생성하는 환경

**1단계 - 도구 설치**
- `media_producer` 패키지 설치
- 필요하다면 `web` 패키지도 설치 (이미지 검색용)

**2단계 - 프로젝트 생성**
- "콘텐츠"라는 이름의 프로젝트 생성

**3단계 - 에이전트 구성**
- **디자이너**: "당신은 프레젠테이션 디자이너입니다. 2025-2026 디자인 트렌드를 반영한 슬라이드를 만듭니다."
- 도구 할당: create_slides, render_html_to_image, generate_ai_image, create_video

**4단계 - 사용 예시**
- "회사 소개 슬라이드 5장 만들어줘. 다크 테마로."
- "이 슬라이드들로 나레이션 포함 홍보 영상 만들어줘."
- "AI 이미지로 미래도시 배경 생성해줘."

**media_producer 테마 종류**:
- 기본: `modern`, `tech`, `business`
- 트렌드: `title_bold`(대형 제목), `dark_tech`(네온), `glassmorphism`(유리효과), `gradient_modern`(그라데이션), `split_asymmetric`(비대칭), `minimal_white`(미니멀), `image_fullscreen`(전체배경), `data_card`(데이터강조)

### 핵심 원칙

이 예제들에서 보이는 패턴:

1. **필요한 도구만 설치** - 모든 패키지를 설치할 필요 없음
2. **관련 있는 일끼리 묶기** - 프로젝트로 맥락 분리
3. **에이전트에게 명확한 역할** - 페르소나가 응답 품질을 높임
4. **도구는 선택적 할당** - 에이전트가 필요한 것만 보게

시간이 지나면서 자신만의 프로젝트, 에이전트, 스위치가 쌓이고, 그것이 "나만의 AI 운영체제"가 됩니다.

## 10. 원격 접근 (Finder & 런처)

집에 있는 PC를 어디서든 접근할 수 있는 개인 클라우드로 만드는 기능입니다. Cloudflare Tunnel을 통해 파일 접근과 AI 제어를 할 수 있습니다.

### 두 가지 원격 기능

| 기능 | 경로 | 목적 |
|-----|------|------|
| **원격 Finder** | `/nas/app` | 파일 탐색, 동영상 스트리밍, 다운로드 |
| **원격 런처** | `/launcher/app` | 시스템 AI/에이전트 채팅, 스위치 실행 |

각 기능은 **별도의 비밀번호**로 독립적으로 활성화할 수 있습니다.

### 무엇을 할 수 있나요?

**원격 Finder:**
- **파일 탐색**: 스마트폰이나 다른 컴퓨터에서 홈 PC의 파일을 브라우저로 탐색
- **동영상 스트리밍**: 여행 중에도 집에 저장된 영화/드라마를 스트리밍 재생
- **문서 미리보기**: 텍스트 파일 내용을 빠르게 확인
- **파일 다운로드**: 필요한 파일을 어디서든 내려받기

**원격 런처:**
- **시스템 AI 채팅**: 어디서든 시스템 AI에게 작업 지시
- **에이전트 채팅**: 프로젝트별 AI 에이전트와 대화
- **스위치 원탭 실행**: 자동화된 작업을 한 번의 터치로 실행
- **모바일 최적화**: 다크 테마, 반응형 UI

### 왜 Cloudflare Tunnel인가?

일반적으로 외부에서 집 PC에 접속하려면 포트 포워딩, DDNS, SSL 인증서 등 복잡한 네트워크 설정이 필요합니다. Cloudflare Tunnel은 이 모든 것을 대신해줍니다:

- **포트 오픈 불필요**: 공유기 설정 없이 안전하게 연결
- **자동 HTTPS**: Cloudflare가 SSL 인증서를 자동 관리
- **DDoS 보호**: Cloudflare 네트워크가 공격을 차단
- **전세계 엣지**: 어디서 접속하든 빠른 응답

### 시작하기 전에 필요한 것

원격 접근을 사용하려면 **Cloudflare 계정**이 필요합니다.

**1. Cloudflare 가입**
- https://dash.cloudflare.com 에서 무료 계정 생성
- 자신이 소유한 도메인을 Cloudflare에 연결 (무료)
  - 도메인이 없다면 저렴한 도메인을 하나 구매하는 것을 추천

**2. API 토큰 발급**
- Cloudflare 대시보드 → My Profile → API Tokens
- "Create Token" 클릭
- "Edit Cloudflare Workers" 템플릿 선택 또는 커스텀 권한 설정
- 생성된 토큰 복사

**3. Account ID 확인**
- Cloudflare 대시보드 우측 사이드바에서 확인 가능
- 또는 도메인 선택 후 Overview 페이지 우측

**4. IndieBiz OS 설정**
- 설정 → 환경변수에서 다음 값 입력:
  - `CLOUDFLARE_API_TOKEN`: 발급받은 API 토큰
  - `CLOUDFLARE_ACCOUNT_ID`: 계정 ID

### 설정 방법 요약

**원격 Finder 설정:**
1. **런처 설정** → **원격 Finder** 탭
2. "NAS 활성화" 토글 켜기
3. 비밀번호 설정
4. 접근 허용할 폴더 추가 (예: `/Users/me/Videos`)

**원격 런처 설정:**
1. **런처 설정** → **원격 런처** 탭
2. "원격 런처 활성화" 토글 켜기
3. 비밀번호 설정 (원격 Finder와 별도 가능)

**터널 설정:**
시스템 AI에게 요청: **"원격 접근용 Cloudflare 터널을 설정해줘"**

시스템 AI가 터널 생성, DNS 설정, 설정 파일 생성을 자동으로 처리합니다.

### 터널 실행

설정이 완료되면 터널을 실행해야 합니다:

```bash
cloudflared tunnel run indiebiz
```

PC를 상시 서버로 사용하려면 이 명령을 시스템 시작 시 자동 실행되도록 설정하세요.

### 접속

브라우저에서 설정한 주소로 접속합니다:
```
https://home.yourdomain.com/nas/app      # 파일 접근
https://home.yourdomain.com/launcher/app # AI 제어
```
