# IndieBiz OS

**개인과 소규모 비즈니스를 위한 AI 운영체제**

[홈페이지](https://indiebiz-homepage.vercel.app) | [English](README.md) | 한국어

IndieBiz OS는 단순한 AI 챗봇이 아닙니다. 나만의 AI 인격을 설계하고, 원클릭으로 작업을 자동화하며, 탈중앙 네트워크로 연결되는 완전한 운영체제입니다.

---

## 설치 방법 (권장)

가장 쉬운 설치 방법은 **Claude Desktop**을 이용하는 것입니다.

1. **Claude Desktop**을 열고 **Claude Code** 탭으로 전환
2. Claude에게 말하세요:

```
"https://github.com/kangkukjin/indiebizOS 를 내 PC에 설치해줘"
```

3. Claude가 알아서 처리합니다:
   - 저장소 클론
   - 의존성 설치 (Python, Node.js)
   - 사용자의 필요와 선호를 질문
   - 맞춤형 시스템 구성

> **왜 Claude Desktop인가?**  IndieBiz OS는 `npm install`로 끝나는 정적인 앱이 아닙니다. 설치하는 AI 에이전트가 곧 당신의 시스템을 계속 단조하는 대장장이가 됩니다. Claude Desktop의 Claude Code 탭은 이 대화형 맞춤 설치에 가장 적합한 환경입니다.

**필요한 것:**
- **LLM API 키** (Anthropic, Google, 또는 OpenAI)
- **몇 가지 질문에 대한 답변** — 시스템으로 무엇을 하고 싶은지
- **외부 API 키** — 해당 기능을 실제로 사용할 때만 필요

## 핵심 가치

### 1. 나만의 AI 인격 설계

단순히 "의사처럼 행동해"가 아닙니다. **누구**인지, **어떻게** 대화하는지를 정의하세요.

```yaml
# 예시: 따뜻한 내과 전문의
agents:
  김_원장:
    role: |
      당신은 20년 경력의 내과 전문의 김원장입니다.
      항상 환자의 걱정을 먼저 인정한 후 질문합니다.
      의학 용어는 일상 언어로 설명합니다.
      모든 상담은 명확한 다음 단계와 안심의 말로 마무리합니다.
    model: claude-sonnet
    allowed_nodes: [sense, self, limbs, others, engines]
```

| 일반 AI | IndieBiz 인격 |
|---------|--------------|
| "고혈압일 수 있습니다. 병원에 가보세요." | "걱정되시는 마음 충분히 이해합니다. 몇 가지 먼저 여쭤볼게요... 기록을 보니, 이렇게 하시면 좋겠습니다..." |
| 정보 전달 | 맥락을 가진 공감 대화 |

**각 에이전트는 당신의 맥락을 기억합니다** - 복용 중인 약, 선호사항, 지난 대화들. 범용 어시스턴트가 아니라 **당신만의** 전문가입니다.

### 2. 스위치 & 워크플로우로 원클릭 자동화

같은 AI 대화를 반복하지 마세요. **스위치나 워크플로우로 저장**하고 원클릭으로 실행하세요.

**스위치** - 프롬프트 + 에이전트 쌍을 저장해서 원클릭 실행:
```
[토글] 일일 기술 뉴스  →  클릭  →  끝!
         ↓
    AI가 정의된 프롬프트 실행
         ↓
    결과가 outputs/daily_news.md에 저장
```

**워크플로우** - IBL 문법으로 여러 액션을 재사용 가능한 파이프라인으로 연결:
```
[sense:search_ddg]{query: "AI 뉴스"}
  >> [others:ask_sync]{agent_id: "컨텐츠/컨텐츠", message: "이 기사들을 요약해줘"}
  >> [self:file]{path: "news_report.html", format: "html"}
```

**기능:**
- **자연어 또는 IBL 코드** - 평소처럼 프롬프트를 쓰거나 액션 체인을 프로그래밍
- **예약 실행** - 새로운 **스케줄러**를 통해 매일 오전 8시, 매주 금요일 자동 실행
- **에이전트 지정** - 어떤 에이전트가 처리할지 지정
- **파이프라인 연산자** - 순차(>>), 병렬(&), 폴백(??)
- **별칭 실행** - 저장된 워크플로우를 이름으로 실행: `[self:run]{workflow_id: "kinsight"}`

### 3. P2P 네트워크 (IndieNet) & 원격 접근

탈중앙화된 네트워크로 연결하고 어디서나 시스템에 접속하세요.

- **Nostr 프로토콜** - 중앙 서버 없는 탈중앙 DM 및 공개 게시판 (DM은 최신 NIP-17 gift-wrap 암호화)
- **폰 네이티브 — 두 번째 독립 자아** - 네이티브 안드로이드 앱이 IndieBiz OS를 *폰에서 직접* 구동하며, **폰 자체의 LLM 두뇌**(인-프로세스 Gemini — 분류용 경량 티어 + 실행용 본격 티어)를 가집니다. 폰은 집 PC의 리모컨이 아니라 *다른 몸*을 가진 두 번째 자아이며, 다른 몸이므로 다른 정체성입니다. 온디바이스 파이썬 백엔드(Chaquopy)가 앱모드 슈퍼앱을 서빙하고, 실제 IBL 엔진이 폰 안전 패키지를 로컬 실행하며, 폰에서 파이썬을 인-프로세스로 실행할 수도 있어 일종의 기기 프로그래밍 에이전트가 됩니다. 모델은 **자기 하드웨어를 감지**(`detect_body`)해 자신을 맥이 아닌 "폰"으로 인식합니다. `runs_on` 능력 태그가 각 액션을 정직하게 표시 — `anywhere`(각 몸 로컬 실행)·`mac_only`(폰→맥 포워드)·`phone_only`(맥→폰 포워드) — 빌린 액션은 leaf 단위로 투명하게 포워드됩니다. 폰은 자체 온디맨드 감각(`[sense:here]` 위치·`[sense:listen]` 마이크·`[sense:see]` 카메라·`[sense:phone]` 알림)을 가지고, 네이티브 AccessibilityService로 자기 화면을 조작합니다. **기억은 두 부류:** 사용자 세계-데이터(연락처·비즈니스·일정·의료기록)는 공유·동기화(CRDT 합집합 머지)하고, 각 자아의 주관적 기억(대화·해마·자기상태)은 그 자아의 사적 기억으로 동기화하지 않습니다.
- **원격 접근 (Phase 25)** - Cloudflare Tunnel 기반 원격 Finder(파일 스트리밍) 및 원격 런처(AI 제어/스위치 실행)
- **비즈니스 네트워크** - 파트너 관리, **자동응답 V3**(Tool-use 기반)를 통한 지능형 문의 응대

### 4. 에이전트 자기 조절 (Phase 26)

에이전트는 단순한 도구가 아닙니다 — **언제 행동하고, 언제 멈추고, 언제 전략을 바꿀지** 스스로 판단합니다.

- **목표/시간/조건 시스템** — 태스크별 목표, 시간 제한, 조건부 트리거를 설정. 목표 달성 또는 시간 초과 시 에이전트가 자동 종료.
- **전략 에스컬레이션** — 같은 접근 범주에서 3회 연속 실패 시 에이전트가 강제로 전략을 전환. 모든 범주가 소진되면 반복하지 않고 사용자에게 보고.
- **라운드 메모리** — 모든 시도를 접근 범주, 결과, 배운 점과 함께 구조적으로 기록. 에이전트가 다음 시도 전에 과거 이력을 검토하여 중복 접근 방지.

### 5. 3단 인지 아키텍처 (반사, 계획, 성찰)

대부분의 AI 시스템은 모든 요청을 동일하게 처리합니다. IndieBiz OS는 인간의 인지 과정을 모델링하여 **세 가지 에이전트**가 순차적으로 처리합니다:

```
사용자 메시지
    |
    v
[무의식 에이전트] -- 반사: EXECUTE or THINK?
    |                        |
    | EXECUTE (단순)          | THINK (복잡)
    v                        v
[직접 실행]            [의식 에이전트] -- 계획: 문제 정의 + 달성 기준
                             |
                             v
                      [AI 에이전트 실행]
                             |
                             v
                      [평가 에이전트] -- 성찰: 달성 여부 → 재시도?
                             |
                      (최대 3라운드)
```

**무의식 에이전트** (Phase 28) — 경량 모델을 사용하는 게이트키퍼. 각 요청을 EXECUTE(메타인지 불필요한 단순 작업) 또는 THINK(의식 처리가 필요한 복잡 작업)로 분류합니다. 단순 요청에 대한 불필요한 오버헤드를 제거합니다.

**의식 에이전트** — 복잡한 요청이 AI 에이전트에 도달하기 전에 메타적 판단을 수행합니다: "무엇을 해야 하는가"뿐 아니라 **"내가 가진 것으로 이것이 얼마나 어려운가?"** 를 판단합니다. 이제 **달성 기준(achievement_criteria)** 도 함께 출력하여 평가 에이전트가 활용합니다.

핵심 원칙: **문제는 나의 한계와 환경의 제약이 만나는 곳에서 생긴다.** 같은 명령이라도 어떤 도구가 있느냐에 따라 다른 문제가 됩니다.

- **태스크 프레이밍** — 도구의 한계를 인식한 상태에서 문제를 정의
- **달성 기준** — 실행 후 평가에 사용할 측정 가능한 성공 조건
- **자기 인식** — 각 IBL 액션의 `implementation` 필드로 도구 품질 판단
- **히스토리 정제** — 현재 문제와 관련된 맥락만 추려 요약
- **IBL 포커싱** — 관련 노드/액션의 구현 상세 포함
- **가이드 파일 선택** — 가장 관련 있는 가이드 문서 선택
- **상황 메모** — 세계 상태, 상황 인식 정보

**평가 에이전트** (Phase 28) — AI 에이전트가 응답한 후, 의식 에이전트의 달성 기준(achievement_criteria)에 대해 결과를 평가합니다. NOT_ACHIEVED이면 구체적인 피드백과 함께 재실행합니다. 최대 3라운드로 무한 루프를 방지합니다.

시스템 AI와 모든 프로젝트 에이전트에 적용됩니다. 내부 위임 메시지는 의식 에이전트를 거치지 않습니다.

### 3단계 모델 티어 (Phase 30)

비용/속도 최적화를 위한 모델 분리:

| 티어 | 용도 | 사용처 |
|------|------|--------|
| **경량** | 분류, 평가, 원샷 판단 | 무의식/평가 에이전트 |
| **중급** | Reflex 경로 실행 | 해마가 고확신(점수 ≥ 0.85)으로 이미 아는 작업 — 무의식 분류조차 건너뜀 |
| **본격** | EXECUTE + THINK 경로 실행 | EXECUTE는 의식을 건너뛰되 본격 모델 유지(오분류해도 잃는 건 프레이밍뿐, 지능이 아님); THINK는 의식 프레이밍 추가 |

경량/중급 AI의 API 키가 비어있으면 시스템 AI 키를 자동 사용합니다.

### Consciousness Pulse (자의식 시스템)

시스템이 자기 자신과 세계를 인식합니다:

- **World Pulse** — 매 1시간: 경제 delta, 날씨, 뉴스(6시간마다), 사용자 활동, 시스템 건강
- **Self-Check** — 매 12시간: 5개 노드 전체 대상 부작용 없는 IBL 액션 전수 자가 점검 (면역 순찰). 사용자가 "자기 점검해줘" 명령 시 시스템 AI가 실패 액션을 재시도해 transient/reproducible 분류 + 수정 난이도(쉬움/중간/어려움) 평가
- **건강 모니터링** — 서비스 alive 체크, 디스크 사용량, 이상 감지

## 런처 — 세 가지 표면

한 작업을 세 가지 방식으로 실행할 수 있습니다. 모두 **같은 IBL 신경계 위에 얹힌 표면**이며, 각각 **{속도·표현력·주권}** 중 둘을 갖고 하나를 내줍니다.

| 모드 | 무엇 | 트레이드오프 |
|------|------|--------------|
| **자율주행** | 고급 AI에게 의도만 말하면 알아서 다단계로 처리. 새롭고 일회성·탐색적인 일에 적합. (구 '프로젝트') | 속도+표현력 −주권 |
| **수동** | 경량 모델이 해마 기억에 기대 자연어 명령을 IBL로 *번역*하고, 사용자가 효과(dry-run)로 검수·편집한 뒤 실행. **컴파일러 프론트엔드** — 모델은 번역만, 지능은 언어(IBL)에 쌓임. 비용 거의 0. | 표현력+주권 −속도 |
| **앱** | 아이콘을 눌러 GUI로 직접 조작 (예: 부동산 실거래가·상권, 도서검색). 가장 빠르고 발견 가능성 최고. (구 '액션') | 속도+주권 −표현력 |

작업의 **빈도**가 모드 사이로 흘려보냅니다: 자율주행이 한 번 해보고(탐색) → 그 IBL 흔적이 수동의 초안으로 → 자주·안정적으로 반복되면 앱으로 결정화. **굳히는 건 증명된 것만.**

수동 모드의 두 규칙: **부작용 있는 단계는 명시적 확인을 거쳐야 실행**되고(읽기 전용은 무마찰), 해마는 **사용자가 인정한 결과만** 학습합니다.

앱 모드 계기는 **선언 기반**입니다: 각 계기는 IBL 액션의 `app:` 블록(`data/ibl_nodes_src/`)이며, **하나의 선언이 데스크탑·원격 런처·폰에 똑같이 렌더**됩니다 — 새 앱 하나 = UI 셋이 아니라 선언 하나. 지도·네이티브 창처럼 더 풍부한 손제작 계기 몇 개만 escape hatch로 남습니다.

## 통합 AI 아키텍처 (Phase 25: 5-Node 통합)

IndieBiz OS는 **시스템 AI**와 **프로젝트 에이전트**가 동일한 코드베이스와 도구 구조를 공유하는 통합 AI 코어를 특징으로 합니다.

- **execute_ibl 단일 도구**: 모든 AI 엔티티가 하나의 `execute_ibl` 도구로 시스템의 모든 기능에 접근합니다.
- **노드 추상화**: 5개 노드(sense, self, limbs, others, engines)와 124개의 원자적 액션으로 구성됩니다.
- **액션 라우팅 이원화**: `api_engine`(자동 API 발견)과 `handler`(복잡 로직 처리)를 통해 효율적으로 실행됩니다.
- **사용자 소통의 통합**: 질문(`[self:ask_user]`), 할일(`[self:todo]`), 승인(`[self:approve]`), 알림(`[self:notify_user]`)이 모두 `self` 노드로 통합되었습니다.

## 핵심 아키텍처: IBL (IndieBiz Logic)

IndieBiz OS는 AI 에이전트와 시스템 사이의 통합 인터페이스로 **IBL**이라는 언어를 사용합니다.

### 5개 노드, 124개 원자적 액션

| 노드 | 액션 수 | 설명 |
|------|---------|-------------|
| **sense** | 41 | 데이터 검색 (웹, 네이버 한국어 검색, 금융, 여행, 사진, 블로그, 건강, 부동산, 법률, 통계, 고전 원문, 공연·전시)과 폰 온디맨드 감각 (알림·위치·마이크·카메라) |
| **self** | 41 | 시스템 관리, 워크플로우, 트리거, 파일, 심층 메모리, 비즈니스(사업·아이템·문서·지침), 폰 동기화, 캘린더, 건강 기록 |
| **limbs** | 17 | UI 자동화 (브라우저, 안드로이드 폰, macOS 화면), 폰 네이티브 동작, 미디어 재생 (유튜브, 라디오), 지도 |
| **others** | 11 | 협업·위임, 메시징(DM/피드/보드/Nostr), 이웃 CRM, 연락처, 자동응답 |
| **engines** | 14 | 콘텐츠 생성 (슬라이드, 영상, 차트, 이미지, 웹사이트, 스프레드시트, TTS) |

> 관련 도구들을 단일 액션 + 파라미터/`op` 분기로 통합하면서 액션 수가 332개에서 124개로 줄었습니다 (예: 안드로이드 전용 액션 45개가 `[limbs:android]{op}` 하나로) — AI가 학습할 어휘는 더 적게, 조합은 더 쉽게. 액션이 식별자를 내부에서 해소(도시명→코드, 자연어 날짜, cron 문자열→일정)하므로 사람이든 경량 모델이든 한 줄만 쓰면 동작합니다.

### 작동 방식

```
사용자: "AI 뉴스 검색하고 파일로 저장해"
         ↓
AI 판단: execute_ibl(code='[sense:search_ddg]{query: "AI 뉴스"} >> [self:file]{path: "result.md"}')
         ↓
IBL 엔진: 파싱 → 핸들러/API 엔진 디스패치 → 결과 반환
```

**핵심 설계 원칙:**
- **하나의 도구, 하나의 언어** - AI 에이전트는 하나의 문법(`execute_ibl`)만 학습
- **동적 도구 정의** - 설치된 패키지에 따라 도구 설명이 런타임에 자동 생성
- **에이전트별 필터링** - `allowed_nodes`를 통한 권한 제어
- **용례 RAG** - 과거 성공 사례를 검색하여 참조로 주입, 시행착오 최소화
- **원자적 액션** - 대부분의 액션이 단일 API/DB 호출로 구성
- **감각 피드백** - 파이프라인 결과를 전체 누적하여 전달 (절삭 없음), `>>` 에러 시 즉시 중단, `_action_count × 16KB` 동적 한도로 AI가 모든 중간 결과를 확인
- **$file:N 파라미터** - `files` 파라미터로 긴 코드/텍스트를 IBL 파서 밖에서 전달, IBL 코드에서 `$file:0`, `$file:1`로 참조

## 주요 기능

### 프로젝트 시스템 (무제한 작업공간 · 런처의 '자율주행' 모드)
- 필요한 만큼 프로젝트 생성 (현재 24개 활성 프로젝트)
- 각 프로젝트는 자체 대화 기록, 에이전트, 맥락 보유
- 런처의 자율주행 모드 — 의도를 말하면 에이전트가 다단계로 처리 (수동·앱 모드는 위 "런처 — 세 가지 표면" 참조)

### 에이전트 팀 (위임 체인)
- 프로젝트별로 다른 페르소나를 가진 여러 에이전트 정의
- 에이전트 간 작업 위임(동기/비동기) 및 프로젝트 간 위임 지원

### 3단계 위임 시스템 (Phase 27)

| 방식 | 설명 | 적합한 상황 |
|------|------|------------|
| **동기 위임** (`call_agent`) | 에이전트가 다른 에이전트를 직접 호출하고 결과를 기다림 | 즉시 응답이 필요한 단순 작업 |
| **스케줄 위임** (`schedule`) | 다른 에이전트 소유의 스케줄을 등록하여 시간 기반으로 일을 맡김 | 정기적/지연 실행이 필요한 작업 |
| **작업계획서 위임** (`create_plan` + `execute_plan`) | 자연어 계획서를 작성하고 에이전트들이 순서대로 실행 | 다단계 복잡 워크플로우 |

**스케줄 위임** - `target_project_id`/`target_agent_id`를 지정하면 해당 에이전트가 스케줄 소유자가 되어 지정 시간에 자동 실행:
```
[self:schedule]{at: "09:00",
  target_project_id: "투자", target_agent_id: "analyst",
  pipeline: "[sense:search_ddg]{query: '오늘 뉴스'}"}
```

**작업계획서 위임** - 구조화된 계획서를 통한 멀티에이전트 순차 실행:
```
[self:create_plan]{title: "데이터 분석", steps: [...]}
  → [self:execute_plan]{file: "plan.md"}
  → 에이전트가 자기 파트 실행 → 계획서 상태 업데이트 → 다음 에이전트에게 위임
```
- 단계별 상태 추적 (`대기` → `진행중` → `완료`/`실패`)
- 실패 시 자동 재시도 및 대안 경로 지원

### 메모리: 경험으로부터 학습

두 기억이 대화에서 자동으로 배우고, 스스로를 청소합니다:

- **해마 (절차기억)** — fine-tuned 768차원 임베딩 모델이 자연어를 과거 IBL 코드로 연결합니다 (홀드아웃 테스트 코드 Top-5 ~92.6% / 설명 ~92.8%; Apple Silicon 로컬 재학습, 코퍼스 ~2,400건). 성공한 실행은 자동으로 재사용 가능한 용례로 증류되어, *당신의* 반복 작업이 점점 빨라집니다.
- **심층메모리 (관계기억)** — 매 대화 후 경량 패스가 당신에 대한 지속적 사실(선호·결정·중요 날짜)을 추출하고, 마지막 확인일과 함께 관련 시점에 회상합니다.
- **하이브리드 검색** — 시맨틱 검색 + 키워드(FTS5) 결합
- **닫힌 피드백 루프** — 회상된 용례가 실행을 주도하면 그 성공/실패가 기록되어, 입증된 패턴은 떠오르고 나쁜 것은 가라앉습니다. 증류된 코드는 코퍼스 진입 전에 액션 레지스트리로 검증됩니다.
- **자정(정리 패스)** — 주기적 정리(면역 순찰의 일부)가 근접중복을 병합하고, 낡거나 입증된 나쁜 항목을 가지치고, 모순을 정정해 기억이 비대해지지 않고 날카롭게 유지됩니다.

### 고급 자동화 및 원격 제어
- **스케줄러**: 정기적인 정보 수집 및 리포트 생성 자동화
- **자동응답 V3**: Tool-use 기반 판단으로 지능적인 메시지 응대
- **원격 Finder/런처**: 보안 터널을 통해 어디서나 파일 접근 및 AI 제어
- **실시간 CCTV (UTIC + ITS + Windy)**: UTIC 실시간 API로 전국 시내 도로 CCTV 16,000+대, ITS 고속도로, Windy 전세계 웹캠 통합

## 기술 스택

- **Backend**: Python FastAPI (25개 모듈 라우터, AI 에이전트 코어)
- **Frontend**: Electron + React (WebSocket 기반 실시간 스트리밍)
- **AI Providers**: Anthropic (Claude), OpenAI (GPT), Google (Gemini), Ollama (Local)
- **Database**: SQLite (비즈니스, 채팅, IBL RAG 데이터)
- **Deployment**: 로컬 실행 및 Cloudflare Tunnel을 통한 원격 접근 지원
