# IndieBiz OS

**개인과 소규모 비즈니스를 위한 AI 운영체제**

[English](README.md) | 한국어

IndieBiz OS는 단순한 AI 챗봇이 아닙니다. 무한한 AI 에이전트 팀을 만들 수 있는 완전한 운영체제입니다. 각 팀은 독립된 프로젝트 작업공간에서 전문화된 도구들과 함께 일합니다.

---

## 왜 IndieBiz OS인가?

### "만능 AI"의 문제점

대부분의 AI 시스템은 하나의 전지전능한 어시스턴트를 목표로 합니다. 이 접근 방식은 다음과 같은 이유로 실패합니다:

- **맥락 오염**: 의료 상담 내용이 하드웨어 프로젝트와 섞입니다
- **도구 과부하**: 100개의 도구는 단일 에이전트를 혼란스럽게 합니다
- **자동화 부재**: 모든 작업을 수동으로 시작해야 합니다
- **분리 불가**: 작업 영역을 쉽게 삭제하거나 격리할 수 없습니다

### IndieBiz의 접근 방식

> "하나의 만능 AI보다 여러 목적별 AI가 낫다"

```
IndieBiz OS
├── 의료 프로젝트
│   ├── 내과 에이전트 (GPT-4o-mini, 저렴)
│   ├── 정형외과 에이전트 (Claude Sonnet)
│   └── 약사 에이전트 (로컬 LLM, 무료)
│
├── 부동산 프로젝트
│   ├── 세무 에이전트 (Claude Opus, 정확)
│   └── 법률 에이전트 (Claude Sonnet)
│
└── 창업 프로젝트
    ├── 마케팅 에이전트
    └── 개발 에이전트
```

각 프로젝트는 **완전히 격리**됩니다. 프로젝트를 삭제하면 관련된 모든 것이 깔끔하게 사라집니다.

## 주요 기능

### 1. 프로젝트 시스템 (무제한 작업공간)

- 필요한 만큼 프로젝트 생성
- 각 프로젝트는 자체 대화 기록, 에이전트, 컨텍스트 보유
- 프로젝트 복사로 변형 버전 생성
- 다른 프로젝트에 영향 없이 삭제

### 2. 에이전트 팀 (위임 체인)

- 프로젝트별로 다른 역할의 여러 에이전트 정의
- 에이전트 간 작업 위임 가능
- 병렬 위임: 하나의 에이전트가 여러 에이전트에게 동시 배포
- 체인을 통한 자동 결과 보고

```yaml
# 예시: agents.yaml
agents:
  coordinator:
    role: "요청을 전문가에게 라우팅"
    model: claude-sonnet

  researcher:
    role: "깊은 연구와 분석"
    model: claude-opus
    allowed_tools: [web-search, web-crawl]

  coder:
    role: "코드 작성 및 실행"
    model: gpt-4o
    allowed_tools: [python-exec, nodejs]
```

### 3. 동적 도구 로딩

- 도구는 전역이 아닌 **에이전트별**로 로드
- 1000개의 도구를 추가해도 단일 에이전트가 압도당하지 않음
- 각 전문가는 필요한 도구만 확인

**설치된 도구 패키지:**

| 패키지 | 설명 |
|--------|------|
| android | 안드로이드 기기 관리 (adb) |
| blog | 블로그 RAG 검색 및 인사이트 |
| browser-automation | 웹 자동화 (Playwright) |
| health-record | 개인 건강 데이터 관리 (측정값, 증상, 투약) |
| information | API Ninjas, 여행 정보, 맛집 검색 (카카오 API) |
| kosis | 통계청 데이터 조회 (KOSIS API) |
| nodejs | JavaScript/Node.js 실행 |
| python-exec | Python 코드 실행 |
| real-estate | 부동산 실거래가 (아파트/주택 매매/전월세) |
| startup | 창업 지원 정보 (K-Startup, 중소벤처기업부) |
| system_essentials | 파일 관리, 검색, 유틸리티 |
| web-crawl | 웹 페이지 크롤링 |
| web-search | 웹 검색 (DuckDuckGo, Google News) |
| youtube | YouTube 동영상/오디오 관리 |

### 4. 스케줄러 (자동 실행)

- cron 표현식으로 작업 예약
- 수동 트리거 없이 에이전트가 자동으로 작업
- 예시:
  - 매일 오전 9시 뉴스 요약
  - 매시간 서버 상태 확인
  - 주간 리포트 생성

### 5. 스위치 (이벤트 트리거)

- 에이전트 작업을 트리거하는 조건 정의
- 외부 이벤트에 자동 반응

### 6. 비즈니스 네트워크 (통신 채널)

- **Gmail 연동**: 이메일 수신 및 처리
- **Nostr 연동**: 실시간 탈중앙 메시징
- **이웃 관리**: 비즈니스 파트너 및 연락처 추적
- **자동 응답 V2**: 고급 프롬프트 엔지니어링을 적용한 AI 응답:
  - **Chain-of-Thought (CoT)**: 메시지 분류를 위한 단계별 추론
  - **Few-shot 학습**: 일관된 품질을 위한 예시 기반 응답 생성
  - 근무 지침, 비즈니스 문서, 대화 기록 참조
  - 환각 방지: 검색 결과에 있는 정보만 언급

### 7. 다중채팅방 (그룹 대화)

- **별도 창**: 각 채팅방이 독립 창에서 열림
- **에이전트 소환**: 모든 프로젝트에서 에이전트를 대화로 초대
- **AI 설정 유지**: 소환된 에이전트가 원래 프로바이더/모델/API 키 유지
- **전체 시작/중단**: 모든 참여자를 한 번에 활성화/비활성화
- **도구 할당**: 드롭다운으로 각 에이전트에 특정 도구 할당
- **@지목**: @이름 문법으로 특정 에이전트 호출

### 8. 시스템 AI (메타 컨트롤러)

- 모든 프로젝트 위에 위치
- 시스템 전체 설정 관리
- 시스템 문서를 장기 기억으로 참조
- 안내 및 조정 제공

## 빠른 시작

### 사전 요구사항

- Python 3.10+
- Node.js 18+
- AI 제공자 API 키 (Claude, OpenAI 등)

### 설치

```bash
# 저장소 클론
git clone https://github.com/your-repo/indiebizOS.git
cd indiebizOS

# 백엔드 설정
cd backend
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate
pip install -r requirements.txt

# 프론트엔드 설정
cd ../frontend
npm install
```

### 실행

```bash
# 옵션 1: 시작 스크립트 사용
./start.sh

# 옵션 2: 개별 실행
# 터미널 1 - 백엔드
cd backend && python api.py

# 터미널 2 - 프론트엔드
cd frontend && npm run electron:dev
```

### 빌드

```bash
cd frontend
npm run electron:build:mac   # macOS
npm run electron:build:win   # Windows
```

## 철학

### 휴먼 인 더 루프 AGI

우리는 단일 전지전능 AGI를 믿지 않습니다. 대신 이렇게 생각합니다:

> **AGI = 전문화된 AI 연합 + 인간**

```
        ┌─────────────────────────────────┐
        │            인간                 │
        │   (연결자, 판단자, 소유자)      │
        └─────────────┬───────────────────┘
                      │
        ┌─────────────┼─────────────┐
        ↓             ↓             ↓
   ┌─────────┐   ┌─────────┐   ┌─────────┐
   │  의료   │   │  금융   │   │  창업   │
   │   팀    │   │   팀    │   │   팀    │
   └─────────┘   └─────────┘   └─────────┘
```

**인간은 시스템의 일부**입니다. 단순한 사용자가 아닙니다:
- 다른 도메인 간의 인사이트를 연결
- 최종 결정을 내림
- 책임을 짐
- 윤리적 판단 제공

**단일 AGI보다 나은 이유:**

| 단일 AGI | IndieBiz 접근 방식 |
|----------|-------------------|
| 하나의 시스템이 모든 것을 결정 | 인간이 최종 판단 |
| 하나의 실패가 전체를 깨뜨림 | 실패가 격리됨 |
| 불명확한 책임 | 인간이 책임 |
| 환각이 퍼짐 | 도메인 전문가가 검증 가능 |
| 블랙박스 | 투명하고 모듈화됨 |

이것은 한계가 아닙니다—기능입니다. 범용 지능으로 가는 가장 안전하고 실용적인 길은 **인간 감독이 있는 분산 지능**입니다.

---

### 프로젝트 격리가 중요한 이유

1. **깔끔한 삭제**: 프로젝트 제거 시 모든 데이터 제거
2. **맥락 정확성**: 의료 에이전트가 하드웨어 코드를 보지 않음
3. **비용 최적화**: 필요한 곳에만 비싼 모델 사용
4. **실험**: 일회용 에이전트 생성, 프롬프트 자유롭게 테스트

### 여러 에이전트가 하나보다 나은 이유

- 단순 작업 → 저렴/로컬 모델 ($0)
- 중간 작업 → 중급 모델 ($0.01)
- 복잡한 작업 → 프리미엄 모델 ($0.10)

단일 "범용 에이전트"는 이를 최적화할 수 없습니다. IndieBiz는 가능합니다.

### 대안과의 비교

| 기능 | Claude Desktop | OpenDAN | IndieBiz OS |
|------|---------------|---------|-------------|
| GUI | 네이티브 앱 | CLI/Telegram | Electron 앱 |
| 프로젝트 격리 | 없음 | 없음 | 있음 |
| 무제한 에이전트 | 없음 | 제한적 | 있음 |
| 도구 확장성 | MCP (제한적) | 제한적 | 무제한 |
| 스케줄러 | 없음 | 없음 | 있음 |
| 비즈니스 네트워크 | 없음 | 없음 | 있음 |
| 자동 응답 | 없음 | 없음 | 있음 |
| 다중채팅방 | 없음 | 없음 | 있음 |

## 활용 사례

- **1인 기업가**: 고객 커뮤니케이션, 리서치, 콘텐츠 제작 자동화
- **컨설턴트**: 클라이언트별 별도 프로젝트, 전문 에이전트 팀
- **개발자**: 코드 실행, 문서화, 테스트 에이전트
- **투자자**: 시장 리서치, 포트폴리오 분석, 뉴스 모니터링
- **의료**: 별도 의료 도메인, 프라이버시 존중 격리

## 상태

**이 프로젝트는 활발히 개발 중입니다.**

현재 운영 중인 개인 프로젝트:
- 14개의 활성 프로젝트 운영 중
- 15개의 도구 패키지 설치됨
- 다중채팅방으로 그룹 대화 지원
- 스케줄러, 스위치, 비즈니스 네트워크 작동 중
- 고급 프롬프트 엔지니어링 (CoT + Few-shot) 적용

### 설계 철학: 당신만의 OS, 당신만의 방식

IndieBiz OS는 **각 사용자가 커스터마이즈**하도록 설계되었습니다. "올바른" 사용법은 없습니다.

- **포크하고 수정하세요**: 이것은 당신의 개인 운영체제입니다—무엇이든 바꾸세요
- **자신만의 도구 추가**: 당신의 워크플로우에 맞는 패키지 생성
- **자신만의 에이전트 정의**: 당신의 사고 방식에 맞는 팀 구성
- **정해진 구조 없음**: 당신에게 맞는 방식으로 프로젝트 구성

목표는 모두가 같은 방식으로 사용하는 제품을 만드는 것이 아니라, 각 사람이 고유한 필요에 맞게 적응할 수 있는 기반을 제공하는 것입니다. 완성품이 아닌 시작점이라고 생각하세요.

기여와 피드백을 환영합니다.
