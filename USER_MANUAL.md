## 1. 소개

### IndieBiz OS란?

IndieBiz OS는 단순한 AI 챗봇이 아닙니다. 당신만의 AI 페르소나를 설계하고, 작업을 원클릭으로 자동화하며, 탈중앙화 네트워크를 통해 연결하는 완전한 운영 체제입니다.

### 핵심 차별점

**기존 AI 서비스의 문제점:**
- 컨텍스트 오염: 의료 상담과 하드웨어 프로젝트가 섞임
- 개인화 부족: 모든 AI가 똑같이 말함
- 자동화 없음: 매번 수동으로 트리거
- 플랫폼 종속: 데이터가 그들의 서버에 있음

**IndieBiz OS의 해결책:**
- **프로젝트 격리**: 각 프로젝트가 완전히 독립된 작업 공간
- **AI 페르소나 설계**: "어떤" AI인지, "어떻게" 대화하는지 직접 정의
- **스위치 자동화**: 반복 작업을 원클릭으로 실행
- **데이터 주권**: 모든 데이터가 당신의 PC에 저장

### 세 가지 핵심 가치

1. **AI 페르소나 설계**: 단순히 "의사 역할을 해줘"가 아닌, 어떤 의사인지, 어떻게 대화하는지까지 정의
2. **원클릭 자동화 (스위치)**: 반복되는 AI 대화를 저장하고 한 번의 클릭으로 실행
3. **P2P 네트워크 (IndieNet)**: 탈중앙화, 검열 저항 네트워크를 통한 연결

## 2. 시작하기

### 2.1 시스템 요구사항

- Python 3.10 이상
- Node.js 18 이상
- AI 프로바이더 API 키 (Claude, OpenAI, Gemini 등) 또는 Ollama (로컬 LLM)

### 2.2 설치 방법

```bash
# 저장소 클론
git clone https://github.com/kangkukjin/indiebizOS.git
cd indiebizOS

# 백엔드 설정
cd backend
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate
pip install -r requirements.txt

# 프론트엔드 설정
cd ../frontend
npm install
```

### 2.3 실행 방법

```bash
# 방법 1: 시작 스크립트 사용
./start.sh

# 방법 2: 개별 실행
# 터미널 1 - 백엔드
cd backend && python api.py

# 터미널 2 - 프론트엔드
cd frontend && npm run electron:dev
```

### 2.4 의존성 설치 (처음 사용자)

IndieBiz OS를 처음 사용할 때는 필요한 라이브러리들을 설치해야 합니다.

**권장 방법: 시스템 AI에게 요청하기**

시스템 AI에게 다음과 같이 말하세요:
> "나는 이게 처음이야. 필요한 것들을 하나씩 설치해줘"

시스템 AI가 자동으로 필요한 패키지들을 분석하고 설치합니다.

⚠️ **참고**: 처음 설치 시 많은 패키지가 설치될 수 있어 시간이 걸릴 수 있습니다. 인내심을 가지고 기다려 주세요.

**필수 핵심 패키지 (수동 설치 시):**
```bash
pip install fastapi uvicorn pydantic python-dotenv pyyaml aiofiles anthropic openai google-genai pynostr websocket-client
```

**자주 사용하는 패키지:**
```bash
pip install beautifulsoup4 requests feedparser pillow matplotlib plotly kaleido numpy yt-dlp duckduckgo-search
```

### 2.5 데이터 저장 위치

빌드된 앱의 데이터 저장 위치:
- **macOS**: `~/Library/Application Support/IndieBiz/`
- **Windows**: `%APPDATA%/IndieBiz/`

앱 업데이트 시에도 데이터가 유지됩니다.

## 3. 핵심 개념

### 3.1 프로젝트 (Project)

프로젝트는 IndieBiz OS의 기본 작업 단위입니다. 마치 운영체제의 폴더처럼, 각 프로젝트는 독립된 AI 작업 공간입니다.

**프로젝트의 특징:**
- 무제한 생성 가능
- 각 프로젝트는 독립된 대화 기록, 에이전트, 컨텍스트 보유
- 프로젝트 복사하여 변형 가능
- 삭제 시 다른 프로젝트에 영향 없음

**프로젝트 구조 예시:**
```
IndieBiz OS
├── 의료 프로젝트
│   ├── 김 내과의사 (공감하는 내과 전문의)
│   ├── 박 정형외과의사 (꼼꼼한 정형외과 전문의)
│   └── 이 약사 (약물 상호작용 전문가)
│
├── 부동산 프로젝트
│   ├── 세무 자문가 (보수적, 꼼꼼함)
│   └── 법률 자문가 (사실 중심)
│
└── 스타트업 프로젝트
    ├── 마케팅 에이전트 (창의적)
    └── 개발자 에이전트 (실용적)
```

### 3.2 에이전트 (Agent)

에이전트는 프로젝트 내에서 특정 역할을 수행하는 AI입니다. 단순히 "역할을 수행"하는 것이 아니라, **누구인지**, **어떻게 말하는지**까지 정의합니다.

**에이전트 정의 예시 (agents.yaml):**
```yaml
agents:
  dr_kim:
    role: |
      당신은 20년 경력의 내과 전문의 김 선생입니다.
      항상 환자의 걱정을 먼저 인정한 후 질문합니다.
      의학 용어는 일상 언어로 설명합니다.
      상담을 마칠 때 항상 명확한 다음 단계와 안심의 말을 전합니다.
    model: claude-sonnet
    allowed_tools: [health-record, web-search]
```

**일반 AI vs IndieBiz 페르소나:**

| 일반 AI | IndieBiz 페르소나 |
|---------|-------------------|
| "고혈압일 수 있습니다. 의사와 상담하세요." | "걱정되시는 마음 이해합니다. 몇 가지 여쭤볼게요... 병력을 보니 이렇게 권장드립니다..." |
| 정보 전달 | 맥락을 가진 공감적 대화 |

### 3.4 스위치 (Switch)

스위치는 반복되는 AI 대화를 저장하고 원클릭으로 실행하는 자동화 기능입니다.

**스위치 이전 (수동):**
```
1. AI 채팅 열기
2. "오늘 테크 뉴스 찾아줘" 입력
3. 결과 대기
4. 복사 붙여넣기
5. "이거 요약해줘" 입력
6. 파일로 수동 저장
7. 내일 다시 반복...
```

**스위치 이후 (자동):**
```
[토글] 데일리 테크 뉴스 → 클릭 → 완료!
         ↓
    AI가 미리 정의된 프롬프트 실행
         ↓
    결과가 outputs/daily_news.md에 자동 저장
```

**스위치 기능:**
- **자연어 정의**: 평소처럼 프롬프트 작성
- **예약 실행**: 매일 8시, 매주 금요일 등 자동 실행
- **에이전트 지정**: 어떤 에이전트가 처리할지 지정
- **출력 저장**: 결과가 자동으로 파일에 저장

**스위치 예시:**

| 스위치 | 프롬프트 | 스케줄 |
|--------|----------|--------|
| 데일리 뉴스 | "AI/블록체인 뉴스 수집해서 5개 요점으로 요약해줘" | 매일 8시 |
| 주간 리포트 | "이번 주 작업 로그 분석해서 생산성 보고서 생성해줘" | 금요일 6시 |
| 시장 관찰 | "관심 종목 체크하고 이상 신호 알려줘" | 매일 장 마감 |

### 3.5 위임 체인 (Delegation Chain)

에이전트들이 서로 협업할 수 있는 시스템입니다.

**기본 흐름:**
```
사용자 → 에이전트A → call_agent(B) → 에이전트B → 작업 완료 → 자동 보고 → 에이전트A → 사용자
```

**위임 유형:**

1. **단일 위임**: A → B → 결과 → A → 사용자
2. **순차 위임**: A → B → 결과 → A → C → 결과 → A → 사용자
3. **병렬 위임**: A → (B, C, D 동시) → 모든 결과 → A → 사용자
4. **체인 위임**: A → B → C → D (재귀적)

**중요 원칙:**
- 위임받는 에이전트는 **스테이트리스** (이전 작업을 기억하지 못함)
- 위임 메시지에 **모든 필요한 정보**를 포함해야 함
- 파일 경로는 **절대경로**로 전달

### 3.6 시스템 AI

시스템 AI는 모든 프로젝트 위에 있는 메타 컨트롤러입니다.

**시스템 AI의 역할:**
- 시스템 전체 설정 관리
- 시스템 문서를 장기 기억으로 참조
- 프로젝트 에이전트에게 작업 위임 가능

**시스템 AI 도구:**
- `list_project_agents`: 모든 프로젝트와 에이전트 목록 조회
- `call_project_agent`: 특정 에이전트에게 작업 위임

**위임 예시:**
```
시스템AI → call_project_agent("의료", "내과", "두통 진단")
    ↓
프로젝트 "의료" 전체 활성화 (내과, 비뇨기과, 심장전문, ...)
    ↓
"내과"가 작업 수행
    ↓ (필요시)
내과 → call_agent("심장전문", "심장 관련 확인")
```

## 4. AI 프로바이더 설정

### 4.1 지원 프로바이더

IndieBiz OS는 다양한 AI 프로바이더를 지원합니다:

| 프로바이더 | 설명 | API 키 발급처 |
|------------|------|---------------|
| Anthropic | Claude 모델 | https://console.anthropic.com |
| OpenAI | GPT 모델 | https://platform.openai.com |
| Google AI | Gemini 모델 | https://aistudio.google.com |
| Ollama | 로컬 LLM | 로컬 설치 (API 키 불필요) |

### 4.2 API 키 설정 방법

1. 시스템 AI 또는 에이전트 설정 화면으로 이동
2. 프로바이더 선택
3. API 키 입력
4. 모델 이름 입력

### 4.3 모델 이름

각 프로바이더별 사용 가능한 모델:

**Anthropic (Claude):**
- `claude-sonnet-4-20250514` (권장, 빠르고 경제적)
- `claude-opus-4-20250514` (더 강력하지만 비쌈)

**OpenAI:**
- `gpt-4o` (권장)
- `gpt-4-turbo`

**Google AI (Gemini) - 2026.1 기준:**
- `gemini-2.5-flash-preview` (권장, 빠르고 경제적)
- `gemini-2.5-pro-preview` (더 강력하지만 느리고 비쌈)

⚠️ **참고**: Pro 모델은 Flash보다 비용이 높고 응답 속도가 느립니다. 일반적인 작업에는 Flash 모델을 권장합니다.

### 4.4 Ollama (로컬 LLM)

인터넷 없이 로컬에서 LLM을 실행할 수 있습니다.

```bash
# Ollama 설치 후
ollama pull llama2
ollama run llama2
```

IndieBiz에서 프로바이더를 "Ollama"로 선택하면 됩니다.

## 5. 주요 기능 상세

### 5.1 다중채팅방 (Multi-Chat Rooms)

여러 에이전트와 사용자가 함께하는 그룹 채팅방입니다.

**특징:**
- 각 채팅방이 별도 창으로 열림
- 어떤 프로젝트의 에이전트든 불러올 수 있음
- `@이름` 문법으로 특정 에이전트 지목
- 각 에이전트에게 특정 도구 할당 가능

**활용 예시:**
- 이순신 장군, 원균, 본인이 함께하는 3자 대화
- 의료팀과 법률팀 에이전트가 함께하는 복합 상담

### 5.2 통신 채널

IndieBiz OS는 GUI 외에도 외부 채널을 통해 소통합니다.

**지원 채널:**
- **Gmail**: OAuth2 인증, 폴링 방식 수신
- **Nostr**: 탈중앙화 프로토콜, 실시간 WebSocket

**원격 명령:**
외출 중에도 Gmail이나 Nostr 앱을 통해 PC의 IndieBiz OS에 명령을 보낼 수 있습니다.

예: Nostr 메시지로 "신문 발행해서 이메일로 첨부해줘" → 시스템 AI가 작업 실행

### 5.3 자동응답 시스템 V3

외부인의 메시지에 AI가 자동으로 응답합니다.

**처리 흐름:**
```
새 메시지 도착
    ↓
컨텍스트 수집 (이웃 정보, 근무지침, 비즈니스 문서, 대화 기록)
    ↓
AI 호출 (Tool Use) - 1회
    ↓
AI가 도구 선택:
├─ search_business_items → 비즈니스 DB 검색
├─ no_response_needed → 스팸/광고 무시
└─ send_response → 응답 즉시 발송
```

**활용:**
- 고객 문의 자동 응대
- 예약 안내
- 비즈니스 정보 제공

### 5.7 원격 접근 (Finder & 런처)

Cloudflare Tunnel을 통해 외부에서 안전하게 홈 서버에 접근하는 두 가지 기능입니다. 포트 포워딩이나 DDNS 설정 없이 사용할 수 있습니다.

#### 5.7.1 원격 Finder (개인 NAS)

PC의 파일에 접근하고 동영상을 스트리밍합니다.

**핵심 기능:**
- **외부 파일 접근**: 스마트폰이나 다른 컴퓨터에서 홈 PC의 파일 탐색
- **동영상 스트리밍**: 여행 중에도 집에 있는 동영상을 스트리밍 재생 (Range 요청 지원으로 구간 탐색 가능)
- **텍스트 파일 미리보기**: 문서 내용 빠르게 확인
- **파일 다운로드**: 필요한 파일을 어디서든 다운로드

**설정 방법:**
1. 설정 → 원격 Finder 탭
2. "NAS 활성화" 토글 켜기
3. 비밀번호 설정
4. 접근 허용할 경로 추가 (예: `/Users/me/Videos`)

**접속:** `https://your-tunnel.com/nas/app`

#### 5.7.2 원격 런처 (AI 제어)

외부에서 시스템 AI와 프로젝트 에이전트를 제어합니다.

**핵심 기능:**
- **시스템 AI 채팅**: 어디서든 시스템 AI와 대화
- **프로젝트 에이전트 채팅**: 개별 프로젝트의 AI 에이전트 제어
- **스위치 원탭 실행**: 자동화된 작업을 한 번의 터치로 실행
- **모바일 최적화 UI**: 다크 테마, 반응형 디자인

**설정 방법:**
1. 설정 → 원격 런처 탭
2. "원격 런처 활성화" 토글 켜기
3. 비밀번호 설정 (원격 Finder와 별도 가능)

**접속:** `https://your-tunnel.com/launcher/app`

#### 공통: Cloudflare Tunnel 설정

**아키텍처:**
```
스마트폰/노트북
     │
     ▼ (HTTPS)
Cloudflare Edge (전세계 분산)
     │
     ▼ (암호화된 터널)
cloudflared (홈 PC에서 실행)
     │
     ▼ (localhost:8765)
IndieBiz OS 백엔드
     ├─ /nas/app (파일 접근)
     └─ /launcher/app (AI 제어)
```

**터널 설정 방법:**

1. **cloudflared 설치**
   ```bash
   # macOS
   brew install cloudflared
   ```

2. **Cloudflare 로그인**
   ```bash
   cloudflared tunnel login
   # 브라우저에서 도메인 인증
   ```

3. **터널 생성** (시스템 AI에게 요청하거나 수동)
   ```
   시스템 AI에게: "원격 접근용 터널을 설정해줘"
   또는 수동: cf_tunnel(action='setup_nas', hostname='home.mydomain.com')
   ```

4. **터널 실행**
   ```bash
   cloudflared tunnel run indiebiz
   ```

**보안 특징:**
- **Cloudflare Tunnel**: 인바운드 포트를 열지 않아 외부 공격에 안전
- **자동 SSL**: Cloudflare가 HTTPS 인증서 자동 관리
- **독립 인증**: 원격 Finder와 원격 런처에 각각 다른 비밀번호 설정 가능
- **경로 보호**: 원격 Finder는 명시적으로 허용한 폴더만 접근 가능

**활용 시나리오:**
- 여행 중 호텔에서 집 PC의 영화 감상 (원격 Finder)
- 외근 중 스마트폰으로 시스템 AI에게 작업 지시 (원격 런처)
- 출퇴근길 스위치 원탭으로 일일 리포트 생성 (원격 런처)
- 가족과 사진/동영상 공유 (원격 Finder)
- 미니 PC를 개인 AI 서버 + 미디어 서버로 활용

### 5.8 에이전트 퍼블리싱 & 웹앱 생성

자신이 만든 에이전트, 도구 패키지, 웹앱을 커뮤니티와 공유하고 다른 사용자의 작품을 설치할 수 있습니다.

**에이전트 퍼블리싱:**

IndieBiz의 가장 큰 특징 중 하나는 AI 에이전트의 인격(페르소나)을 직접 설계한다는 것입니다. 잘 만들어진 에이전트 설정을 다른 사용자와 공유할 수 있습니다.

**공유 가능한 항목:**
- **역할 정의 (Role)**: 에이전트의 성격, 말투, 전문성
- **도구 설정**: 에이전트가 사용할 수 있는 도구 목록
- **프롬프트 프래그먼트**: 특정 상황에서 주입되는 추가 지시사항
- **컨텍스트 템플릿**: 동적으로 주입되는 맥락 정보 구조

**공유 방식:**
```
1. 프로젝트 설정에서 "에이전트 내보내기"
2. AI가 에이전트 설정을 분석하여 설명서 생성
3. Nostr 네트워크에 #indiebizOS-agent 해시태그로 게시
4. 다른 사용자가 검색하여 시스템 AI에게 설치 요청
```

**커스텀 웹앱 생성:**

IndieBiz OS는 단순한 채팅 인터페이스를 넘어 **완전한 웹 애플리케이션**을 만들고 서비스할 수 있습니다.

**웹앱의 특징:**
- **인라인 서비스**: API 엔드포인트가 완전한 HTML/CSS/JS 앱을 직접 제공
- **별도 배포 불필요**: IndieBiz 백엔드와 함께 실행되므로 추가 서버 필요 없음
- **Cloudflare 연동**: Tunnel로 전세계에 노출하거나, Pages/Workers에 배포 가능
- **AI 생성**: 에이전트에게 요청하면 대시보드, 도구, 게임 등을 생성

**웹앱 예시:**

| 웹앱 | 설명 | 생성 방법 |
|------|------|----------|
| **사진 갤러리** | GPS 기반 지도 뷰 + 타임라인 | "사진 폴더를 지도에 표시하는 웹앱 만들어줘" |
| **개인 신문 뷰어** | 스위치로 생성된 뉴스를 신문 형태로 표시 | "신문 HTML을 보기 좋게 표시하는 페이지 만들어줘" |
| **IoT 제어판** | ESP32 기기 제어 대시보드 | "LED 스위치를 제어하는 웹 인터페이스 만들어줘" |
| **건강 대시보드** | 혈압, 체중 등 건강 기록 시각화 | "내 건강 기록을 그래프로 보여주는 대시보드 만들어줘" |
| **포트폴리오 트래커** | 주식/암호화폐 현황 대시보드 | "내 투자 현황을 실시간으로 보여주는 페이지 만들어줘" |

**웹앱 생성 흐름:**
```
1. 에이전트에게 원하는 웹앱 설명
      ↓
2. AI가 HTML/CSS/JS 코드 생성
      ↓
3. 백엔드 엔드포인트 또는 정적 파일로 저장
      ↓
4. Cloudflare Tunnel로 외부 접근 가능하게 설정 (선택)
      ↓
5. 브라우저에서 접속하여 사용
```

**웹앱 배포 옵션:**

| 방식 | 장점 | 사용 시나리오 |
|------|------|--------------|
| **로컬 전용** | 설정 간단, 보안 | 개인 사용 |
| **Tunnel 노출** | 어디서든 접근, 동적 데이터 | 외부에서 홈 서버 접근 |
| **Cloudflare Pages** | 빠른 정적 호스팅 | 공개 웹사이트 |
| **Cloudflare Workers** | 서버리스 동적 기능 | API 엔드포인트 |

**실전 예시: 사진 갤러리 웹앱**

```
사용자: "내 사진 폴더(/Users/me/Photos)를 지도에 표시하는 웹앱을 만들어줘.
        날짜별로 필터링할 수 있었으면 좋겠어."

에이전트 작업:
1. photo-manager 도구로 사진 메타데이터 스캔
2. GPS 좌표, 촬영 날짜 추출
3. Leaflet.js 기반 지도 + 날짜 필터 UI 생성
4. /outputs/photo_gallery.html로 저장
5. 원격 Finder로 외부 접근 가능하게 설정

결과:
https://nas.mydomain.com/nas/file?path=/outputs/photo_gallery.html
→ 어디서든 내 사진 지도 보기
```

**패키지 공유 생태계:**

```
┌─────────────────────────────────────────────────────────────────┐
│                     IndieBiz 커뮤니티                            │
│                                                                  │
│  사용자A ────(공개)───► Nostr 네트워크 ◄───(검색)──── 사용자B    │
│     │                        │                          │       │
│  에이전트 설정              해시태그                  시스템AI    │
│  도구 패키지            #indiebizOS-*               자동 설치    │
│  웹앱 템플릿                                                    │
└─────────────────────────────────────────────────────────────────┘
```

**공유 철학:**

프로그램 자체를 다운로드하는 것이 아니라, **설명과 구조 정보**를 공유합니다. 받는 쪽의 시스템 AI가 해당 정보를 바탕으로 자신의 환경에 맞게 재구성합니다. 이것이 "AI가 AI를 가르치는" IndieBiz만의 방식입니다.

### 6.10 💬 다중채팅방

### 6.12 🏠 부동산 분석

아파트, 다가구 실거래가, 월세/전세 조회 + 지도 시각화

### 6.13 📈 투자 상담

주식 포트폴리오 기반 투자 상담

주가, 금, 암호화폐 가격 그래프 시각화

### 6.14 📱 원격 명령 & 자동응답

외출 중 Gmail이나 Nostr 앱으로 PC의 IndieBiz OS에 명령

예시: Nostr 메시지 "신문 발행해서 이메일로 첨부해줘" → 시스템 AI가 작업 실행

내장 자동응답: 소유자가 아닌 사람이 메시지를 보내면 AI가 적절히 응답 (단, 명령은 실행 안 함)

→ 고객 문의 처리, 예약 안내 등 자동화된 비즈니스 운영의 기반

### 6.15 🔄 도구 패키지 공유

Nostr 공개 메시지로 도구 패키지 설치 정보 공유

다른 사용자가 정보를 받아 시스템 AI에게 설치 요청

프로그램 자체가 아닌 공유된 정보(설명, 구조, 의존성 등)만으로 시스템 AI가 빌드 및 설치

### 6.16 ⚡ 빠른 연락

등록된 연락처에 빠른 연락 버튼으로 즉시 메시지 전송

## 7. 문제 해결

### 7.1 "모듈을 찾을 수 없습니다" 에러

특정 도구 패키지에 필요한 라이브러리가 설치되지 않았습니다.

**해결책:**
시스템 AI에게 물어보세요:
> "이 도구를 사용하려면 무엇이 필요해?"

또는 `INSTALL_DEPENDENCIES.md` 파일에서 해당 도구의 의존성을 찾아 설치하세요.

### 7.2 Windows에서 pip 명령이 안 될 때

```bash
python -m pip install <패키지명>
```

### 7.3 권한 에러 (Permission denied)

```bash
pip install --user <패키지명>
```

### 7.4 Playwright 브라우저 설치 실패

```bash
# 관리자 권한으로 실행
playwright install --with-deps chromium
```

### 7.5 API 키 관련 에러

1. API 키가 올바르게 입력되었는지 확인
2. API 키에 충분한 크레딧이 있는지 확인
3. 모델 이름이 정확한지 확인

### 7.6 에이전트가 응답하지 않음

1. 백엔드 서버가 실행 중인지 확인 (`python api.py`)
2. 시스템 AI 설정에서 프로바이더와 API 키 확인
3. 콘솔 로그 확인 (오류 메시지)

### 7.7 도구가 작동하지 않음

1. 해당 도구 패키지가 설치되어 있는지 확인
2. 에이전트의 `allowed_tools`에 도구가 포함되어 있는지 확인
3. 도구에 필요한 외부 서비스 (API 키 등)가 설정되어 있는지 확인
