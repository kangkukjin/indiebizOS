# 책 조판 가이드 (typst 기반 한국어 단행본)

원고 단계가 끝나고 **PDF 책으로 조판**할 때 이 가이드를 따른다. 원고 관리(목차/플랜/챕터 작성)는 `book_publishing.md` 참조 — 이 가이드는 그 다음 단계인 typst 조판/표지/검증/인쇄용 출력에 집중한다.

<core_rules priority="critical">
조판 작업에서 가장 자주 빠지는 4가지. 위반 시 사용자 앞에서 결함이 드러나거나 인쇄소에 잘못된 파일이 나간다:

1. **출고 전 PNG 검증 1~2바퀴 의무** — PDF를 사용자에게 보내기 전, 핵심 페이지(표지/차례/프롤로그/장 시작/막간/에필로그/찾아보기)를 PNG로 변환해 자체 검토한다. 미세조정 모드(§6 A)에서는 적용 안 됨. 출고 모드(§6 B)에서는 §10 체크리스트를 한 줄씩 직접 확인.
2. **표지 두 파일 동기화** — `cover.typ`(디지털용)과 `cover_print.typ`(인쇄용)은 항상 같이 수정한다. `cover.typ`만 고치고 인쇄용을 잊으면 인쇄소 파일이 옛 표지로 나간다.
3. **사용자 명시 컴포넌트 보존** — "이 책을 읽는 법" 같이 사용자가 명시적으로 요청한 컴포넌트는 자의적으로 삭제·재구성·이동 금지. 표준 결정사항과 충돌하더라도 사용자 명시가 우선.
4. **표준 결정사항 적용, 차이만 확인** — 미래상상(출판사), 강국진(저자), 신국판, Pretendard, 책등 정자. 사용자가 명시적으로 바꾸자고 한 항목만 변경, 매번 묻지 말 것 (§2).
</core_rules>

## 1. 작업 환경

```
{typst_dir}/                # 예: ~/Desktop/hj/typst/
├── main.typ                # 조립 엔트리 — typst compile 대상
├── content.typ             # 본문 (convert.py가 자동 생성)
├── manuscript.md           # 원고 (markdown — 본문 수정은 여기에서)
├── convert.py              # manuscript.md → content.typ 변환기
├── cover.typ               # 디지털용 표지 (단일 페이지)
├── cover_print.typ         # 인쇄용 표지 (앞+뒤+책등+날개 펼침)
├── config/
│   ├── theme.typ           # 폰트/크기/마진/색상 변수
│   ├── layout.typ          # footer/header/페이지 번호
│   └── index-markers.typ   # 색인 마커
├── components/             # 각 페이지 타입별 컴포넌트
│   ├── title-page.typ      # 속표지
│   ├── colophon.typ        # 판권지
│   ├── toc.typ             # 차례
│   ├── howto.typ           # 이 책을 읽는 법
│   ├── front-piece.typ     # 프롤로그/에필로그
│   ├── part.typ            # 부 표지
│   ├── chapter.typ         # 장 표지
│   ├── interlude.typ       # 막간
│   ├── blocks.typ          # 단락/이미지/소절
│   ├── factbox.typ         # 팩트박스
│   ├── author-bio.typ      # 저자 소개
│   └── index-page.typ      # 찾아보기
└── media/                  # 이미지 (또는 images/)
```

**원고 → 조판 파이프라인:**
```
manuscript.md → convert.py → content.typ → main.typ → typst compile → output.pdf
```

본문 미세 수정은 `manuscript.md`에서, 디자인 미세 수정은 `config/theme.typ` 또는 `components/*.typ`에서. 디자인을 만지기 전에 어느 파일이 그 요소를 담당하는지 본 가이드의 §5 표를 먼저 확인한다.

## 2. 표준 결정사항 (이 사용자의 책)

새 책 프로젝트를 시작할 때 이 결정사항을 출발점으로 삼고, 사용자에게 **차이만** 확인한다. 기본값이 그대로면 묻지 않는다.

| 항목 | 기본값 | 비고 |
|------|--------|------|
| 출판사 | **미래상상** | "미래상사" 오타 주의. 표지에 "출판사"라는 단어는 안 넣음 — 로고/이름만 |
| 저자명 | 강국진 | |
| 판형 | 신국판 (152×225mm) | 다른 판형은 사용자에게 확인 |
| 본문 폰트 | Pretendard | 한국 출판/디자인 표준, 9 weight, OFL |
| Fallback | Noto Sans KR | 시스템 기본 |
| 책등 글자 | **정자로 세움** (눕히지 말 것) | 사용자 명시 선호 |
| 출판사 로고 (책등) | 굵은고딕 "미래상상" 2자씩 쌓아 4글자가 하나의 박스 | |
| 섹션 구분 | 빈 줄만 | 다른 분리자(`* * *`, `※`, `❦` 등) 쓸 거면 사용자에게 확인 |
| 폰트 계열 | 산세리프 (Pretendard) | 명조 계열 원하면 사용자에게 확인 |

영문 본문 섞임이 많아도 Pretendard만으로 충분.

</standards>

## 3. 책 구성 (front matter → body → back matter)

표준 순서. 각 항목은 typst 컴포넌트로 분리되어 있다.

1. **표지 (cover)** — 앞표지, 뒤표지, 책등, 선택적으로 날개
2. **속표지 (title-page)**
3. **판권지 (colophon)** — ISBN, 출판사, 인쇄일. 최소판은 생략 가능하지만 정식 출판 시 필수
4. **차례 (toc)**
5. **이 책을 읽는 법 (howto)** — 사용자가 명시적으로 요청한 컴포넌트, 자의적 추가/삭제 금지
6. **프롤로그 (front-piece)**
7. **부 표지 (part) → 장 표지 (chapter) → 본문 → 막간 (interlude)** 반복
8. **에필로그 (front-piece)**
9. **저자 소개 (author-bio)**
10. **찾아보기 (index-page)**

---

## 4. 표지 디자인 결정사항

- **앞표지**: 제목 + 부제 + 저자. 메인+서브 구조(예: "AI 시대의 새로운 몸" / "나만의 인공지능을 설계하다")가 흔히 쓰임.
- **뒤표지**: 광고 카피 + 저자 약력 + 출판사 로고.
- **책등**: 제목 + 저자 + 출판사. **정자로 세움** (눕히지 말 것).
- **날개 (flap)**: 책날개 있는 판형이면 좌/우 날개에 추가 정보. 인쇄 시 `cover_print.typ` 따로 빌드.
- **PDF 메타데이터**: 제목/저자/출판사를 PDF metadata에도 기입. 일반 독자는 안 보지만 **서점/도서관 시스템에서 필수**.
- **컬러스페이스**: 인쇄용은 CMYK, 디지털은 RGB. 표지 색상이 살아 보이도록 확인.

## 6. 작업 흐름 — 미세조정 vs 출고 검증

조판 작업에는 두 가지 모드가 있다. **둘을 섞으면 사용자가 불편해진다.** 작업 진입 시 모드를 명확히 한다.

### 모드 판단 기준
| 신호 | 모드 |
|------|------|
| 사용자가 typst watch를 켜놓고 있다고 명시 | **A 미세조정** |
| "표지 다듬어줘", "여백 좀 조정해줘" 같은 진행 중 수정 요청 | **A 미세조정** |
| "PDF 보내줘", "이거 인쇄 보낼게", "최종본 뽑아줘", "공유해줘" | **B 출고** |
| 새 작업 첫 진입이고 모드 불명확 | 사용자에게 확인 후 진행 |

### A. 미세조정 모드 (사용자가 `typst watch`로 직접 봄)
- 사용자가 watch 모드를 켜놓고 있다고 가정한다.
- 에이전트는 디자인을 수정만 하고, **자발적으로 PNG로 변환해서 검토 사이클을 돌리지 않는다**.
- 사용자가 "어때?" 같이 명시적으로 물어볼 때만 결과를 확인한다.
- 빠른 반복이 핵심 — 군더더기 검증 없이 수정 → 사용자가 watch에서 즉시 확인.

### B. 출고 모드 (사용자에게 PDF를 보내기 직전)
- PDF를 사용자에게 전달하기 **전에** 핵심 페이지를 PNG로 변환해서 자체 검토 **1~2바퀴**.
- 검토 대상: 표지 / 차례 / 프롤로그 / 장 시작 / 막간 / 에필로그 / 찾아보기.
- §7의 결함 체크리스트를 한 줄씩 직접 확인한다.
- 발견된 결함은 수정하고 다시 한 번 확인한 뒤 사용자에게 전달.
- 출고 전 §10 체크리스트(`<pre_send_checklist>`)를 통과해야 한다.

**PNG 변환 표준 명령:**
```bash
pdftoppm -jpeg -r 150 output.pdf page    # JPEG
pdftoppm -png -r 150 output.pdf page     # PNG
```

## 7. 자주 발견되는 결함 체크리스트 (출고 모드에서 한 줄씩 확인)

PNG 검토 시 다음 항목을 머릿속으로만 통과시키지 말고 **직접 눈으로 확인**한다:

- [ ] 본문이 짧게 끝나 다음 페이지가 비어 보이지 않는가 → leading 미세 조정 (예: 1em → 1.2em)
- [ ] 홀수쪽 시작 룰 위반 → 빈 페이지 삽입 필요 여부 확인
- [ ] 페이지 번호가 속표지/판권지/장 표지에 노출되지 않았는가 → `footer-fn`에서 숨김 처리
- [ ] 한글-영문 혼합 시 baseline 어긋남 → 폰트 패밀리 fallback 순서
- [ ] 책등 글자가 눕혀지지 않고 정자로 서 있는가
- [ ] 표지 색상이 칙칙하지 않은가 → 디지털(RGB) vs 인쇄용(CMYK) 컬러스페이스
- [ ] PDF 메타데이터(제목/저자/출판사)가 채워져 있는가
- [ ] `cover.typ` 변경이 `cover_print.typ`에도 반영되었는가
- [ ] 출판사 이름 오타("미래상사") 없는가 — **정확한 명칭은 미래상상**
- [ ] 사용자 명시 컴포넌트가 그대로 있는가 ("이 책을 읽는 법" 등)

</defect_checklist>

## 8. 디지털 PDF vs 인쇄용 PDF

| 구분 | 파일 | 내용 |
|------|------|------|
| 디지털 PDF | `output.pdf` | 단일 페이지 흐름, 메타데이터 포함, 표지 1쪽 |
| 인쇄용 표지 PDF | `cover_print.pdf` 등 | 앞표지+뒤표지+책등+날개를 한 면에 펼쳐 배치. 인쇄소 사양에 맞춰 별도 출력 |

**주의**: 표지를 수정할 때 `cover.typ`만 고치면 인쇄용에 반영되지 않는다. **두 파일 모두 일관성을 확인**한다 (core_rules 2번).

## 9. 회피해야 할 실수

- ❌ 표지에서 한 요소(예: 출판사 이름)를 지우고 다른 요소 위치를 안 옮기면 **디자인 비율이 깨짐**. 항상 "이 변경이 전체 비율을 어떻게 바꾸는지" 확인.
- ❌ **책등 글자를 눕히는 것** — 사용자가 명시적으로 정자 세움 선호.
- ❌ 표지 색상이 죽어 보이게 함 — **컬러스페이스 확인** 누락 (인쇄용은 CMYK, 디지털은 RGB).
- ❌ **미세조정 모드에서 자발적 PNG 사이클** — 사용자가 watch로 보고 있는데 에이전트가 매번 PNG 뽑으면 산만함. §6 A에서는 사용자가 명시적으로 물을 때만.
- ❌ 사용자가 직접 정한 컴포넌트(예: "이 책을 읽는 법")는 **임의로 삭제·재구성 금지** (core_rules 3번).
- ❌ `cover.typ`만 고치고 `cover_print.typ`를 잊으면 **인쇄소 파일이 옛 표지가 됨** (core_rules 2번).
- ❌ 출판사 이름 **"미래상사" 오타** — 정확한 명칭은 **미래상상**.
- ❌ **출고 모드에서 PNG 검증을 건너뛰고 PDF 곧장 전달** (core_rules 1번). §10 체크리스트 통과 필수.

</antipatterns>

## 10. PDF를 사용자에게 보내기 직전 체크리스트 (출고 모드 의무)

미세조정 모드(§6 A)에서는 적용 안 됨. **출고 모드(§6 B)에서는 모든 항목을 직접 확인한 후에만 사용자에게 PDF를 전달한다:**

1. **PNG 1차 변환 완료** — `pdftoppm`로 핵심 페이지(표지/차례/프롤로그/장 시작/막간/에필로그/찾아보기)가 다 뽑혔는가?
2. **§7 결함 체크리스트 한 줄씩 확인** — 머릿속으로 통과시키지 말고 직접 봤는가?
3. **결함 발견 시 수정 후 2차 변환** — 1차에서 발견된 항목을 고치고 다시 PNG를 뽑아 재검토했는가?
4. **표지 두 파일 일관성** — `cover.typ` 변경분이 `cover_print.typ`에도 반영되었는가?
5. **PDF 메타데이터** — 제목·저자·출판사가 채워졌는가? (서점/도서관 시스템에 필수)
6. **사용자 명시 컴포넌트 보존** — "이 책을 읽는 법" 같이 사용자가 정한 항목이 그대로 살아있는가?
7. **출판사 이름 오타 검사** — "미래상상" 맞는가? "미래상사" 아닌가?

이 7개를 모두 통과한 후에만 사용자에게 PDF를 전달한다. 통과하지 못한 항목이 있으면 **PDF를 전달하지 말고** 수정 후 재검증한다.

</pre_send_checklist>

## 11. 새 책 시작할 때의 출발점

새 책 프로젝트를 시작하면:
1. `book_publishing.md`에 따라 원고 관리 구조를 잡고 manuscript를 작성한다.
2. typst 프로젝트 디렉토리를 만든다 (기존 책 디렉토리 복사 후 정리 권장).
3. §2의 표준 결정사항을 그대로 적용한다. 사용자가 명시적으로 바꾸자고 한 항목만 변경.
4. `manuscript.md` → `convert.py` → `content.typ` → `main.typ` 파이프라인을 한 번 돌려서 빈 책 PDF가 나오는지 확인.
5. 표지 디자인을 잡는다 (`cover.typ` 먼저, 합의 후 `cover_print.typ`).
