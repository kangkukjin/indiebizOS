# 심층연구 가이드

## 개요
이 가이드는 단순 정보 전달이 아닌, **전문가 수준의 분석과 보고서**를 요구하는 작업에 적용된다.
"심층연구", "깊이 있게 분석", "전문가 수준으로", "보고서 작성" 등의 요청에 이 가이드를 따른다.

> **핵심 원칙**: 사용자가 심층연구를 요청하면, 코딩 태스크의 "최소한으로 해결" 원칙을 적용하지 않는다. 대신, 충분한 깊이와 다각적 관점을 갖춘 결과물을 만든다.

---

## Phase 1: 리서치 (정보 수집)

### 1.1 주제 범위 설정
- 사용자의 요청에서 **명시적 요구**와 **암묵적 기대**를 모두 파악한다
- 예: "이란 전쟁 분석해줘" → 명시적: 전쟁 상황 분석 / 암묵적: 배경, 국제 관계, 향후 전망까지 기대
- 범위가 불확실하면 `ask_user_question`으로 확인하되, 과도하게 질문하지 않는다

### 1.2 다층적 정보 수집
최소 **3개 이상의 독립적 관점**에서 정보를 수집한다:

| 관점 층위 | 예시 |
|-----------|------|
| 사실/현황 | 무슨 일이 일어나고 있는가? 데이터, 수치, 타임라인 |
| 원인/배경 | 왜 이런 상황이 발생했는가? 역사적 맥락, 구조적 원인 |
| 이해관계자 | 누가 관련되어 있으며 각자의 입장은? |
| 영향/파급 | 어떤 결과가 예상되는가? 단기/중기/장기 |
| 반론/대안 | 주류 해석에 대한 반론은? 다른 시나리오는? |

### 1.3 출처 품질 관리
- 웹 검색 시 **최소 3개 이상의 서로 다른 출처**를 참조한다
- 공식 기관, 전문 매체, 학술 자료를 우선한다
- 단일 출처에 의존하지 않는다
- 출처 간 상충하는 정보가 있으면 명시적으로 밝힌다

## Phase 2: 분석 (사고 프로세스)

### 2.1 구조화된 분석 프레임워크
주제에 맞는 분석 프레임워크를 선택한다:

- **국제 정세/지정학**: 이해관계자 분석 + 시나리오 분석
- **경제/시장**: PEST 분석 + 데이터 기반 추세 분석
- **기술/산업**: 기술 성숙도 + 경쟁 구도 + 채택 동향
- **사회/문화**: 다층적 원인 분석 + 사례 비교
- **일반 주제**: 5W1H 심화 + 인과관계 분석

### 2.2 비판적 사고 적용
모든 심층연구에는 다음을 반드시 포함한다:

1. **주장과 근거의 분리**: "~라고 한다"(보도)와 "~이기 때문이다"(분석)를 구분
2. **반론 검토**: 주류 견해에 대한 반대 논거를 최소 1개 이상 제시
3. **불확실성 명시**: 확실한 사실과 추정/전망을 명확히 구분
4. **편향 인식**: 특정 관점에 치우치지 않았는지 자기 점검

### 2.3 깊이 기준
| 수준 | 특징 | 예시 |
|------|------|------|
| 표면적 | 사실 나열 | "A국이 B국을 공격했다" |
| 중간 | 원인과 결과 연결 | "A국의 공격은 C 사건에 대한 보복이며, D에 영향을 미칠 것이다" |
| **심층** | 구조적 분석 + 다관점 + 전망 | "이 충돌의 근본 원인은 X 구조에 있으며, 이해관계자별 입장은 다르다. 3가지 시나리오가 가능하다" |

**심층연구에서는 항상 '심층' 수준을 목표로 한다.**

---

## Phase 3: 결과물 품질 기준

### 3.1 보고서 구조
심층연구 결과물은 다음 구조를 기본으로 한다:

```
1. 핵심 요약 (Executive Summary)
   - 3~5문장으로 핵심 결론 먼저 제시
   - 바쁜 독자가 이것만 읽어도 핵심을 파악할 수 있어야 함

2. 배경 및 맥락
   - 이 주제가 왜 중요한지
   - 역사적/구조적 배경

3. 본론 (핵심 분석)
   - 주제에 맞는 분석 프레임워크 적용
   - 데이터/증거 기반 논증
   - 다양한 관점 포함

4. 시사점 및 전망
   - 단기/중기/장기 시나리오
   - 불확실성 요인 명시

5. 출처 및 참고자료
   - 참조한 출처 목록
```

### 3.2 전문가 수준의 글쓰기 기준
- **구체성**: 추상적 진술 대신 구체적 수치, 사례, 데이터 사용
- **논리적 흐름**: 각 문단이 이전 문단의 논의를 발전시킴
- **간결하되 충분히**: 불필요한 수식어를 피하되, 필요한 설명은 충분히
- **전문 용어**: 적절한 전문 용어를 사용하되, 필요시 설명 추가
- **시각화**: 복잡한 비교나 데이터는 표/차트로 정리

### 3.3 분량 기준
- 심층연구 결과물은 **최소 1500자 이상**을 목표로 한다
- 짧은 답변은 심층연구가 아니다. 충분한 분석 깊이를 확보한다
- 단, 불필요한 반복이나 뻔한 내용으로 분량을 채우지 않는다

## Phase 4: 자기검토 (Self-Review)

### 4.1 완성 후 반드시 다음을 점검한다:

**내용 품질 체크리스트:**
- [ ] 핵심 요약만 읽어도 결론을 알 수 있는가?
- [ ] 주장에 대한 근거가 제시되어 있는가?
- [ ] 반대 관점이나 반론이 포함되어 있는가?
- [ ] 확실한 사실과 추정/전망이 구분되어 있는가?
- [ ] 빠진 중요한 관점이 없는가?

**형식 품질 체크리스트:**
- [ ] 논리적 흐름이 자연스러운가?
- [ ] 추상적 진술 없이 구체적인가?
- [ ] 적절한 시각화(표, 차트)가 포함되었는가?
- [ ] 출처가 명시되어 있는가?

### 4.2 개선 요청 시 (사용자가 "다시 살펴봐", "개선해줘" 등)
단순 반복이 아닌, **실질적 개선**을 수행한다:

1. 위 체크리스트로 현재 결과물을 평가
2. 부족한 항목을 구체적으로 식별
3. 해당 부분을 보강:
   - 근거가 부족하면 → 추가 리서치 수행
   - 관점이 단편적이면 → 반론/대안 관점 추가
   - 구조가 불명확하면 → 재구성
   - 깊이가 부족하면 → 원인 분석/시나리오 분석 추가
4. **무엇을 어떻게 개선했는지** 간략히 설명

## 사용 예시

### 예시 1: 지정학 분석
```
사용자: 이란 전쟁에 대해 심층연구해줘

에이전트:
1. search_guide("심층연구") → 이 가이드 참조
2. web_search로 최신 정보 수집 (최소 3개 출처)
3. 이해관계자 분석 프레임워크 적용
4. 보고서 구조에 맞춰 작성
5. 자기검토 체크리스트 통과 후 제출
```

### 예시 2: 산업 분석
```
사용자: AI 반도체 시장을 깊이 있게 분석해줘

에이전트:
1. search_guide("심층연구") → 이 가이드 참조
2. 시장 데이터, 기업 동향, 기술 트렌드 수집
3. 경쟁 구도 + 기술 성숙도 프레임워크 적용
4. 데이터 기반 표/차트 포함
5. 자기검토 후 제출
```

### 예시 3: 개선 요청
```
사용자: 이거 다시 살펴보고 개선해줘

에이전트:
1. 자기검토 체크리스트로 현재 결과물 평가
2. "반론 미포함, 시나리오 분석 부재" 식별
3. 추가 리서치 → 반론 섹션 추가 → 3가지 시나리오 추가
4. "반론 관점과 시나리오 분석을 추가했습니다" 설명
```
