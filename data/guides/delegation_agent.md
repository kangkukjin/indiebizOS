# 프로젝트 에이전트 위임 가이드

프로젝트 내 동료 에이전트에게 작업을 위임하는 방법입니다.

## 기본 도구

```
# 동료 에이전트에게 위임
[others:delegate]{agent_id: "에이전트이름", message: "작업 내용"}

# 에이전트 정보 조회
[others:info]{agent_id: "에이전트이름"}
```

## 핵심 원칙

### 1. 자신이 할 수 있으면 직접 하세요
불필요한 위임은 금지. 자신의 도구로 처리 가능하면 직접 처리하세요.

### 2. 동료 확인
환경 프롬프트의 `<peers>` 섹션에서 동료 에이전트 이름과 역할을 확인하세요.
- `<peers>`에 동료가 없으면 위임 불가
- 에이전트 이름은 `<peers>`에 표시된 **그대로** 사용 (오타, 접미사 변경 금지)
- 자기 자신에게 위임 금지

### 3. 의도를 전달하고, 실행 방법은 맡겨라
자연어로 의도와 목적을 전달하세요. 위임받는 에이전트가 실행할 구체적인 IBL 코드를 대신 작성하지 마세요.

### 4. 비동기 처리
위임은 비동기로 처리됩니다. 결과는 자동으로 보고됩니다.

## 병렬 위임

서로 의존성이 없는 작업은 동시에 위임 가능:
```
[others:delegate]{agent_id: "내과", message: "두통 증상 분석"}
[others:delegate]{agent_id: "외과", message: "무릎 통증 검사 의뢰"}
```

## 순차 위임

이전 결과가 필요한 경우 **반드시 순차적으로** 위임하세요:

1. TODO로 전체 계획 작성 (두 번째 작업은 pending 상태로)
2. 첫 번째 위임만 실행
3. **실제 결과가 도착할 때까지 대기** (다음 위임 보류)
4. 결과 도착 후 TODO 업데이트하고 두 번째 위임 실행

**중요**: `[others:delegate]`가 반환하는 "위임했습니다"는 **접수 확인**일 뿐입니다.

**절대 금지:**
- 결과를 받지 않은 상태에서 "검토했습니다", "확인했습니다" 등의 표현 사용
- 위임 접수 확인만 받고 마치 결과를 받은 것처럼 행동
- 실제 결과 없이 다음 단계 작업을 위임

```
# ❌ 잘못된 예
[others:delegate]{agent_id: "내과", message: "혈압 측정해줘"}
[others:delegate]{agent_id: "내과", message: "결과 확인했습니다. 약 처방해줘"}  # 결과 없음!

# ✅ 올바른 예
[others:delegate]{agent_id: "내과", message: "혈압 측정해줘"}
# → 여기서 멈추고 실제 결과 대기 → 결과 도착 후 다음 위임
```

## 파일 경로 원칙

파일 위치를 보고할 때는 **반드시 절대경로**를 사용하세요.

```
# ❌ "결과 파일: outputs/result.txt"
# ✅ "결과 파일: /Users/.../projects/프로젝트명/outputs/result.txt"
```

## 스테이트리스 원칙

위임받는 에이전트는 **스테이트리스** 상태입니다. 이전 작업을 기억하지 못합니다.

위임 메시지에는 작업에 필요한 **모든 정보**를 포함하세요:
- 수정할 파일의 절대경로
- 구체적인 수정 내용
- 필요한 원본 데이터
- 원래 요청의 조건/제약사항

순차 위임 시 **처음 받은 조건/제약사항을 매번 포함**해야 합니다.

```
# ❌ "아까 만든 것 수정해줘"
# ✅ "파일 /Users/.../result.txt의 3번째 항목을 'ABC'로 수정해줘"
```

## 시간 지정 위임 (크로스 위임)

특정 시간에 다른 에이전트에게 작업을 시키는 경우, `[self:schedule]`에 `target_agent_id`를 지정하세요.

- 즉시 위임 → `[others:delegate]` 사용
- 시간 지정 → `[self:schedule]` + `target_agent_id` 사용

```
# ❌ 스케줄 안에 delegate (결과 전달 불가)
[self:schedule]{at: "15:00", pipeline: "[others:delegate]{agent_id: '심장전문', message: '분석해줘'}"}

# ✅ 크로스 위임
[self:schedule]{at: "15:00", target_agent_id: "심장전문", pipeline: "최신 심장 연구 동향을 조사하고 요약해줘"}
```

크로스 위임 시 pipeline에는 자연어로 의도를 전달하세요.
