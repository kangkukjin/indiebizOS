# 디스크 검색 가이드 (Disk Search Guide)

내 컴퓨터(로컬 디스크)에서 파일·자료를 찾을 때의 방법과 지도. 웹 검색과 다르다 — 디스크는 **열거 가능한 공간**이라, "어디 있나"뿐 아니라 **"정말 없나"를 측정**할 수 있다.

## 1. 먼저 냄새지도(포식 기억)를 회상하라 — 어디부터 뒤질까
- **`[self:forage]{op: recall, query: "<주제>"}`** — 과거 디스크 포식에서 배운 *폴더 정체·정리 관습·죽은 가지·주인 모델*을 회상한다. 콜드 스타트를 피하는 첫 수.
  - 예: "논문 어디" → 이전에 "연구 아카이브=`~/Desktop/total/work2`, 논문=`note/<저널명>` 계층" 같은 지도를 배웠다면 바로 그리로.
  - 무관하면 빈 결과(비용 ~0). 항상 먼저 한 번 던져볼 값어치가 있다.
- 회상이 비어 있거나 새 공간이면 아래 도구로 실제 탐색한다.

## 2. 도구 선택 — 무엇으로 찾나
| 찾는 것 | 액션 | 비고 |
|---|---|---|
| **파일명·확장자·크기·종류** | `[self:fs_query]{search_term·extension·kind·min_size_mb·path}` | **1순위.** OS 색인(맥 Spotlight) 직접 — 선스캔 불요·항상 최신 |
| **glob 패턴** (`*.md`, `**/*.tsx`) | `[self:file_find]{pattern·path}` | 이름 패턴·재귀. path로 검색 루트(절대/`~`/상대) |
| **파일 *내용*** (텍스트·코드) | `[self:grep]{pattern·path·include}` | 기본 정규식(`a\|b` OR 됨), `regex:false`면 리터럴 |
| **파일 읽기** | `[self:read]{path·format·pages·offset·limit}` | 텍스트/PDF/DOCX/XLSX 통합 |
| **디스크 용량** | `[self:storage]{op: scan→summary}` · 여유공간=`[sense:host]` | 폴더·확장자별 롤업은 scan 선행 |

- **이름/메타는 `fs_query`, 내용은 `grep`, 패턴은 `file_find`.** 헷갈리면: "무슨 파일인지"는 fs_query, "그 안에 무슨 말이 있는지"는 grep.
- 비색인 경로(외장·특수 위치)는 라이브 walk 폴백이 돈다 — 되지만 느리다.

## 3. 디스크 랜드마크 = 집중 폴더 루트 (안정) — 내용은 라이브로
- **"어디가 중요한가"의 안정 지도 = 집중 폴더 루트**다. 웹 랜드마크(URL 목록)의 디스크판 — 다만 여기선 포인터가 *폴더 루트*다. 단일 소스 = 사용자 선언 `data/focus_folders.json`(없으면 기본값: mac은 `~/Desktop`, + forager territory "자주 되돌아온 곳"). 편집하면 즉시 반영 — 중요 폴더가 늘면 여기에 선언하라.
- **그 아래 *현재 내용*은 미리 구운 트리를 기대하지 말고 `fs_query`로 신선하게 조회하라.** OS 색인이라 항상 최신·선스캔 불요. (디스크가 자꾸 바뀌므로 정적 스냅샷은 즉시 stale — 웹 페이지 본문을 매번 크롤로 가져오듯, 폴더 내용도 매번 조회하는 게 맞다.)
- 순서: 집중 폴더 루트를 검색 루트(`path`)로 좁혀 → 넓게(fs_query) → 좁게(file_find/grep).

## 4. "정말 없나"는 단정하지 말고 측정하라
- 디스크는 열거 가능하므로 음성-단언을 **측정**할 수 있다.
- **`[self:residual]{op: sample, kind·q·path, seen: [이미 본 경로]}`** → 미관측 더미의 균일 무작위 표본을 열어 보고, **`{op: estimate, relevant·sampled·unseen}`** 로 누락률+신뢰구간 추정.
- "다 찾았다 / 정말 없다"고 말하기 전에 residual로 근거를 만들어라. 목표 recall에 따른 판단은 AI 몫, 도구는 측정만.

## 5. 판단 원칙
- **"없다"고 단정하지 마라** — 못 찾은 것이지 없는 것이 아닐 수 있다(§4로 측정).
- **검색 루트를 바꿔 재시도** — 홈 전체가 안 되면 외장·특정 폴더로, 이름이 안 되면 내용(grep)으로.
- **동명·모호**하면 종류(kind)·확장자·크기·경로로 좁혀라.

## 6. 배운 것은 냄새지도에 남겨라 (다음 검색을 싸게)
- 포식하며 새로 안 *일반화 가능한 공간 지식*(폴더 정체·정리 관습·"여긴 그거 없음")은 **`[self:forage]{op: note, layer: map, ...}`** 로 누적 → 다음 탐색이 콜드 스타트를 면한다.
- 이건 대개 인지 파이프라인이 자동 증류하지만, 명시적으로 남길 값어치가 있으면 note 하라.

---
*디스크 검색은 웹과 달리 포식 에이전트에 자동 주입되지 않는다 — 포식은 웹 우선이라, 디스크가 필요할 때만 이 가이드를 `read_guide`로 읽는다.*
