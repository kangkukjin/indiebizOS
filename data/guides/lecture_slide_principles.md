# 강의 슬라이드 작성 원칙

강의·교육용 슬라이드를 만들 때의 **작성 규칙**. 교육학·인지심리학 연구가 검증한 원칙을 indiebizOS의 슬라이드 도구에 그대로 적용한다. 코드가 아니라 작성 방식이 결정한다.

관련 가이드:
- 슬라이드 도구 사용법 → `slides.md`
- 일러스트 통합 강의 슬라이드 워크플로우 → `lecture_slides_with_illustrations.md`
- 본 가이드 → **무엇을 어떻게 채울 것인가**

---

## 1. 언제 이 가이드를 따르는가

- 책 강의, 학교 수업, 강연, 교육용 발표, 워크숍, 마스터 클래스
- 사용자가 "강의용", "강사가 사용할", "수업", "교육 자료", "전문 발표", "마스터 클래스" 같은 단어를 쓸 때
- 일반 카드뉴스·마케팅·홍보 슬라이드는 이 가이드 불필요

---

## 2. 네 가지 핵심 원칙 (Mayer · Reynolds · Tufte 통합)

이 네 가지를 어기지 않으면 강의 슬라이드는 절반 성공이다.

### ① 일관성 원칙 (Coherence) — 관련 없는 건 다 빼라
주제와 직접 관련 없는 장식·이미지·효과는 학습을 **방해**한다 (Mayer가 실험으로 입증).
- 메타포가 아닌 단순 장식 이미지 금지
- 본문과 무관한 인용 금지
- "예쁘게 보이려는" 그라데이션·도형 금지 — 의미를 가질 때만

### ② 중복 회피 (Redundancy) — 같은 글을 보이고 읽지 마라
강사가 슬라이드의 문장을 그대로 읽으면 청중은 **양쪽 모두 놓친다** (읽기와 듣기가 같은 인지 채널을 차지).
- 슬라이드 본문 = 키워드·핵심 명제만
- 강사 발화의 상세 설명은 **스피커 노트**(§7)로 분리
- "전체 문장을 슬라이드에 적고 강사가 그대로 읽기" — 최악의 패턴

### ③ 시그널링 (Signaling) — 핵심을 시각적으로 강조하라
중요한 부분은 색·굵기·박스로 표시. 시그널이 없으면 청중은 어디를 봐야 할지 모름.
- 핵심 명제는 `quote` 박스나 강조 색
- 부차적 정보는 `factbox`로 분리 (본문과 시각적 구분)
- 모든 슬라이드의 강조 방식이 **일관**되어야 함 (디자인 시스템 활용)

### ④ 공간 근접성 (Spatial Contiguity) — 캡션은 관련 이미지 옆에
이미지 설명은 **이미지 바로 옆에** 두어야 함. 이미지 위에 설명을 띄우거나 다른 슬라이드로 분리하면 인지 부담이 커짐.
- `hero_image`/`content_image` 사용 시 텍스트가 이미지와 같은 슬라이드에
- 이미지와 본문을 별도 슬라이드로 나누지 않기

## 3. 시맨틱 블록 라벨 표준 (eyebrow 활용)

슬라이드 종류를 청중이 즉시 인식할 수 있어야 함. 각 슬라이드의 `eyebrow` 필드에 **표준 라벨**을 통일해서 쓴다. 같은 라벨은 같은 종류를 의미.

| eyebrow 라벨 | 의미 | 적합 layout |
|---|---|---|
| `정의` | 개념 정의 | `lecture_body` 또는 `factbox` |
| `예시` | 구체 사례 | `lecture_body` 또는 `metaphor_story` |
| `비교` | 두 항목 대비 | `comparison_table` |
| `메타포` | 비유·은유 | `metaphor_story` |
| `사례` | 실제 케이스 | `lecture_body` |
| `사실` 또는 `팩트` | 객관 정보 | `factbox` |
| `주의` | 경고·오해 | `lecture_body` (warn 톤) |
| `심화` | 더 깊은 내용 | `lecture_body` |
| `생각해보기` | 청중 질문 | `lecture_body` 또는 `quote` |
| `요약` | 챕터 마무리 | `lecture_body` |
| `질문` | 토론 유도 | `quote` |
| `인용` | 외부 인용 | `quote` |

eyebrow에 부·장·진행도 함께 묶을 수 있음 (§4 참조).

---

## 4. 진행 표시 (청중이 강의 어디쯤인지 알게)

eyebrow 또는 footer에 **위치·진행 정보**를 표준 형식으로 적는다.

### 형식
```
eyebrow: "1부 · 1장 · 정의"      # 부·장·시맨틱 라벨 결합
footer:  "Slide 3 / 15"          # 슬라이드 번호 / 총 수
```

또는 합쳐서:
```
eyebrow: "1부 · 1장 · 3/15 · 정의"
```

### 권장
- **eyebrow**: 시맨틱 라벨 + 부·장 (의미적 위치)
- **footer**: 슬라이드 번호 + 출처 (수치·근거)

청중이 매 슬라이드에서 *"지금 어디 부의 어떤 종류 슬라이드구나"*를 한눈에 인식.

---

## 5. 출처 표기 (책 강의의 신뢰성)

책 강의·학술 발표는 **출처 명기가 필수**. 청중이 더 깊이 알고 싶을 때 원본을 찾아갈 수 있어야 함.

### 슬라이드 단위 출처
```
footer: "강국진, 『하네스란 무엇인가?』 1부 2장 · p.42"
```

### 인용 슬라이드
```
{
  layout: "quote",
  quote: "팔이 없으면 뇌는 사과를 잡는 문제를 정의할 수 없다.",
  attribution: "강국진, 『하네스란 무엇인가?』 1부 2장",
  context: "맥락 부연"
}
```

### 팩트박스 출처
```
{layout: "factbox", title: "...", body: "...", source: "참고 문헌명"}
```

### 강의 마지막 — 참고 문헌 슬라이드
강의 마지막에 `factbox` 또는 `lecture_body`로 전체 인용·참고 자료 정리:
```
{
  layout: "factbox",
  eyebrow: "참고 문헌",
  title: "이 강의에서 인용한 자료",
  items: [
    "강국진, 『하네스란 무엇인가?』 (2026)",
    "Mayer, R. — Multimedia Learning Principles",
    "..."
  ]
}
```

## 6. 한 슬라이드 = 한 아이디어 (분량 규칙)

한 슬라이드는 **한 가지를 가르친다**. 두 가지를 가르치려면 두 슬라이드로 쪼갠다.

### 슬라이드별 텍스트 분량 권장
| 필드 | 권장 분량 | 한계 |
|------|----------|------|
| `title` | 1줄 (15자 이내 한글) | 25자 |
| `body` | 1~2문장 (강사 발화 단위) | 3문장 |
| `bullets` | 3~5개 | 7개 (Miller의 7±2) |
| `quote` | 1줄 핵심 | 2줄 |
| `footer` | 한 줄 (출처·진행) | 한 줄 고정 |
| `story` (metaphor_story) | 3~5문장 | 7문장 |
| `items` (factbox) | 3~5개 | 7개 |

본문이 7문장을 넘어가면 두 슬라이드로 쪼갤 신호. 불릿이 7개를 넘어가면 그룹화 필요.

### 강의 전체 분량 권장
| 강의 길이 | 슬라이드 수 | 슬라이드당 시간 |
|----------|------------|----------------|
| 30분 강의 | 15~20장 | 약 2분 |
| 60분 강의 | 25~35장 | 약 2분 |
| 90분 강의 | 35~50장 | 약 2분 |
| 책 1부 마스터 클래스 | 30~60장 | 부의 분량·깊이에 따라 |

너무 적으면 한 슬라이드에 너무 많이 담는 강요(중복 원칙 위반), 너무 많으면 청중이 따라가지 못함.

---

## 7. 스피커 노트 동반 파일 (강사 ↔ 청중 분리)

**슬라이드 본문 ≠ 강사 발화**. 슬라이드는 청중이 보는 키워드, 강사 발화는 더 풍부한 설명. 둘을 분리하는 표준 방법:

### 워크플로우
1. `[engines:slide_shadcn]`로 슬라이드 PNG 생성
2. `[self:write]`로 동반 파일 `speaker_notes.md` 작성:

```markdown
# 강사용 노트 — 하네스 1부 강의

## Slide 1 — 표지
- 강의 시작: 자기 소개 1분
- 책의 5가지 키 컨셉 예고 (이름만)
- 청중에게 질문: "AI 모델만으로 일이 끝나는가?"

## Slide 2 — 1부 개요
- 강조점: "하네스는 부품이 아니라 언어다"는 1부의 핵심 명제
- 청중이 가질 수 있는 의문: "왜 하필 언어인가?"
- 답: 5장 끝까지 가야 완전히 이해됨 — 예고만 하고 본론으로

## Slide 3 — 햄릿 메타포
- 메타포 차근차근 풀기 (2분 추천):
  1. 배우 = 학습된 AI 모델 (지식은 다 있음)
  2. 연출가의 한마디 = 프롬프트
  3. 한마디가 없으면 → 같은 배우가 다른 연극
- 청중 반응 살피기 — 메타포가 와닿지 않으면 영화 비유로 보강
- 다음 슬라이드 연결: "한마디가 어떻게 한 권의 책으로 자라났는가"

... (각 슬라이드마다)
```

3. PDF로 합치고 싶으면: 슬라이드 데크 + 스피커 노트를 `pandoc` 또는 직접 합본

### 핵심
- **슬라이드 = 청중이 보는 글**
- **스피커 노트 = 강사가 보는 글**
- 둘이 같은 내용 반복하면 중복 원칙(②) 위반

## 8. 회피해야 할 실수

이전 사례에서 반복적으로 발견된 실수:

1. **본문 단락을 슬라이드에 그대로 옮기기** — 강사가 읽으면 중복. body 필드는 키워드 중심으로 짧게.
2. **한 슬라이드에 5개 이상의 다른 컨셉 담기** — 한 슬라이드 = 한 아이디어 원칙 위반.
3. **글머리표 8개 이상** — 인지 한계 초과. 그룹화하거나 슬라이드 쪼개기.
4. **장식 이미지** — 메타포가 아닌 이미지는 학습 방해 (일관성 원칙).
5. **출처 누락** — 책 강의에서 인용한 문장에 attribution 없으면 신뢰성 ↓.
6. **eyebrow 라벨 들쭉날쭉** — "1장", "1부 1장", "Chapter 1"이 섞이면 청중이 혼란.
7. **진행 표시 없음** — 청중이 강의 어디쯤인지 모름.
8. **모든 슬라이드가 lecture_body 한 종류** — 시각적 리듬 없음. metaphor/quote/factbox를 섞어 변화.
9. **스피커 노트 없이 슬라이드만 만들기** — 강사가 슬라이드에 의존하게 됨 → 슬라이드 본문이 길어짐 → 중복 원칙 위반의 악순환.
10. **첫 슬라이드부터 본격 콘텐츠** — 표지 + 1부 개요로 청중이 "지도"를 먼저 갖게 한 다음 본론.

---

## 9. 한 강의의 표준 흐름 (책 한 부 기준)

```
1. 표지 (hero)
2. 1부 개요 — 무엇을 배울 것인지 (lecture_body, eyebrow: "개요")
3. 1부 도입 메타포 (metaphor_story, eyebrow: "메타포")
4-N. 각 장 → 정의 / 예시 / 비교 / 인용 / 팩트박스 조합 (시맨틱 라벨 활용)
N+1. 1부 요약 (lecture_body, eyebrow: "요약")
N+2. 핵심 명제 한 줄 (quote)
N+3. 다음 부 예고 또는 토론 질문 (lecture_body, eyebrow: "생각해보기")
N+4. 참고 문헌 (factbox, eyebrow: "참고 문헌")
```

이 흐름이 청중에게 **시작-본론-마무리**의 명확한 구조를 제공한다.

---

## 10. 한 줄 요약

> 슬라이드 = 청중이 보는 키워드. 강사 발화 = 스피커 노트. 한 슬라이드 = 한 아이디어. eyebrow = 시맨틱 라벨 + 진행. footer = 출처. 이 다섯을 지키면 강의 슬라이드가 된다.
