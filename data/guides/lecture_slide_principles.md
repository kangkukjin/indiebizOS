# 강의 슬라이드 협업 가이드

강의·교육용 슬라이드는 **AI가 한 번에 만들어주는 산출물이 아니다**. 강의자의 호흡·사고·청중 감각을 무시하면 아무리 멋진 슬라이드도 그 강의자의 강의가 아니다. 이 가이드는 AI 에이전트가 **강의자와 한 장씩 협업하며 슬라이드를 빚어가는 방식**을 정의한다.

품질 원칙(§3 이하)은 그대로지만, 그것을 한 번에 일괄 적용하지 않고 **매 협업 라운드마다 한 장씩 적용**한다. AI가 알아서 질문하고 강의자가 결정을 내리는 흐름이 본 가이드의 골격이다.

관련 가이드:
- 슬라이드 도구 사용법 → `slides.md`
- 일러스트 통합 강의 슬라이드 워크플로우 → `lecture_slides_with_illustrations.md`
- 본 가이드 → **어떻게 강의자와 함께 슬라이드를 만들어 갈 것인가**

<core_rules priority="critical">
강의 슬라이드 협업의 4대 원칙. 어기면 결과물이 "강의자의 강의"가 아니라 "AI의 요약본"이 된다:

1. **한 번에 다 만들지 마라** — 강의자의 호흡과 사고를 무시하는 가장 큰 실수. §2 Phase C의 한 장 단위 루프를 따른다. 첫 장 → 코멘트 → 다음 장.
2. **추상적 질문 금지** — "어떤 스타일을 원하세요?", "톤은 어떻게 할까요?"는 강의자에게 결정 부담을 떠넘기는 질문. 항상 **구체적 선택지 2~3개** + **선택지 외 자유 답변을 받는 열린 질문 1개** 형태로.
3. **누적 메모 유지** — 강의자가 거부한 방향·채택한 톤·좋아한 메타포·강조 표현을 매 라운드 기록. 다음 장 제안 시 반영해서 강의자의 호흡을 학습.
4. **메시지 큐레이션은 매 장 적용** — §3의 5단계는 outline 사전 완성 강요가 아니라 한 장 만들 때마다 거치는 사고 도구다. Phase B에서 명제 후보를 던지고, Phase C에서 한 장씩 매핑·구체화한다.
</core_rules>

---

## 1. 언제 이 가이드를 따르는가

- 책 강의, 학교 수업, 강연, 교육용 발표, 워크숍, 마스터 클래스
- 사용자가 "강의용", "강사가 사용할", "수업", "교육 자료", "전문 발표", "마스터 클래스" 같은 단어를 쓸 때
- 일반 카드뉴스·마케팅·홍보 슬라이드는 이 가이드 불필요 (`slides.md`만으로 충분)

본 가이드를 따르기로 결정했다면 **즉시 §2 Phase A부터 시작한다**. 강의자가 "한 번에 다 만들어줘"라고 하더라도 협업 모드를 유지한다 — 그게 강의 슬라이드의 본질이기 때문이다.

## 2. 협업 워크플로우 — 한 장씩 만들어가기

이 워크플로우 자체가 본 가이드의 핵심이다. AI 에이전트는 강의 슬라이드 작업을 시작하면 **즉시 Phase A부터 진행**하고, 강의자의 답변을 따라 Phase B → C → D → E로 흘러간다.

### 전체 흐름

```
A 초기 합의 (5개 질문, 가볍게)
   ↓
B 명제 후보 제시 → 강의자가 채택/거부/수정
   ↓
C 한 장씩 만들기 (5단계 라운드 반복)  ★ 협업의 본체
   ① 다음 장 제안   ② 강의자 코멘트   ③ 한 장 작성
   ④ 누적 메모 업데이트   ⑤ 다음 장으로
   ↓
D 본론 끝 — 회수 슬라이드 2장 합의 (자동 생성 금지)
   ↓
E 스피커 노트 (선택)
```

---

### Phase A — 초기 합의 (가볍게)

전체 outline을 미리 짜놓지 않는다. 다음 5개만 합의되면 충분:

| # | 질문 | 형식 |
|---|------|------|
| 1 | 어떤 책/주제로 강의를 만드시나요? | 책이면 어느 부·장부터인지도 |
| 2 | 누가 듣는 강의인가요? | 일반인 / 학생 / 전문가 / 워크숍 |
| 3 | 청중이 강의장을 떠나며 절대 잊지 말아야 할 단 한 가지는? | 한 줄 단정문 |
| 4 | 강의 분량은? | 30분 / 60분 / 90분 / 책 한 부 마스터 클래스 |
| 5 | 양식 모드는? | 클래식 강의록 vs NotebookLM 인포그래픽 (`lecture_slides_with_illustrations.md` §1 모드 분기 참조) |

**질문 방식**: 다섯 개를 동시에 던지지 말고 1~2개씩, 혹은 강의자 답변에서 자연스럽게 다음으로 이어간다. 강의자가 "내 책 X의 1부, 60분 강의로 일반인 대상"처럼 한 줄로 답하면 부족한 항목만 보강 질문.

---

### Phase B — 명제 후보 제시 → 다듬기

1. AI가 책·주제에서 **명제 후보 5~7개**를 추출 (§3 ① 명제 추출 원칙).
2. **테이블 형태로 제시** — 강의자가 각각에 ✓/✗/수정 표시하기 쉽게:

```markdown
| # | 명제 후보 | 비고 |
|---|---|---|
| 1 | 나는 무지개를 직접 보지 않는다 | 1부 1장 |
| 2 | AI 모델 ≠ AI의 실체 | 1부 2장 |
| 3 | 한 번의 속삭임에서 끊임없는 대화로 | 1부 3장 |
| ... | ... | ... |
```

3. 강의자가 짧게 답: "1, 3은 ✓. 2는 '모델만으론 못 움직인다'로 수정. 7번 거부 — 너무 일반적."
4. **거부 사유를 누적 메모에 즉시 기록** ("강의자는 일반 명제 거부, 구체적 표현 선호").
5. 명제 N개가 합의되면 Phase C로. **outline 표 사전 완성은 강요하지 않는다** — Phase C에서 한 장씩 다듬어 간다.

---

### Phase C — 한 장씩 만들기 (핵심 협업 루프)

이 루프가 본 가이드의 본체. 매 슬라이드마다 5단계 라운드 반복.

#### ① 다음 장 제안 (AI 발화)

선택지 + 열린 질문 형태로:

```
다음 슬라이드는 명제 N번 「나는 무지개를 직접 보지 않는다」를 풉니다.
풀어내는 구도는:
 (a) 단계 다이어그램 — 뇌→신호→눈→무지개의 인과 흐름
 (b) 좌우 대비 — "보는 것 같다 vs 실은 뇌 속에서 만든다"
 (c) 위 두 가지 외 다른 구도

어느 쪽이 좋으실까요? 떠오르는 메타포가 있으시면 알려주세요.
```

- **구체적 선택지 2개** (§3 ③ 5종 수사 장치 중에서).
- **(c) 다른 구도** 항목으로 강의자의 자유도 확보.
- **열린 질문 1개**로 메타포·강조 의견 유도.

#### ② 강의자 코멘트 (사용자 발화 받기)

세 가지 유형:

- **짧은 채택**: "(b)로 가요. '뇌가 만든다' 강조" → ③로 진행.
- **방향 수정**: "둘 다 아니고 '내가 보는 무지개는 실제와 다르다'를 수치 비교로" → 수사 장치 재매핑 후 진행.
- **명제 자체 수정/재배치**: "이 명제는 다음 장에 묶을게요. 먼저 메타포부터" → 명제 순서 재배치, 누적 메모 업데이트.

#### ③ 한 장 작성 (AI)

- 합의된 명제 + 수사 장치 + 강의자 강조점으로 슬라이드 한 장 생성.
- 슬라이드 도구(`slides.md` / `lecture_slides_with_illustrations.md`) 호출 → PNG 한 장.
- §3~§10의 품질 원칙은 매 장에 그대로 적용.
- **한 장만 만든다** — 다음 장은 ⑤에서 새로 제안.

#### ④ 누적 메모 업데이트 (AI 내부 — 다음 라운드를 위해)

매 라운드 끝에 다음 형식으로 기록·갱신:

```markdown
## 누적 메모 (강의자의 호흡)

### 채택한 톤·표현
- "뇌가 만든다" 같은 능동형 표현 선호
- 영문 라벨 OK ("EYE", "BRAIN", "RAINBOW")
- 한 장에 한 메타포

### 거부한 방향
- "AI는 도구다" 같은 일반 명제 → 너무 추상적
- 좌우 대비 일관 사용 → 리듬 변화 원함

### 강의자 메타포 어휘
- "햄릿의 배우와 연출가"
- "유리병 속 뇌"
- "깨진 유리병 + 하네스"

### 강조점
- "뇌가 적극적으로 만드는 것"임을 반복
- 청중에게 던지는 질문 형태 선호
```

이 메모는 의식 에이전트가 다음 라운드 ①번 단계에서 참고한다.

#### ⑤ 다음 장 제안 (①로 돌아감)

직전 메모를 반영해 같은 톤·리듬으로 다음 장 제안. 명제 N개가 모두 슬라이드로 만들어질 때까지 ①~⑤ 반복.

---

### Phase D — 본론 끝, 회수 슬라이드 합의

본론 명제 N장이 모두 만들어졌으면 AI가 회수 슬라이드 두 장을 **제안**한다 (자동 생성 X, 항상 합의 필요):

1. **회수 1 — 비교표**:
   > "1부 전체를 비교표로 정리하려고 합니다. 비교축은:
   > (a) 과거 패러다임 vs 새 패러다임
   > (b) 모델 vs 모델+하네스
   > (c) 강의자가 원하는 축
   > 어느 쪽?"

2. **회수 2 — 체계 정리 다이어그램**:
   > "체계 정리는:
   > (a) 동심원 5층 (AI Model / Harness / Body / Ontology / Language)
   > (b) 단계 흐름
   > (c) 계층 트리
   > 어느 구도?"

3. 합의 후 두 장 생성. **이중 회수는 절대 건너뛰지 않는다** (§3 ⑤). 강의자가 "인용 한 장으로 끝내자"고 해도 회수를 먼저 권유 — 인지로 닫히지 않으면 강의가 닫히지 않는다.

---

### Phase E — 스피커 노트 (선택)

```
슬라이드별 스피커 노트도 같이 만들까요? (실제 강의 발화 시 도움)
 (a) 매 장 Phase C ③ 단계에서 같이
 (b) 전체 슬라이드 완성 후 마지막에 일괄
 (c) 필요 없음
```

(a)나 (b)를 선택하면 §9 형식으로 `speaker_notes.md` 동반 파일 작성.

---

### Phase C 한 라운드 예시 (참고)

```

     구도는 (a) 좌우 대비 (유리병 속 뇌 vs 깨진 유리병+하네스),
            (b) 등식 (모델 + 하네스 = 운동),
            (c) 다른 구도

강사: (a). 그런데 "갇혀있다"는 느낌을 강하게.

AI:  (한 장 생성 — 유리병이 더 폐쇄적으로, 깨진 유리병은 동적으로)

     ※ 누적 메모 추가: "갇힘 → 풀려남" 동적 대비를 선호. 다음 메타포도 동적 표현.

AI:  다음 슬라이드는 명제 4번 「한 번의 속삭임에서 끊임없는 대화로」입니다.
     ...
```

## 3. 메시지 큐레이션 5단계 — 매 장 적용하는 사고 도구

**이 5단계가 슬라이드 품질을 결정한다.** 사전에 전체 outline에 일괄 적용하는 게 아니라, **Phase C 매 라운드에서 한 장씩 적용하는 사고 도구**다.

| 단계 | 적용 시점 |
|------|----------|
| ① 명제 추출 | Phase B에서 후보 5~7개, Phase C에서 매 장 선택 |
| ② 슬라이드 제목 = 명제 | Phase C ③ 작성 시 검증 |
| ③ 수사 장치 매핑 | Phase C ① 제안 시 선택지 구성에 사용 |
| ④ 구체화 의무 | Phase C ③ 작성 시 적용 |
| ⑤ 이중 회수 | Phase D에서 강의자와 합의 |

### ① 명제 추출 (Thesis Extraction)
원문(책 한 부·한 챕터·논문)에서 **"청중이 절대 잊으면 안 될 한 줄 단정문" N개**를 추출.

- 단정문 형식: ✓ "나는 무지개를 직접 보지 않는다" / ✗ "시각의 작동 원리"
- 챕터 제목·소제목을 그대로 가져오는 것은 금지 — 보통 명제가 아니라 라벨이다.
- 청중에게 던졌을 때 "그게 정말인가?", "왜 그런가?"를 일으키는 문장이 좋은 명제.
- **Phase B에서 5~7개를 후보로 제시하고, 강의자가 채택한 N개를 슬라이드 명제로 확정**.

### ② 슬라이드 제목 = 명제 (Title = Thesis)
**슬라이드 제목 자체가 명제여야 한다.** 청중이 슬라이드 제목만 봐도 핵심을 얻을 수 있도록.

- ✗ "지워진 신경신호, 그리고 몸을 주는 언어" (챕터명 — 라벨)
- ✓ "나는 무지개를 직접 보지 않는다" (명제 — 단정문)
- ✗ "에이전틱 루프: 행동하고 관찰하는 지능" (라벨)
- ✓ "한 번의 속삭임에서 끊임없는 대화로" (명제)
- ✗ "AI의 좁은 시야와 시스템의 순환" (라벨)
- ✓ "AI에게 디버깅을 맡기면 금지 목록만 늘어난다" (명제)

판단 기준: 슬라이드 제목을 청중 앞에서 큰 소리로 읽었을 때 "그래서?"가 따라오면 라벨, "정말?"이 따라오면 명제.

### ③ 수사 장치 매핑 (Rhetorical Device Mapping)
각 명제를 **5종 수사 장치 중 하나**에 매핑. Phase C ① 제안 시 이 표에서 선택지 2~3개를 구성.

| 장치 | 적합 명제 | 권장 레이아웃 |
|---|---|---|
| **(a) 좌우 대비** | A는 X, B는 Y처럼 두 항을 대비하는 명제 | `split_concept` 또는 `illustration_overlay`(한 일러스트 안에 좌우 패널) |
| **(b) 단계/체인 다이어그램** | A→B→C→D처럼 흐름·과정·인과를 가진 명제 | `illustration_overlay`(상단 가로 화살표 + 단계별 라벨) |
| **(c) 등식·매핑** | A + B = C 또는 A↔B 1:1 매핑 명제 | `illustration_overlay` 또는 `comparison_iconic` |
| **(d) 수치 폭증·감소** | 시간에 따라 X가 N배로 변한 명제 (수치가 핵심) | `factbox` 또는 `illustration_overlay`의 수치 박스 |
| **(e) 동심원·계층** | 외부→내부 또는 표층→심층 구조 명제 | `illustration_overlay`(동심 다이어그램) |

예시 매핑:
- "AI 모델 ≠ AI의 실체" → (a) 좌우 대비 (유리병 속 뇌 vs 깨진 유리병+하네스)
- "나는 무지개를 직접 보지 않는다" → (b) 단계 다이어그램 (의식→신호→망막→외부세계)
- "AI 모델 + 하네스 = 실제 운동" → (c) 등식 (F=ma + 진자 = 운동)
- "컨텍스트 창이 폭발적으로 커졌다" → (d) 수치 (4,096 → 1,000,000+ 토큰)
- "하네스는 다섯 층 구조다" → (e) 동심원 (AI Model / Harness / Body / Ontology / Language)

### ④ 구체화 의무 (Concretization)
본문에 **수치·인명·연도·고유명사·결정 사례**가 있으면 평문 단락에 묻어두지 말고 **시각 요소로 격리**.

- 4,096 → 1,000,000+ 토큰 같은 수치는 본문 줄에 두지 말고 별도 박스/하이라이트.
- v0(600줄) → v5(200줄) 같은 대비는 좌우 패널.
- "Mayer의 다중미디어 학습 원리", "얀 르쿤의 시각", "2026-02 출시" 같은 고유명사·연도도 시각적으로 분리.
- 구체적이지 않은 추상 명제는 ①로 돌아가 다듬는다. **"진짜 좋은 슬라이드 = 청중이 1초 안에 수치·구도·대비를 알아본다"**.

### ⑤ 이중 회수 (Double Closure)
강의 한 부의 **마지막 두 장**은 회수 슬라이드로 고정. Phase D에서 강의자와 합의.

- **회수 1 — 비교표** (`comparison_iconic` 또는 `comparison_table`): N축으로 "과거 패러다임 vs 새 패러다임"을 정리. 청중의 인지적 마무리.
- **회수 2 — 체계 정리 다이어그램** (`illustration_overlay` 동심원 또는 계층): 1부 전체에서 등장한 개념들의 관계를 한 장으로. 청중의 구조적 마무리.
- **(선택) 인용 슬라이드**: 그 다음에 `quote`로 정서적 마무리. 단, 회수 슬라이드를 건너뛰고 인용으로만 끝내면 안 된다 — 정서로는 닫혀도 인지로는 안 닫힌다.

### 작업 산출물 (협업 모드 개정)
협업 모드에서는 **사전 outline 표를 강요하지 않는다**. 대신:
- Phase B의 산출물: 채택된 명제 리스트 (단순 목록으로 충분)
- Phase C 매 라운드의 산출물: 한 장 PNG + 누적 메모 업데이트
- Phase D의 산출물: 회수 슬라이드 2장에 대한 강의자 합의

기존의 "표 형식 outline 사전 작성"은 **선택사항**. 강의자가 "전체 outline을 먼저 보고 싶다"고 명시적으로 요청할 때만 만들고, 기본은 한 장씩 진행.

</critical_workflow>

## 4. 네 가지 핵심 원칙 (Mayer · Reynolds · Tufte 통합)

이 네 가지를 어기지 않으면 강의 슬라이드는 절반 성공이다. **매 라운드 Phase C ③ 작성 단계에서 자동 적용**.

### ① 일관성 원칙 (Coherence) — 관련 없는 건 다 빼라
주제와 직접 관련 없는 장식·이미지·효과는 학습을 **방해**한다 (Mayer가 실험으로 입증).
- 메타포가 아닌 단순 장식 이미지 금지
- 본문과 무관한 인용 금지
- "예쁘게 보이려는" 그라데이션·도형 금지 — 의미를 가질 때만

### ② 중복 회피 (Redundancy) — 같은 글을 보이고 읽지 마라
강사가 슬라이드의 문장을 그대로 읽으면 청중은 **양쪽 모두 놓친다** (읽기와 듣기가 같은 인지 채널을 차지).
- 슬라이드 본문 = 키워드·핵심 명제만
- 강사 발화의 상세 설명은 **스피커 노트**(§9)로 분리
- "전체 문장을 슬라이드에 적고 강사가 그대로 읽기" — 최악의 패턴

### ③ 시그널링 (Signaling) — 핵심을 시각적으로 강조하라
중요한 부분은 색·굵기·박스로 표시. 시그널이 없으면 청중은 어디를 봐야 할지 모름.
- 핵심 명제는 `quote` 박스나 강조 색
- 부차적 정보는 `factbox`로 분리 (본문과 시각적 구분)
- 모든 슬라이드의 강조 방식이 **일관**되어야 함

### ④ 공간 근접성 (Spatial Contiguity) — 캡션은 관련 이미지 옆에
이미지 설명은 **이미지 바로 옆에** 두어야 함. 이미지 위에 설명을 띄우거나 다른 슬라이드로 분리하면 인지 부담이 커짐.
- 통합형 레이아웃(`hero_illustration` / `illustration_anchor` / `split_concept` / `illustration_background`) 우선 — 이미지와 텍스트가 한 화면 안에서 융합됨
- 단순 좌우 분할이 꼭 필요할 때는 `hero_image`/`content_image`
- 이미지와 본문을 별도 슬라이드로 나누지 않기 — NotebookLM 같은 융합 효과는 한 화면 안에 다 담아야 나옴

## 5. 시맨틱 블록 라벨 표준 (eyebrow 활용)

슬라이드 종류를 청중이 즉시 인식할 수 있어야 함. 각 슬라이드의 `eyebrow` 필드에 **표준 라벨**을 통일해서 쓴다.

| eyebrow 라벨 | 의미 | 적합 layout |
|---|---|---|
| `정의` | 개념 정의 | `lecture_body` 또는 `factbox` |
| `예시` | 구체 사례 | `lecture_body` 또는 `metaphor_story` |
| `비교` | 두 항목 대비 | `comparison_table` |
| `메타포` | 비유·은유 | `metaphor_story` |
| `사례` | 실제 케이스 | `lecture_body` |
| `사실` 또는 `팩트` | 객관 정보 | `factbox` |
| `주의` | 경고·오해 | `lecture_body` (warn 톤) |
| `심화` | 더 깊은 내용 | `lecture_body` |
| `생각해보기` | 청중 질문 | `lecture_body` 또는 `quote` |
| `요약` | 챕터 마무리 | `lecture_body` |
| `질문` | 토론 유도 | `quote` |
| `인용` | 외부 인용 | `quote` |

eyebrow에 부·장·진행도 함께 묶을 수 있음 (§6 참조).

## 6. 진행 표시 (청중이 강의 어디쯤인지 알게)

eyebrow 또는 footer에 **위치·진행 정보**를 표준 형식으로 적는다.

### 형식
```
eyebrow: "1부 · 1장 · 정의"      # 부·장·시맨틱 라벨 결합
footer:  "Slide 3 / 15"          # 슬라이드 번호 / 총 수
```

또는 합쳐서:
```
eyebrow: "1부 · 1장 · 3/15 · 정의"
```

### 권장
- **eyebrow**: 시맨틱 라벨 + 부·장 (의미적 위치)
- **footer**: 슬라이드 번호 + 출처 (수치·근거)

청중이 매 슬라이드에서 *"지금 어디 부의 어떤 종류 슬라이드구나"*를 한눈에 인식.

## 7. 출처 표기 (책 강의의 신뢰성)

책 강의·학술 발표는 **출처 명기가 필수**.

### 슬라이드 단위 출처
```
footer: "강국진, 『하네스란 무엇인가?』 1부 2장 · p.42"
```

### 인용 슬라이드
```
{
  layout: "quote",
  quote: "팔이 없으면 뇌는 사과를 잡는 문제를 정의할 수 없다.",
  attribution: "강국진, 『하네스란 무엇인가?』 1부 2장",
  context: "맥락 부연"
}
```

### 팩트박스 출처
```
{layout: "factbox", title: "...", body: "...", source: "참고 문헌명"}
```

### 강의 마지막 — 참고 문헌 슬라이드
```
{
  layout: "factbox",
  eyebrow: "참고 문헌",
  title: "이 강의에서 인용한 자료",
  items: [
    "강국진, 『하네스란 무엇인가?』 (2026)",
    "Mayer, R. — Multimedia Learning Principles",
    "..."
  ]
}
```

## 8. 한 슬라이드 = 한 아이디어 (분량 규칙)

한 슬라이드는 **한 가지를 가르친다**. 두 가지를 가르치려면 두 슬라이드로 쪼갠다.

### 슬라이드별 텍스트 분량 권장
| 필드 | 권장 분량 | 한계 |
|------|----------|------|
| `title` | 1줄 (15자 이내 한글) | 25자 |
| `body` | 1~2문장 (강사 발화 단위) | 3문장 |
| `bullets` | 3~5개 | 7개 (Miller의 7±2) |
| `quote` | 1줄 핵심 | 2줄 |
| `footer` | 한 줄 (출처·진행) | 한 줄 고정 |
| `story` (metaphor_story) | 3~5문장 | 7문장 |
| `items` (factbox) | 3~5개 | 7개 |

본문이 7문장을 넘어가면 두 슬라이드로 쪼갤 신호. 불릿이 7개를 넘어가면 그룹화 필요.

### 강의 전체 분량 권장
| 강의 길이 | 슬라이드 수 | 슬라이드당 시간 |
|----------|------------|----------------|
| 30분 강의 | 15~20장 | 약 2분 |
| 60분 강의 | 25~35장 | 약 2분 |
| 90분 강의 | 35~50장 | 약 2분 |
| 책 1부 마스터 클래스 | 30~60장 | 부의 분량·깊이에 따라 |

이 권장 수치는 Phase A 4번 질문의 답을 받아 N(본론 명제 수)을 정할 때 참고.

## 9. 스피커 노트 동반 파일 (강사 ↔ 청중 분리)

**슬라이드 본문 ≠ 강사 발화**. 슬라이드는 청중이 보는 키워드, 강사 발화는 더 풍부한 설명.

### 워크플로우 (Phase E)
1. `[engines:slide_shadcn]`로 슬라이드 PNG 생성 (Phase C ③)
2. `[self:write]`로 동반 파일 `speaker_notes.md` 작성 (Phase E):

```markdown
# 강사용 노트 — 하네스 1부 강의

## Slide 1 — 표지
- 강의 시작: 자기 소개 1분
- 책의 5가지 키 컨셉 예고 (이름만)
- 청중에게 질문: "AI 모델만으로 일이 끝나는가?"

## Slide 2 — 1부 개요
- 강조점: "하네스는 부품이 아니라 언어다"는 1부의 핵심 명제
- 청중이 가질 수 있는 의문: "왜 하필 언어인가?"
- 답: 5장 끝까지 가야 완전히 이해됨 — 예고만 하고 본론으로

## Slide 3 — 햄릿 메타포
- 메타포 차근차근 풀기 (2분 추천):
  1. 배우 = 학습된 AI 모델 (지식은 다 있음)
  2. 연출가의 한마디 = 프롬프트
  3. 한마디가 없으면 → 같은 배우가 다른 연극
- 청중 반응 살피기 — 메타포가 와닿지 않으면 영화 비유로 보강
- 다음 슬라이드 연결: "한마디가 어떻게 한 권의 책으로 자라났는가"

... (각 슬라이드마다)
```

3. PDF로 합치고 싶으면: 슬라이드 데크 + 스피커 노트를 `pandoc` 또는 직접 합본

### 핵심
- **슬라이드 = 청중이 보는 글**
- **스피커 노트 = 강사가 보는 글**
- 둘이 같은 내용 반복하면 중복 원칙(§4 ②) 위반

## 10. 한 강의의 표준 흐름 (협업 모드 적용)

협업으로 한 장씩 만들지만, **최종 데크의 구조적 흐름은 다음과 같이 일관**되어야 한다.

```
1.   표지 (hero / illustration_background)
2.   1부 개요 — 무엇을 배울 것인지 (lecture_body, eyebrow: "개요")
3.   1부 도입 명제 / 메타포 (metaphor_story 또는 illustration_overlay, eyebrow: "메타포")
4-N. 각 명제별로 1슬라이드: §3 ③ 수사 장치 매핑
N+1. **회수 1 — 비교표** (Phase D 합의 후) ★
N+2. **회수 2 — 체계 정리 다이어그램** (Phase D 합의 후) ★
N+3. (선택) 핵심 명제 인용 (quote)
N+4. 다음 부 예고 또는 토론 질문 (lecture_body, eyebrow: "생각해보기")
N+5. 참고 문헌 (factbox, eyebrow: "참고 문헌")
```

이 흐름이 청중에게 **시작-본론-이중 회수-마무리**의 명확한 구조를 제공한다. N+1·N+2를 생략하면 강의가 인지적으로 닫히지 않는다 (§3 ⑤).

협업 모드에서는 이 흐름을 **사전에 다 짜놓고 한 번에 생성하지 않는다**. Phase C 라운드를 거치며 한 장씩 채워가고, Phase D에서 N+1·N+2를 합의해서 완성한다.

## 11. 회피해야 할 실수

### 협업 단계 실수 (§2 위반 — 가장 자주 발생, 가장 치명적)

1. ❌ **한 번에 전체 데크 생성** — 강의자에게 PNG 30장을 한꺼번에 던지기. 강의자의 호흡 무시. **이게 가장 큰 실패** (§2 Phase C 위반, core_rules 1).
2. ❌ **추상적·열린 질문 남발** — "어떤 스타일을 원하세요?", "톤은 어떻게 할까요?". 강의자가 결정 부담을 안음. 항상 **구체적 선택지 2~3개 + 보조 열린 질문** 형태로 (§2 Phase C ①, core_rules 2).
3. ❌ **누적 메모 없이 진행** — 매 라운드 강의자의 코멘트를 학습하지 않으면 같은 실수 반복. 다음 장 제안 시 직전 라운드의 거부 사유를 다시 안내 (§2 Phase C ④, core_rules 3).
4. ❌ **회수 슬라이드 자동 생성** — Phase D에서 합의 없이 비교표·체계 정리 다이어그램을 자동 생성. 회수 축·구도는 강의자가 결정 (§2 Phase D).
5. ❌ **Phase A 5개 질문을 동시에 던지기** — 강의자가 짐 느낌. 1~2개씩 묻거나 답변에서 자연스럽게 다음 질문으로 (§2 Phase A).
6. ❌ **명제 후보를 5개 미만 또는 8개 이상 제시** — 너무 적으면 강의자의 선택 폭 부족, 너무 많으면 검토 부담. 5~7개가 적정 (§2 Phase B).
7. ❌ **outline 사전 완성 강요** — 강의자가 한 장씩 가고 싶은데도 "먼저 전체 outline 표를 만들겠습니다"로 일괄 작성. outline은 강의자가 명시적으로 원할 때만 (§3 작업 산출물).

### 기획·작성 단계 실수 (§3 이하 위반)

8. ❌ **명제 추출 없이 본문을 슬라이드로 옮기기** — 책의 본문 단락을 그대로 슬라이드 body에 붙여넣기. §3 ① 명제 추출 단계가 빠진 가장 흔한 실패.
9. ❌ **챕터명을 슬라이드 제목으로** — "지워진 신경신호, 그리고 몸을 주는 언어" 같은 라벨이 제목이 되면 청중은 핵심을 못 잡음. 제목은 명제여야 함 (§3 ②).
10. ❌ **수사 장치 매핑 누락** — 모든 슬라이드가 lecture_body 한 종류로 나옴. Phase C ①에서 5종 중 선택지를 제시하지 않으면 발생 (§3 ③).
11. ❌ **구체적 수치·사례를 평문에 묻기** — 본문에 "4,096 → 1,000,000+ 토큰"이 있는데 그냥 한 문장으로 흘려보내기. 격리해서 시각 요소로 분리 (§3 ④).
12. ❌ **회수 슬라이드 없이 인용으로만 마무리** — 인지로는 안 닫힘 (§3 ⑤).
13. ❌ **본문 단락을 슬라이드에 그대로 옮기기** — 강사가 읽으면 중복. body 필드는 키워드 중심으로 짧게.
14. ❌ **한 슬라이드에 5개 이상의 다른 컨셉 담기** — 한 슬라이드 = 한 아이디어 원칙 위반.
15. ❌ **글머리표 8개 이상** — 인지 한계 초과.
16. ❌ **장식 이미지** — 메타포가 아닌 이미지는 학습 방해 (일관성 원칙).
17. ❌ **출처 누락** — 책 강의에서 인용한 문장에 attribution 없으면 신뢰성 ↓.
18. ❌ **eyebrow 라벨 들쭉날쭉** — "1장", "1부 1장", "Chapter 1"이 섞이면 청중이 혼란.
19. ❌ **모든 슬라이드가 lecture_body 한 종류** — 시각적 리듬 없음.
20. ❌ **스피커 노트 없이 슬라이드만** — 강사가 슬라이드에 의존하게 됨 → 본문 길어짐 → 중복 원칙 깨짐.
21. ❌ **첫 슬라이드부터 본격 콘텐츠** — 표지 + 1부 개요로 청중이 "지도"를 먼저 갖게 한 다음 본론.

</antipatterns>

## 13. 한 줄 요약

> **강의 슬라이드는 산출물이 아니라 강의자와 AI의 협업 과정의 외화(外化)다.**
>
> - **방식 (§2)**: A 초기 합의 → B 명제 후보 → C 한 장씩(제안→코멘트→작성→메모→다음) → D 회수 합의 → E 스피커 노트. 한 번에 다 만들지 말 것.
> - **품질 (§3~§10)**: 매 장이 명제이고, 수사 장치 매핑, 구체화 의무, 이중 회수 필수.
> - **AI의 자세**: 강의자의 호흡을 학습하는 도우미. 자동 생성기가 아니다. 추상적 질문 금지, 구체적 선택지 + 누적 메모.
