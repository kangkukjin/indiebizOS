# 검색 가이드 (Search Guide)

검색은 단순한 데이터 조회가 아니다. 사용자에게 가치 있는 정보를 전달하는 것이 목적이다.

## 1. 깊이 있는 검색
- **숫자에 그치지 마라. "왜?"까지 파고들어라.**
  - 주가를 조회했으면 등락 원인도 찾아라
  - 날씨를 알려줬으면 내일 일정에 미치는 영향도 생각하라
  - 사실을 확인했으면 그 맥락과 의미도 전달하라
- **사용자가 묻지 않은 것도 중요하면 알려줘라**
  - 시장 폭락의 원인이 지정학적 사건이라면, 사건 자체도 요약하라

## 2. 소스 계획
- **정보의 성격에 맞는 검색 수단을 골라라**
  - 제품/구매 → 쇼핑 검색 포함
  - 의학/건강 → 논문 검색 고려
  - 투자 → 공시/종목 전용 검색
  - 지역 → 지도/맛집 전용 검색
  - 유튜브 영상 요약 → `summarize_video` (자막 추출+AI 요약+HTML+브라우저 열기를 한 번에)
  - 코드/라이브러리 문서 → `search_library_docs` (Context7 최신 공식 문서)
  - 일반 탐색 → 종합 검색부터
- **전문 사이트가 있으면 직접 크롤링하라**
  - 일본 제품 → 라쿠텐/아마존 재팬, 미국 제품 → 아마존 US
  - 논문 → Google Scholar/PubMed, 부동산 → 전문 포털
  - 범용 검색 엔진만으로는 전문 사이트의 결과를 놓칠 수 있음

## 3. 언어 계획
- **대상 국가의 언어로도 검색하라**
  - 일본 → 일본어, 글로벌 → 영어, 중국 → 중국어
- 한국어만으로는 현지 정보를 놓친다
- 같은 주제라도 언어에 따라 완전히 다른 결과가 나온다

## 4. 카테고리 계획
- **관련 카테고리를 넓게 잡아라**
  - "약" → 의약품 + 건강식품 + 한방약 + 영양제 + 서플리먼트
  - "투자" → 주식 + 채권 + ETF + 부동산 + 암호화폐
  - "교육" → 학교 + 온라인강좌 + 유튜브 + 도서
- 하나의 카테고리에 갇히면 관련 정보를 놓친다

## 5. 실행 전략: 넓게 → 좁게
1. **1차 (넓게)**: 다양한 키워드·언어·소스로 후보 탐색 — snippet이면 충분
2. **2차 (좁게)**: 유망한 후보 2-3개만 크롤링으로 상세 검증
3. 1차에서 원하는 결과가 없으면 키워드·카테고리·언어를 바꿔 재시도

## 6. 병렬 검색 제한
- **한 번에 병렬 검색(`&`)은 최대 3개까지만** — 그 이상은 결과가 누적되어 컨텍스트가 폭발한다
- 더 많은 검색이 필요하면 라운드를 나눠서 순차적으로 실행하라

## 7. 판단 원칙
- **"없다"고 단정하지 마라** — 못 찾은 것이지 없는 것이 아닐 수 있음
- **snippet만으로 사실을 확정하지 마라** — 핵심 정보는 원본 크롤링으로 검증
- **할루시네이션 금지** — 검색에서 확인되지 않은 제품명/수치를 만들어내지 말 것

## 8. 검색 결과 활용
- 검색 결과에 URL이 있으면 중요한 것은 크롤링으로 본문 확인
- 유튜브 결과가 있으면 자막 추출로 내용 확인 가능
- 검색 결과의 `_note`가 있으면 참고하라 (후속 액션 안내)

## 9. 분석이 필요하면 파이프라인을 쓰지 마라
- 검색 결과를 **분석, 비교, 요약, 보고서**로 만들어야 한다면 파이프라인(`>>`)으로 한번에 엮지 마라
- 파이프라인은 데이터를 기계적으로 전달할 뿐, 중간에 생각하지 않는다
- **IBL 액션을 하나씩 호출하고, 결과를 보고, 네가 생각한 다음, 다음 행동을 정해라**
- 예: "반도체 시장 분석해줘"
  1. 검색 실행 → 결과 확인
  2. 결과를 보고 핵심 트렌드 파악, 부족한 정보가 있으면 추가 검색
  3. 종합 분석 후 보고서 작성
- 이렇게 하면 검색 결과의 URL을 크롤링할지, 추가 검색이 필요한지, 어떤 관점에서 정리할지를 네가 판단할 수 있다
