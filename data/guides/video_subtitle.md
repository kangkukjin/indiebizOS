# 동영상 자막 제작 가이드: 진심을 전하는 이야기의 호흡

전문 스토리텔러로서, 자막은 단순히 정보를 전달하는 도구가 아니라 영상의 '언어적 잔상'을 남기는 예술적 장치입니다. 이 가이드는 나레이션의 감성을 온전히 보존하면서도 시각적 조화를 이루는 자막 제작법을 다룹니다.

## 1. 스토리텔링 및 심리학적 원칙
- **나레이션의 온전성**: 요약은 효율적이지만, 화자의 독특한 말투와 호흡은 나레이션 전문(Full Transcript)에 담겨 있습니다. 시청자가 화자의 감정에 깊이 공감하게 하려면 나레이션 그대로를 자막으로 제공하는 것이 좋습니다.
- **시각적 겸손함**: 자막은 주인공이 아닙니다. 영상의 미장센을 가리지 않도록 폰트 크기를 약간 줄이고, 배경과 대비되는 얇은 외곽선을 사용하여 '읽히되 보이지 않는' 상태를 지향합니다.
- **인지적 일치**: 들리는 말과 보이는 글자가 일치할 때 뇌는 정보를 더 편안하게 수입합니다.

## 2. 제작 단계별 체크리스트
1. **전사(Transcription)**: 나레이션 전체를 오타 없이 텍스트로 변환합니다.
2. **싱크(Syncing)**: 말의 시작과 끝에 맞춰 자막의 타임코드를 정밀하게 조정합니다.
3. **스타일링(Styling)**: 
   - **폰트 크기**: 일반적인 경우보다 10~20% 작게 설정 (예: FontSize=12~14)
   - **가독성**: 그림자보다는 얇은 검정 외곽선을 선호합니다.
   - **위치**: 하단 여백을 충분히 두어 시각적 답답함을 해소합니다.

## 3. 기술적 실행 (ffmpeg 명령어)

나레이션 전문을 포함한 `subtitle.srt` 파일을 영상에 입힐 때 사용하는 표준 명령어입니다.

```bash
ffmpeg -i "원본영상.mp4" -vf "subtitles='자막파일.srt':force_style='FontSize=12,PrimaryColour=&HFFFFFF,OutlineColour=&H000000,BorderStyle=3,Outline=0.5,Shadow=0,MarginV=25,Fontname=NanumGothic'" -c:a copy "결과영상.mp4"
```

### force_style 파라미터 설명

| 파라미터 | 값 | 설명 |
|----------|-----|------|
| `FontSize` | 12~14 | 일반 영상보다 10~20% 작게. 1080p 기준 12~14 권장 |
| `PrimaryColour` | `&HFFFFFF` | 자막 글자 색 (ASS 형식: &HBBGGRR) |
| `OutlineColour` | `&H000000` | 외곽선 색 (검정) |
| `BorderStyle` | 3 | 불투명 박스 배경 (1=외곽선만, 3=박스) |
| `Outline` | 0.5 | 외곽선 두께 (얇게) |
| `Shadow` | 0 | 그림자 없음 (깔끔한 느낌) |
| `MarginV` | 25 | 하단 여백 (픽셀). 화면 아래 답답함 방지 |
| `Fontname` | NanumGothic | 한글 폰트. 시스템에 설치되어 있어야 함 |

### SRT 파일 형식

```
1
00:00:01,000 --> 00:00:04,500
첫 번째 자막 문장입니다.

2
00:00:05,000 --> 00:00:08,200
두 번째 자막 문장입니다.
긴 문장은 이렇게\N줄바꿈합니다.
```

- 타임코드 형식: `HH:MM:SS,mmm` (밀리초는 콤마 구분)
- 줄바꿈: SRT 내에서 `\N` 사용
- 빈 줄이 자막 블록 구분자

## 4. 주의사항

- 문장이 화면의 2/3를 넘지 않도록 `\N`으로 줄바꿈하세요
- 배경이 밝아 자막이 안 보일 경우: `BackColour=&H80000000` (반투명 검정 배경) 추가
- 자막 시작/끝 타이밍은 말이 시작되기 0.1~0.2초 전, 말이 끝난 후 0.3~0.5초 후가 자연스러움
- 한 자막 블록에 2줄 초과는 피하기 — 읽기 부담이 커짐
- 자막 길이가 영상보다 길면 ffmpeg가 무시함 (에러는 아님, 단 뒷부분 누락)
