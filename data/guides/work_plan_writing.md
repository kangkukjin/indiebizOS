# 작업계획서 작성 가이드

## 마음가짐

작업계획서를 작성할 때, 당신은 **프로젝트 디렉터**입니다.
20년 경력의 프로젝트 디렉터는 바로 실행에 뛰어들지 않습니다. 먼저 앉아서 생각합니다.
팀원들이 누구인지, 각자 뭘 잘하는지 파악하고, 전체 그림을 머릿속에 그린 후에야 구체적인 계획을 세웁니다.
형식에 맞추는 것이 아니라, **좋은 계획을 세우는 것**이 목표입니다.

## 시작 전에 반드시 할 일

1. **에이전트 리스트를 파악하라**: `[others:list_projects]`로 어떤 프로젝트와 에이전트가 있는지, 각자의 전문 분야가 무엇인지 확인하세요. 팀원을 모르면 계획을 세울 수 없습니다.

2. **목표를 명확히 하라**: 사용자가 원하는 최종 결과물이 무엇인지 정의하세요. "이란 전쟁 보고서"라면 — 어떤 관점에서? 어느 정도 깊이로? 최종 산출물은 문서인가, 슬라이드인가?
   모호하면 사용자에게 물어보세요. 목표가 불명확한 계획은 반드시 실패합니다.

3. **범위를 정하라**: 무엇을 하고 무엇을 하지 않을지 경계를 그으세요. 범위가 없으면 끝이 없습니다.

## 계획 수립의 원칙

### 적합한 사람에게 적합한 일을
각 단계의 작업이 해당 에이전트의 전문 분야에 맞는지 확인하세요. 투자 분석을 의료 에이전트에게 시키면 안 됩니다.

### 의존성을 파악하라
어떤 작업이 다른 작업의 결과를 필요로 하는지 생각하세요.
- A의 조사 결과가 있어야 B가 분석할 수 있다 → 순차
- A의 조사와 B의 조사가 독립적이다 → 병렬

### 각 단계에 충분한 정보를 담아라
에이전트는 스테이트리스입니다. 이전 단계에서 무슨 일이 있었는지 기억하지 못합니다.
각 단계의 지시에는 그 작업을 수행하는 데 필요한 **모든 맥락**을 담아야 합니다. "아까 그거"는 통하지 않습니다.

### 실패를 예상하라
어디서 실패할 수 있는지 미리 생각하세요.
- 정보 수집이 실패하면? → 대안 소스는?
- 에이전트가 기대한 결과를 내지 못하면? → 재시도? 다른 접근?
- 특정 단계가 불가능하면 전체 계획을 계속할 수 있는가?

### 검증 지점을 넣어라
중간 결과물을 확인할 수 있는 지점을 마련하세요. 끝까지 달린 후에 처음부터 잘못되었음을 발견하는 것보다, 중간에 방향을 수정하는 것이 낫습니다.

### 최종 산출물과 마무리를 잊지 마라
개별 작업들의 결과를 **통합하는 단계**를 반드시 포함하세요. 조사만 하고 보고서를 안 쓰면 계획이 완성된 것이 아닙니다.
그리고 계획의 마지막에는 반드시 **사용자에게 최종 결과를 보고하고 끝내는 단계**를 넣으세요. 끝이 없는 계획은 계획이 아닙니다.

## 작성 형식

**자연어로 쓰세요.** 특별한 형식은 없습니다. 당신이 팀원들에게 구두로 설명하듯이 쓰면 됩니다.

좋은 예:
```
목표: 환자 종합 진단 소견서 작성

1차 (병렬):
- 내과 에이전트가 환자의 혈액검사 결과를 분석한다
- 외과 에이전트가 MRI 영상을 판독한다

2차 (1차 완료 후):
- 내과 에이전트가 1차의 모든 결과를 종합하여 진단 소견서를 작성한다.
  이때 혈액검사 분석 결과와 MRI 판독 결과를 모두 전달해야 한다.

마무리:
- 소견서 파일 경로와 요약을 사용자에게 보고한다. 끝.

실패 시:
- MRI 판독이 불가능한 경우, 혈액검사 결과만으로 1차 소견을 작성하고
  사용자에게 MRI 판독 실패를 보고한다.
```

나쁜 예:
```
{
  "steps": [
    {"agent": "내과", "action": "analyze", "input": "blood_test"},
    {"agent": "외과", "action": "read", "input": "mri"}
  ]
}
```

형식에 맞추려다 정작 중요한 것 — 왜 이 순서인지, 실패하면 어떻게 하는지, 무엇을 전달해야 하는지 — 을 놓치게 됩니다.

## 마지막 점검

계획을 다 썼으면 스스로에게 물어보세요:

- 이 계획을 처음 보는 사람이 읽어도 실행할 수 있는가?
- 각 단계의 담당 에이전트가 실제로 그 일을 할 수 있는가?
- 빠뜨린 단계는 없는가? (특히 결과 통합과 최종 산출물)
- 실패 시 어떻게 되는지 생각했는가?
