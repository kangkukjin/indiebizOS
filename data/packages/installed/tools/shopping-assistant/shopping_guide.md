# Shopping Assistant 패키지 가이드

## search_shopping 사용법

상품 검색 및 가격 비교를 위한 도구.

---

## 사이트별 특성

| 사이트 | 방식 | 강점 | 약점 |
|--------|------|------|------|
| `naver` | API 기반 | 정확한 가격, 빠른 응답, 카테고리 분류 | 전자제품 스펙 비교 약함 |
| `danawa` | 크롤링 | 전자제품/가전 스펙 비교에 강함, 최저가 추적 | 응답 느림, 비전자제품 약함 |
| `all` | 동시 검색 | 양쪽 결과 통합 비교 가능 | 응답 시간 증가 |

**추천**: 일반 상품은 `naver`, 전자제품/가전은 `danawa`, 가격 비교가 목적이면 `all`

---

## 반환 데이터

items 배열로 반환되며, 각 항목의 구조:

```json
{
  "name": "상품명",
  "price": "가격",
  "mall": "판매처",
  "link": "상품 링크",
  "image": "상품 이미지 URL",
  "category": "카테고리",
  "site": "검색 사이트 (naver/danawa)"
}
```

---

## 사용 시점

- 최저가 찾기
- 동일 상품의 판매처별 가격 비교
- 특정 카테고리 상품 가격 조사
- 구매 전 시세 파악

---

## display 파라미터

- 검색 결과 반환 개수 설정
- 최대 10개까지 지정 가능

---

## 검색 팁

- **구체적 모델명** 사용: "노트북" 보다 "맥북 프로 M3 14인치"
- **용량/사이즈 포함**: "냉장고" 보다 "냉장고 800L 양문형"
- **브랜드 명시**: 특정 브랜드를 원하면 반드시 포함
- **불필요한 수식어 제거**: "가성비 좋은", "인기 있는" 등은 검색 정확도를 떨어뜨림
