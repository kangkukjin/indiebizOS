# indiebizOS 해부도 (Anatomy)

> indiebizOS는 자동차처럼 복잡한 물건이 됐다. 그래서 이 문서는 *부품 목록*이 아니라, **하나의 생각에서 각 기관이 어떻게 자라났는지**를 보여준다.
> 처음 오는 사람은 이 문서만 위→아래로 읽으면 전체가 잡힌다. 각 절 끝의 **→ 링크**가 상세로 데려간다.
> 자동차를 처음 보는 사람에게 "이건 엔진, 이건 변속기"는 도움이 안 된다. 필요한 건 "이건 네가 *모는* 기계다, 여기가 운전대다, 그리고 *왜 이렇게 생겼는지*"이다.

---

## 0. 한 문장 — 무엇인가

**indiebizOS는 데려다주는 *자율주행차*가 아니라, 입는 *인지 외골격*이다.** 사용자의 PC에서 도는 개인 소유 프로그램(하네스)이 웨어러블 로봇의 *몸*이고, AI 모델은 그 *두뇌*(빌리는 것·교체됨)다. 진짜 자산은 **핏(fit)** — 한 사람의 의지가 응고된 것.

두 패러다임으로 가른다:
- **정답 패러다임** (업계 주류·AI 브라우저): 인간 명령 → 정답 하나 도출. 가능성을 *줄여* 시야를 좁힌다.
- **시야 패러다임** (indiebizOS): 인간 판단에 기반해 가능성을 *늘린다*. 힘은 AI가 대되, 어디로 뻗을지는 매 순간 사람이 정한다. 웨어러블 로봇처럼.

깊은 일(연구·집필·창업·"뭘 원하는지 알아내기")은 목적지가 루프 *안*에서 생성된다 — 그래서 한 방에 정답을 뽑는 게 아니라 사람과 함께 좁혀 간다.
→ 정본 telos: [vision.md](vision.md)

## 1. 어떻게 만지는가 — 런처의 3표면 (운전대·페달)

같은 IBL 신경계 위에, 사용자가 직접 모는 세 표면이 있다. 각각 {속도·표현력·주권} 중 둘을 갖고 하나를 내준다 (트릴레마).

| 표면 | 무엇 | 비용 | 큐레이션 |
|------|------|------|----------|
| **자율주행** | 의도 → 플래그십 AI가 다단계 처리 (아래 3장 인지 파이프라인) | 비쌈 | AI |
| **조종실** (구 수동/계기판) | 경량 모델이 자연어→IBL 번역 → dry-run 검수 → 실행. 시스템 상태·모델 기어·프레즌스·주행기록이 모인 **감독·개입의 주권 기관** | 거의 0 | 인간+언어 |
| **앱** | 결정화된 IBL 호출을 아이콘/GUI로 직접 조작 (부동산·도서검색 등) | 0 (코드 실행) | 결정화된 워크플로 |

**생애주기**: 새 일은 자율주행이 탐색 → IBL 흔적이 조종실 초안으로 → 검증된 고빈도 워크플로가 앱으로 결정화. *굳히는 건 증명된 것만.* 이 빈도-결정화가 "언제 시야를, 언제 한 번의 실행을"의 자동 규율이다.
→ 상세: [architecture.md](architecture.md) "사용자 표면 — 런처의 세 모드"

## 2. 표면 아래의 언어 — IBL (신경계)

세 표면이 무엇을 하든, 밑바닥에서는 전부 **하나의 문법**으로 번역된다:

```
[node:action]{params}      예) [sense:search_ddg]{query: "AI 뉴스"}
```

- **6개 노드** = 에이전트가 세계와 맺는 관계: `sense`(지각) · `self`(내 자원) · `limbs`(장치·신체) · `others`(소통) · `engines`(생성) · `table`(통화 변환 문법)
- API든 크롤링이든 안드로이드든 DB든 같은 문법으로 요청 — 프로토콜 차이는 드라이버가 감춘다.
- **왜 도구 더미가 아니라 *언어*인가**: 표현력은 어휘 크기가 아니라 *조합*(`>>` 순차 · `&` 병렬 · `??` 폴백)에서 나온다. 적게 고른 프리미티브 + 풍부한 조합 = 사실상 무한. 업계의 "자연어 SKILL.md" 대신 정합적 언어로 도구 폭증을 다스린다.
→ 명세·설계 철학: [ibl.md](ibl.md) (현재 노드/액션 수는 이 문서 상단이 정본)

## 3. 어떻게 생각하는가 — 인지 파이프라인 (뇌)

자율주행 표면의 내부. 인간의 인지 과정을 모델링한다: **분류(반사) → 의식(계획) → 실행 → 평가(성찰)**.

```
사용자 메시지
  ↓ [0] 연상   해마(과거 IBL 사례) + 심층메모리(사용자 사실) 1회 검색
  ↓ [1] 반사   해마 점수 ≥ 0.85 → 곧장 실행(무의식 스킵) / 미만 → 경량 AI가 EXECUTE·THINK 분류
  ↓ [2] 의식   (THINK만) 본격 AI가 "지금 무슨 문제를 풀어야 하나" 규정 + 달성 기준
  ↓ [3] 실행   IBL 엔진 → 도구 실행
  ↓ [4] 평가   경량 AI가 달성 기준 대비 검증, NOT_ACHIEVED면 재시도(최대 3라운드)
  ↓ [5] 증류   성공 경험을 해마·심층메모리에 저장 (다음엔 더 빠르게)
```

**모델 기어**(절약/균형/최대 레버, 조종실에 노출)가 4축(분류·평가·실행·의식)을 티어(경량/중급/본격)로 매핑 — 비용/속도를 한 손잡이로 변속. 의식 토글을 끄면 THINK 경로를 차단해 빠르고 싸게 돈다.
→ 상세: [architecture.md](architecture.md) "인지 파이프라인" · 뇌 구조 대응표는 [memory.md](memory.md) 부록

## 4. 어떻게 *너의 것*이 되는가 — 메모리 (기억·직관)

메모리는 곧 **속도·비용 최적화 장치이자 개인화의 엔진**이다. 인간 기억 분류에 대응하는 **7종**:

| 인간 기억 | indiebizOS |
|---|---|
| 의미(정적 지식) | 시스템 문서 (지금 읽는 이 문서들) |
| 작업(단기) | 대화 이력 |
| 일화(경험) | 에피소드 로그 |
| 절차(방법) | IBL 액션 + **해마**(자연어→IBL 코드 자기학습) |
| 관계(사용자 사실) | 심층메모리 (선호·결정·중요날짜 자동 흡수) |
| 자기상태(항상성) | World Pulse + Self-Check |
| 공간(포식) | **포식 기억**(냄새지도 — 어디에 무엇이 사는가) |

핵심은 세세한 사실이 아니라 **직관(감)** — 요리사의 손이 소금에서 알아서 멈추듯. **해마 점수**(과거에 해본 일인가)가 인지 라우팅(반사냐 의식이냐)을 가른다. 두 자기학습 루프(해마 증류 / 심층메모리 증류)가 *쓸수록* 시스템을 빠르고 개인적으로 만든다 — 이게 핏이 쌓이는 방식이다.
→ 상세: [memory.md](memory.md) (7종 지도 + 연상기억 심층)

## 5. 철학이 만져지는 곳 — 검색/포식 브라우저 (손끝)

시야 패러다임이 기능 하나로 응축된 곳. 구글 검색창처럼 보이지만 작동이 다르다:

1. 사용자가 **판(board)의 크기 N**을 정한다 (예: 10).
2. 키워드에 대해 AI가 정답 하나가 아니라 **후보 N개**를 판에 올린다 (아주 좋을 필요 없어 경량 모델로 충분).
3. 사용자가 마음에 안 드는 것을 **지운다**. 들어가 보기도 한다.
4. AI는 지운 수만큼 **새 후보로 채운다** — 남은 것에서 취향을 읽어. (옷가게 점원이 5벌 치우면 다른 5벌을 가져오듯.)
5. 반복하는 가운데 판의 분위기가 바뀌고, 사용자는 *새로운 것을 발견*한다. 판은 통째로 **판 도서관**에 저장돼 나중에 돌아올 수 있다.

정답 AI가 가능성을 줄인다면, 이 브라우저는 *빈칸을 채워야 하므로* 계속 가능성을 늘린다. 뒤에서는 AI 에이전트가 개인적 사실(사는 곳·직업·취미·보유 자산 등, 사용자가 스스로 준 것)을 참고해 후보를 개인화한다 — 구글과 달리 **내 PC의 파일**을 읽기에 가능한 일.
→ 개념 뿌리: [memory.md](memory.md) §7 공간 기억(포식) · [vision.md](vision.md) (융합 vs 자율)

## 6. 어떤 몸에 사는가 (체현)

- **소유**: 네 PC에서 도는 개인 소유 프로그램. 하드웨어도 정보도 네 통제 아래. (중앙 SaaS 아님.)
- **능력**: IBL 노드의 실제 구현체 = **도구 패키지**(폴더 기반 동적 로딩). 설치/철거가 코드+어휘를 원자적으로 넣고 뺀다. → [packages.md](packages.md)
- **두 자아**: 맥과 폰이 각각 독립 자아(폰=온디바이스 두뇌). 사용자 세계-데이터(연락처·일정·비즈니스)는 CRDT로 공유·동기화, 각 자아의 주관적 기억(대화·해마)은 사적. → [architecture.md](architecture.md) "외부 연동" · [communication.md](communication.md)
- **소통**: Gmail·Nostr(NIP-17 DM)·Telegram + 탈중앙 망 **IndieNet**. AI와, 이웃과, 에이전트끼리 소통. → [communication.md](communication.md)

## 닫으며 — 다시 한 문장으로

도구는 인간의 시야를 **가리기보다 넓혀야** 한다. 세상엔 정답이 있는 문제도 있지만, 중요한 문제일수록 사람의 질문 자체가 애매하고 자신도 뭘 원하는지·뭐가 가능한지 모른다. 그럴 때 필요한 건 정답을 뽑아 주는 자율주행이 아니라, 열린 마음으로 함께 좁혀 가되 결정에서 인간을 빼놓지 않는 외골격이다. 위의 모든 기관 — 3표면·IBL·인지·메모리·검색 브라우저 — 은 결국 **이 한 신념의 파생**이다. 나중에 지어낸 철학이 아니라, 이미 만들어진 몸을 서술한 것이다.
