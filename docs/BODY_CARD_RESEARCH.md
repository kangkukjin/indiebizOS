# 몸 독립 소통 연구 — 명함 수준 소통 (2026-07-22)

## 연구 질문

두 하드웨어에 설치된 indiebizOS 가 **서로의 어휘를 통째로 알지 않고** 소통할 수 있는가?
전제: 공통어휘(표준 코어 = 문법 + self/others/table + 통화)는 양쪽 다 가짐.
각자 자기 내용어 사전만 앎. 상대는 **명함**으로만 앎.

현재 상태(비판의 출발점): 폰↔맥은 같은 빌드의 공유 사전 RPC — 폰이 맥 어휘로 IBL
코드를 조립해 보내고 그 역도 성립. N대로 일반화하면 모두가 모두의 사전을 알아야
하는 O(N²). 낯선 두 indiebizOS 는 아무것도 공유하지 않으므로 이 길은 일반화 불가.

## 설계 (사람의 사회적 프로토콜 이식)

| 층 | 내용 | 비용 |
|---|---|---|
| 명함 | 자기 레지스트리에서 파생한 desc-프로젝션. 교환·캐시. | 정적, ~0 |
| 부탁 | 자연어 의도 → **받는 몸이 자기 사전으로 컴파일**·실행 → 통화 반환 | 컴파일 1회 |
| (미래) 진열 | 잦은 부탁의 결정화 = 계기 부여(포털 tool gate 재사용) | 0 토큰 |

명함 원칙:
- 손으로 쓰지 않음 — 레지스트리에서 파생(진실과 어긋날 수 없음). `dictionary_hash` 로 신선도.
- 표준 코어는 안 실음(공통어휘 전제). desc 만 싣고 **파라미터 시그니처는 안 실음**
  — 상대는 내 코드를 조립하지 않는다. params 는 내 컴파일러의 일.
- 각 몸은 자기가 실행 가능한 것만 소개(맥=phone_only 제외 / 폰=runnable_actions).
- 받은 명함은 **사전이 아니라 냄새** — 레지스트리에 절대 합치지 않고 프롬프트에
  요약(몸당 ~70토큰)만. "이 주제는 저 몸에 부탁할 수 있다"의 어포던스 인지.
- 교정은 받는 몸의 책임 — 실행 실패 시 에러+`_param_hint` 를 자기 번역기에 되먹여
  1회 자가교정. 부탁한 상대는 내 사전의 힌트를 이해할 수 없다.
- 어휘 밖 부탁 = 정직한 거절(CANNOT). 흉내내지 않는다.

## 구현 (2026-07-22, 전부 라이브 검증)

| 조각 | 파일 | 상태 |
|---|---|---|

| 명함 파생 | `backend/capability_card.py` (몸-인식, full/summary, hash) | ✅ |

## 실측 (연구 데이터)

- 명함 크기: 맥 full 105액션 ~6.0K 토큰 / summary ~460토큰 / **냄새 ~70토큰**.
  폰 full 57액션 ~4.1K 토큰. (비교: 상대 사전 전체 프롬프트 탑재 ≈ 42K 토큰 —
  명함은 7~600배 저렴, 몸당 O(1))
- 부탁 배터리(맥, 조종실 컴파일): 날씨/호텔+파이프/뉴스/부동산 4/4 성공(5~8초),
  어휘 밖("거실 불") 정직 거절. 파이프 합성(`[sense:stay] >> [table:sort] >> [table:take]`)
  = 내용어와 표준 코어를 받는 쪽이 스스로 조합 — **파이프 대수는 전선을 건너지 않는다**.
- 폰 컴파일러(Gemini 2.5-flash + 57액션 사전): 위치/촬영/알림+파이프 3/3 성공
  **0.9~1.8초**(작은 사전=작은 프롬프트=빠름), 어휘 밖 정직 거절.
- 번역 확률성: 같은 부탁이 city/region/location 으로 흔들림 → 실행 에러의
  `_param_hint` 자가교정 루프가 흡수(라이브에서 재시도 없이도 성공 사례 확인).

## 실기 검증 (2026-07-22 폰 재빌드 후 — A36 실기)

- 폰 명함 실물: `GET /nodes/card` = 57액션·hash 3f630d092591 (미리보기 예측과 일치).
- **컴파일러 선택 축 재설계**: 첫 실기에서 폰이 '지금 위치'를 CANNOT — 원인은 폰에도
  gemini_http 프로바이더(폰 AI config 3.x 트리오)가 있어 "프로바이더 없음" 감지가
  빗나가 조종실 경로(해마 없이 문법 교재만)를 탄 것. **능력 축 = 프로바이더가 아니라
  해마**: 용례 연상 있으면 조종실(축적 기반), 없으면 사전-동봉 컴파일(실물 사전이
  근거라 환각 불가). 몸마다 자기 최선의 컴파일러.
- 재설계 후 폰 실기: 위치/촬영/알림+파이프 3/3 정확(1~2초, compiler=gemini),
  세탁기=정직 거절. 실행 종단(알림 조회 파이프)=1.9초에 통화 반환.
- **상호 명함 자동 교환 실기**: 폰 등록 → 맥이 `data/peer_cards/폰-9f2b.json` 캐시,
  폰이 `맥.json` 캐시(양쪽 등록 훅). 맥 프롬프트에 폰 냄새(~70토큰) 라이브 주입.

## 사전 분리 1단계 (2026-07-22 — "자기가 쓸 수 있는 어휘만 설치"의 인지·학습층)

목표(사용자): 설치가 몸에 맞게 — 폰은 폰 어휘만, PC는 PC 어휘만. 남의 어휘를
학습하지 않는다. 해마도 자기 어휘만 배운다.

구현(1단계 = 인지·학습 표면. 실행층 물리 제거는 2단계):
- **`[others:ask]` 표준 동사 신설**(액션 157→158, others 코어 = 양 몸 공통):
  이웃 몸에 자연어 부탁. 발신=body_ask.ask_peer(레지스트리 별칭 해소→몸별 인증→
  상대 /nodes/ask), 수신=기존 부탁 경로. 반환에 compiled_ibl(상대의 이해) 투명.
  선행 조건인 이유: 맥 사전에서 폰 어휘를 빼려면 부탁할 길이 먼저 있어야 함.
- **카탈로그 소유-필터**(ibl_access.build_environment): 이 몸이 실행 못 하는
  어휘는 에이전트 카탈로그에서 제외 — 맥 153(=158−phone_only 4−hidden 1),
  폰은 runnable 만. 소유 판정 단일 소스 = capability_card._self_can_run.
- **해마 소유-필터**(ibl_usage_rag): 남의 어휘가 담긴 용례는 회상하지 않음
  (code_is_own). '지금 위치' → sense:here 회상 소멸(자기 어휘 stay 등 무손상).
  향후 재학습 코퍼스도 이 필터 기준으로 정리.
- 해마 시드 7건(manual_seed) + guides/body_ask.md + guide_db 등록.

검증: 맥→폰 `[others:ask]{to:"폰-9f2b"}` 실기(폰 gemini 컴파일 1.9초) /
폰→맥 역방향 실기(env 폴백+터널+세션 로그인 춤 → 맥 cockpit → 날씨 실데이터
6.1초) / 카탈로그·해마 필터 정확 / 맥 회귀 무손상.

2단계(미착수): 실행층 물리 분리 — 맥 레지스트리에서 phone_only 항목 제거 +
관문 자동 포워딩 은퇴(기존 계기·스케줄의 ask 전환 후), 폰 번들 물리 필터,
재학습 코퍼스 몸별 분리.
