# 몸 독립 소통 연구 — 명함 수준 소통 (2026-07-22)

## 연구 질문

두 하드웨어에 설치된 indiebizOS 가 **서로의 어휘를 통째로 알지 않고** 소통할 수 있는가?
전제: 공통어휘(표준 코어 = 문법 + self/others/table + 통화)는 양쪽 다 가짐.
각자 자기 내용어 사전만 앎. 상대는 **명함**으로만 앎.

현재 상태(비판의 출발점): 폰↔맥은 같은 빌드의 공유 사전 RPC — 폰이 맥 어휘로 IBL
코드를 조립해 보내고 그 역도 성립. N대로 일반화하면 모두가 모두의 사전을 알아야
하는 O(N²). 낯선 두 indiebizOS 는 아무것도 공유하지 않으므로 이 길은 일반화 불가.

## 설계 (사람의 사회적 프로토콜 이식)

| 층 | 내용 | 비용 |
|---|---|---|
| 명함 | 자기 레지스트리에서 파생한 desc-프로젝션. 교환·캐시. | 정적, ~0 |
| 부탁 | 자연어 의도 → **받는 몸이 자기 사전으로 컴파일**·실행 → 통화 반환 | 컴파일 1회 |
| (미래) 진열 | 잦은 부탁의 결정화 = 계기 부여(포털 tool gate 재사용) | 0 토큰 |

명함 원칙:
- 손으로 쓰지 않음 — 레지스트리에서 파생(진실과 어긋날 수 없음). `dictionary_hash` 로 신선도.
- 표준 코어는 안 실음(공통어휘 전제). desc 만 싣고 **파라미터 시그니처는 안 실음**
  — 상대는 내 코드를 조립하지 않는다. params 는 내 컴파일러의 일.
- 각 몸은 자기가 실행 가능한 것만 소개(맥=phone_only 제외 / 폰=runnable_actions).
- 받은 명함은 **사전이 아니라 냄새** — 레지스트리에 절대 합치지 않고 프롬프트에
  요약(몸당 ~70토큰)만. "이 주제는 저 몸에 부탁할 수 있다"의 어포던스 인지.
- 교정은 받는 몸의 책임 — 실행 실패 시 에러+`_param_hint` 를 자기 번역기에 되먹여
  1회 자가교정. 부탁한 상대는 내 사전의 힌트를 이해할 수 없다.
- 어휘 밖 부탁 = 정직한 거절(CANNOT). 흉내내지 않는다.

## 실측 (연구 데이터)

- 명함 크기: 맥 full 105액션 ~6.0K 토큰 / summary ~460토큰 / **냄새 ~70토큰**.
  폰 full 57액션 ~4.1K 토큰. (비교: 상대 사전 전체 프롬프트 탑재 ≈ 42K 토큰 —
  명함은 7~600배 저렴, 몸당 O(1))
- 부탁 배터리(맥, 조종실 컴파일): 날씨/호텔+파이프/뉴스/부동산 4/4 성공(5~8초),
  어휘 밖("거실 불") 정직 거절. 파이프 합성(`[sense:stay] >> [table:sort] >> [table:take]`)
  = 내용어와 표준 코어를 받는 쪽이 스스로 조합 — **파이프 대수는 전선을 건너지 않는다**.
- 폰 컴파일러(Gemini 2.5-flash + 57액션 사전): 위치/촬영/알림+파이프 3/3 성공
  **0.9~1.8초**(작은 사전=작은 프롬프트=빠름), 어휘 밖 정직 거절.
- 번역 확률성: 같은 부탁이 city/region/location 으로 흔들림 → 실행 에러의
  `_param_hint` 자가교정 루프가 흡수(라이브에서 재시도 없이도 성공 사례 확인).

## 사전 분리 1단계 (2026-07-22 — "자기가 쓸 수 있는 어휘만 설치"의 인지·학습층)

목표(사용자): 설치가 몸에 맞게 — 폰은 폰 어휘만, PC는 PC 어휘만. 남의 어휘를
학습하지 않는다. 해마도 자기 어휘만 배운다.

구현(1단계 = 인지·학습 표면. 실행층 물리 제거는 2단계):
- **`[others:ask]` 표준 동사 신설**(액션 157→158, others 코어 = 양 몸 공통):
  이웃 몸에 자연어 부탁. 발신=body_ask.ask_peer(레지스트리 별칭 해소→몸별 인증→
  상대 /nodes/ask), 수신=기존 부탁 경로. 반환에 compiled_ibl(상대의 이해) 투명.
  선행 조건인 이유: 맥 사전에서 폰 어휘를 빼려면 부탁할 길이 먼저 있어야 함.
- **카탈로그 소유-필터**(ibl_access.build_environment): 이 몸이 실행 못 하는
  어휘는 에이전트 카탈로그에서 제외 — 맥 153(=158−phone_only 4−hidden 1),
  폰은 runnable 만. 소유 판정 단일 소스 = capability_card._self_can_run.
- **해마 소유-필터**(ibl_usage_rag): 남의 어휘가 담긴 용례는 회상하지 않음
  (code_is_own). '지금 위치' → sense:here 회상 소멸(자기 어휘 stay 등 무손상).
  향후 재학습 코퍼스도 이 필터 기준으로 정리.
- 해마 시드 7건(manual_seed) + guides/body_ask.md + guide_db 등록.

검증: 맥→폰 `[others:ask]{to:"폰-9f2b"}` 실기(폰 gemini 컴파일 1.9초) /
폰→맥 역방향 실기(env 폴백+터널+세션 로그인 춤 → 맥 cockpit → 날씨 실데이터
6.1초) / 카탈로그·해마 필터 정확 / 맥 회귀 무손상.

2단계(미착수): 실행층 물리 분리 — 맥 레지스트리에서 phone_only 항목 제거 +
관문 자동 포워딩 은퇴(기존 계기·스케줄의 ask 전환 후), 폰 번들 물리 필터,
재학습 코퍼스 몸별 분리.

## here 지표어 전환 (2026-07-22 — 2단계의 첫 시범: 제거가 아니라 재분류)

`sense:here`의 phone_only 는 의미의 제약이 아니라 구현의 우연이었다("여기"는
지표어 — 어휘는 하나, 값은 몸이 정한다). anywhere 로 재분류하고 몸별 프로브:
- **폰** = fused GPS(±수십 m, 캐시 없음 — 움직이는 몸). 능력 감지 = jclass import.
- **데스크탑** = IP 지오(ipapi.co→ip-api lang=ko 폴백, 도시 수준) +
  **움직임-증거 캐시**: 네트워크 지문(로컬 IP, UDP connect 트릭 — 무권한·stdlib·
  윈도우 호환)이 불변이면 캐시 반환(재측정 0원). refresh:true 강제 재측정.
  정책(증거 없으면 캐시)=보편, 프로브(지문·측정법)=몸의 어휘.
- **통화 정직성**: {source: gps|ip, accuracy_m, measured_at, cached} — 실측에서
  맥 IP 지오가 "성남시"(ISP 위치)를 내놓음 = 정직성 필드가 필요한 이유의 실증.
  note 가 정밀 요구 시 [others:ask] 경로를 안내(두-층 분리: here=자기 몸 /
  사용자 위치=폰에 명시적 부탁).
- 효과: 맥 카탈로그 153→154(here 복귀), 명함 105→106(hash 변경→다음 등록 때
  자동 재교환), 맥→폰 here 자동 포워딩 소멸(인구조사에서 확인 가능).
- 캐시=data/here_cache.json(gitignore). 폰 재빌드 완료(GPS 경로 회귀 확인 —
  실측 오송읍 ±20.6m, source/measured_at 통일).

### see/listen 도 지표어 (같은 날 후속 — 사용자 교정)

"see/listen 은 phone_only 유지가 정직"이라던 위 단정은 **틀렸다** — 이 맥미니에
카메라가 없는 것은 우연이지 어휘의 법칙이 아니다(카메라·마이크 달린 PC 는 흔하다).
사용자 교정: **각자 자기 몸의 하드웨어를 쓰고, 없으면 작동불능** — 어휘를 특정
몸에 묶는 것이 위험한 것.
- phone_only 해제 + 데스크탑 프로브 `desktop_av.py`: ffmpeg 장치 열거
  (avfoundation/dshow/v4l2·alsa — 라디오·유튜브의 ffmpeg 공급 체계 재사용),
  see=웹캠 1프레임(★화면캡처 장치는 카메라 아님=제외), listen=마이크 녹음(m4a)
  + transcribe=Gemini 오디오 받아쓰기(키 없으면 녹음만 정직 반환).
- **하드웨어 없음 = no_hardware 정직 거절** + [others:ask] 힌트 — 맥미니 실측:
  see→"이 몸에 카메라가 없습니다"(카메라 0 정상 사례), listen→유일 오디오
  장치(Zoom 가상)로 녹음·STT 종단('무음→음'). 폰 회귀: Camera2 실촬영 4080×3060.
- 원칙 확정: 지표어 판정은 "이 몸에 프로브를 지을 수 있는가"가 아니라
  **"이 감각이 몸-보편 개념인가"** — 프로브 유무는 런타임의 정직성(no_hardware)이
  담당하고, 어휘는 모든 몸의 것.

## 2단계 착수: 사전 물리 분리 (2026-07-22 — "설치 = 자기 사전만")

사용자 정식화: **배포물(레포·설치파일) = 전체 사전집 / 설치(각 몸의 런타임) =
자기 어휘만.** 폰은 PC 전용 어휘를 알 필요 없고, PC 는 폰 전용 어휘를 알 필요 없다.
(태생=맥 레포 파생은 자연스러움 — PC 연결 폰에 설치하는 게 자연 경로. 문제는 내용물.)

- **로더 설치 필터**(ibl_engine._prune_foreign_vocabulary): 레지스트리 로드 시
  이 몸의 어휘만 남긴다 — "로드가 곧 설치". PC=phone_only 제외(맥 실측: limbs:phone
  실행 시 "액션 없음" 정직 에러), 폰=runnable_actions 만(limbs:desktop/browser 등
  제거, 정당 소유 124개 유지). 코어 어휘는 양 몸 공통이라 @alias 크로스바디
  포워딩(신문 @hub 읽기 등)은 그대로 산다.
- **번들 물리 필터**(build_body_bundle.derive_nodes_registry → android.nodes.yaml,
  gradle 이 zip 의 ibl_nodes.yaml 로 rename): 폰 번들에 애초에 폰 사전(124 어휘,
  182KB)만 싣는다 — 전체 사전집(158 어휘 263KB) 미탑재. 파생본은 빌드마다 재생성
  (gitignore — 정본은 사전집+매니페스트).
- 절단 안전 근거(사전 조사): 인구조사 46건 전부 sense:here(이미 지표어 전환으로
  로컬화 — 자동 포워딩 수요 실측 소멸) / limbs:phone 의 맥-측 실행자 0(전부 폰-측
  게이트: 계기 share 버튼·창 없는 몸의 알림 짝).
- 온디바이스 검증: 번들 사전=182,166바이트 정확 / 폰 명함 57 불변("같은 사전=
  같은 명함") / 폰→맥 ask·자기 어휘 실행 정상.

## 몸 신뢰 원장 — 이웃 통합 (2026-07-22 — "특별함은 배관이 아니라 레벨")

사용자 원칙: 폰↔맥의 절대신뢰는 **관계(파라미터)** 지 메커니즘이면 안 된다.
낯선 두 indiebizOS 가 만나 어떤 절차로 신뢰를 허용받으면 소통하듯, 내 몸들도
같은 절차 — 그래야 폰↔맥이 낯선 소통의 모델이 된다.

- **전송/신뢰 분리**: 전송층(토큰·세션)="너는 너다"까지 / "무엇까지 허용"=원장.
- **원장 = 이웃 명부**(body_trust.py): "몸도 이웃이다" — neighbors.info_level
  (숫자만, 의미 라벨 금지) + contacts(contact_type='body', value=device_id.
  창고주소 선례: 몸의 신원도 접점의 한 종류). business.db 라 CRDT 로 두 몸이
  같은 명부 공유.
- **부여식**: 인증된 등록(=소유주 프로비저닝의 증명) 시 자동 레벨 4 — 맥은
  등록해 온 몸을, 폰은 명함 신원(identity.device_id)으로 맥을. 물리 점유+설치
  행위가 곧 부여식일 뿐, 절차는 이웃과 동일. 낯선 몸의 만남·부여=소유주 절차(미래).
- **ask 신뢰 게이트**: 발신이 신원 동봉 → 수신이 레벨 판정. 낯선 몸="신뢰 부여식
  전에는 부탁을 받지 않습니다"(정직 거절), 신원 없음=소유주 표면(전송층 인증 통과).
- **특권 선언**(PRIVILEGE_LEVELS): ★2026-07-22 사용자 확정으로 재정위 — 특권을
  레벨로 *집행*(실게이트化)하자던 안은 목표로서 폐기. 이 선언은 이제 **철거 대상
  배관의 목록**이다(감독이 아니라 소멸 — docs/NO_PRIVILEGED_RAILS_HANDOFF.md).
  몸 간 문법은 이웃 문법 하나(명함·ask·신뢰레벨·창고)로 통일.
- 실기: 상호 부여식(맥 명부: 폰-9f2b→4 / 폰 명부: 맥→4) · 신원 동봉 ask 양방향
  게이트 통과 · 낯선 신원 양쪽 모두 정직 거절.
- ✅해소(2026-07-22, 특권 소멸 1단계): ask 의 푸시큐 폴백 → **우편함**(ask_mailbox —
  NIP-17 DM 위 ask 봉투, HTTP 직결 불가 시 자동 폴백) / 전송 신원 npub 화 →
  명함 identity.npub + 부여식 npub 접점("npub:<hex>") — 우편함 발신자는 seal
  서명으로 증명된다(자기보고 device_id 의 승격). payload 동봉(리터럴 데이터
  "$payload.<키>" 치환)과 수신측 소유-가드(남의 어휘 컴파일 거절)도 이때 합류.
