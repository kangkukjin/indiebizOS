# 파일 포식(Information Foraging) — 연구 종합 보고서

*작성: 2026-06-20 / 성격: **연구 중간 종합** (완성 설계 아님). 후속 연구 전제.*

> 이 문서는 "AI가 다양한 대상(대표=하드디스크)에서 정보를 *찾아내는* 능력"에 대한 근본 고민의 1차 종합이다. 개념 모델 → 실증 트레이스 3건 → 선행연구 분야별 keep/absorb → 어떤 지능에도 남는 척추 → 열린 연구 질문 순. **섣불리 완성품을 만들 주제가 아니라는 판단**(사용자) 아래, "무엇이 알려져 있고 / 무엇이 우리 고유로 열려 있나"를 못박는 데 목적이 있다.

## 0. 관통하는 한 문장 (이 보고서의 결론을 먼저)

> **옛 분야들(포렌식·eDiscovery·탐색이론)은 *규율은 있는데 지능이 없었다.* 현대 에이전트(agentic retrieval)는 *지능은 있는데 규율을 버렸다.* 우리 일 = 옛 규율을 새 지능 위에 다시 얹는 것 — 그것도 벤더가 못 닿는 사적·자서전적·협업 맥락에서.**

이 한 줄이 아래 모든 분야 조사를 묶는다. 각 분야는 "look-and-understand 지능의 부재"를 보상하려고 기계장치를 지었고, LLM forager에겐 그 보상장치 대부분이 **흡수될 죽은 무게**다. 그러나 지능과 무관하게 **남는 세 가지**(§5)가 있고, 거기에 우리 고유의 네 가지 델타(§6)를 더하는 것이 과제다.

## 0.5 설계 철학 — forager는 AI다, 무엇을 *더할* 것인가 (★이 절이 방향을 지배한다)

선행연구(§4)는 전부 *AI 없는* 시대의 것 — forager가 인간/학습분류기/알고리즘이었다. 그들의 기계장치 대부분은 *지능의 부재를 보상*하려는 비계였다. **우리 forager는 강한 AI이고, 그 베이스라인은 시간이 갈수록 공짜로 좋아진다.** 따라서 문제는 "forager를 어떻게 짓나"가 아니라 **"이미 검색하는 AI가 결여한 것이 무엇이고, 우리가 무엇을 *더하나*"**이다.

- **베이스라인 = AI 혼자 + 좋은 도구.** AI는 이미 싼-것-먼저 포식하고, 질의를 재구성하고, 내용 읽어 판단하고, 정지를 자연어로 추론한다. 우리 세 실증 트레이스(§2)가 확인 — **knee 공식도 elusion 검정도 안 쓰고 *판단*했다.** 막힌 곳은 지능 부족이 아니라 *몸 갭*과 *지속성 부재*였다.
- **침식 테스트**: 어떤 부품이든 *모델이 좋아지면 풀리는가*를 물어라. 풀리면 → 짓지 마라(흡수될 죽은 무게). 안 풀리고 지능과 무관하면 → 더하라.
- **짓지 말 것(흡수)**: 냄새채점·가지치기 판단·정지 공식·세션 내 믿음 부기·추정 apparatus(Quant 보정확률 의존) — AI가 하거나 곧 흡수. *다단 모델 티어*도 침식 쪽(모델이 싸질수록 이유 약화) → 얇게, 단순화 예상하고.
- **★더할 것(비침식·지능무관)**: ① **몸**(싼 탐침: 해시·메타·EXIF·트리·카빙 = 접근 능력) ② **세션 넘는 지속 기억**(냄새지도 — AI는 구조적으로 stateless) ③ **사적 prior**(일화기억·정리습관 주입 — 가중치에 없는 당신 과거) ④ **provenance + 음성 검증 *행위***(출처 구속 + 버린더미 표본 측정 — AI는 환각·판단만).
- **선행연구의 용도 전환**: 베낄 청사진이 아니라, *어떤 문제가 AI로도 안 사라지나*의 지도. knee/elusion의 교훈 = "공식 이식"이 아니라 **"정지는 AI가 판단, 음성 단언을 *측정*할 얇은 도구만 더함."**

**함의**: §3 코드 하네스는 *루프 제어*가 아니라 **지속성+탐침+provenance로 최소화**된다. §8 정지 규칙은 공식 apparatus가 아니라 AI 판단+얇은 측정으로 격하. 다음 실험(§9)은 기계장치 선(先)구축이 아니라 **AI-혼자 베이스라인의 실제 실패점 찾기**다.

## 0.6 개인 서치 forager — 연구의 *진짜 범위* (★범위 격상)

§0.5가 "forager는 AI다"라면, 이 절은 "**그럼 우리가 누적하는 것은 *나라는 개인이 서치하는 방식*이다**"로 범위를 넓힌다.

- **두 공간의 진짜 차이(어떻게 뒤지나의 출처)**: 웹의 "어떻게 뒤지나"는 공동체가 이미 누적(구글·군중·모델 가중치) → 모델은 *웹 서치 베테랑으로 도착*. 내 디스크의 "어떻게 뒤지나"는 *미리 존재하지 않고 가중치에도 없으며 나만의 것* → 모델은 *매번 첫 방문자*. **사적 공간에선 "어떻게 뒤지는지 배우는 일 = 뒤지는 일"이고, 그 누적 기제가 척추다.** → F2를 '추가 하나'에서 *중심 문제*로 격상.
- **통합(대조 아님)**: 개인 서치 경험은 *어떤 공간 위에도 얹히는 층*. 디스크=공동체 기질≈0이라 개인층=전부 / 웹=공동체 기질 거대라 개인층=델타. **디스크가 만들기를 강요하고, 웹이 덕을 본다.** 같은 코퍼스가 디스크·웹·코드·책을 가로지른다.
- **누적 대상 = "내가 서치하는 방식"의 성분**: ①위치 prior(내 답이 사는 곳·신뢰 출처) ②반복 needs ③개인 관련성 기준 ④어휘/의도 매핑 ⑤이력 join ⑥죽은 가지. (= §0.5 "(b) 이-공간 지식"을 '이 디스크'→'이 개인·모든 몸'으로 일반화.)
- **왜 웹에서도 더 효율적일 수 있나**: 보편 서치=인구 중앙값 최적화. 개인 prior로 가지치기 가속·무관 검색 건너뜀·이력 조인 → 같은 웹도 *나를 통해 보면* 더 싸고 정확. 결과 재랭킹이 아니라 **서치 *과정*의 개인화.**
- **단서**: 웹에선 대체 아닌 *합성*(공동체 베이스라인 강력). **필터버블/세렌디피티 위험** — 개인 누적이 감옥 되면 안 됨(surface/explore 안전판).
- **연구 의미**: 분야는 *보편 forager* + 얕은 결과 개인화(PIM)에 머묾 — **누적되는 *개인 forager 정책*은 빈자리.** 데이터지 지능 아님(복리·비침식) + 벤더 미접근 + 모든 몸 일반화 → *전략적 중심*. [[architecture_avoid_vendor_layer]]의 가장 순수형: 구글+군중이 *공개적으로* 축적한 것을 *사적·사용자별로* 다시 짓는 일.

## 1. 문제 정의 — 왜 "검색"이 아니라 "포식"인가

### 1.1 하드디스크 = 세 번째 기억 (반구조·발견된 기억)
| | 하네스 메모리 | 하드디스크 |
|---|---|---|
| 스키마 | 엄밀 | 느슨(폴더·이름·날짜 = *서술적* 관습) |
| 쓴 주체 | AI 자신 | **사용자가 수년에 걸쳐** (남의 정리 논리를 읽어야) |
| 꺼내기 | 정확 조회 | **점진적 좁히기**(가설→가지치기) |

저장이 아니라 **검색**이 문제. 가치 밀도는 높은데 "거기 있는데 못 찾는" 상태.

### 1.2 보편성 — 이건 디스크만의 문제가 아니다
"최신 맥미니 정보 찾기"(웹)도 같은 구조 문제다. 공통 골격 = **큰 공간 + 약간의 질서 + 단서 + 비용 비대칭 + 이미-있는-걸-찾기**. 실종 아이 / 집안 운동화 / 하드디스크 / 책 속 한 정보 / 코드베이스가 전부 같다. → **포식 정책은 몸 독립(상부구조), "싼 탐침"과 "내용 열기"만 몸별(하부구조).** 하나의 forager, 여러 몸.

### 1.3 핵심 비대칭과 "냄새"
- **비용 비대칭**: 메타데이터는 싸다(OS가 이미 색인) / 내용은 비싸다(열어야). → *메타 선분류(triage) → 좁혀진 뒤에만 내용 열기*.
- **베이즈 업데이트는 *파일*이 아니라 *가지*에서**: 사람은 file-by-file이 아니라 folder-by-folder로 내려가며, 한 번의 비-진입이 하위 수천 파일을 통째로 가지치기한다. O(파일)이 O(깊이)로 붕괴. RAG의 평탄화(top-k)는 이 트리를 뭉갠다 — 그래서 큰 반구조 공간에서 진다.

### 1.4 두 개의 prior (합성 가능)
| | 구조 prior (세계모델) | 개인 prior (자서전) |
|---|---|---|
| 출처 | 사전학습(공짜) | world_pulse.db 일화기억·사용자 프로파일 |
| 적용 | **낯선 디스크 포함 어디나** | **내 디스크에서 추가 보너스** |
| 예 | "세금=연도 폴더" | "작년 그 시점 ⋈ mtime" |

★중요 통찰: **LLM은 구조 prior를 *이미* 갖고 있다**(인류 텍스트로 사전학습 — node_modules·Documents·연도폴더 관습을 안다). 그러니 낯선 디스크에서 prior-less가 아니다. 빠진 건 *지식*이 아니라 **인터페이스**(계층 행동·외부 믿음 상태·한계수익 정지).

## 3. 도출된 forager 아키텍처 (가설 수준, 미확정)

실증과 추론으로 수렴한 모양. **확정 설계 아님** — §7 열린 질문으로 흔들릴 수 있음.

### 3.1 finder = *프로그램*이지 "루프 도는 LLM"이 아니다
믿음 상태를 단일 LLM 컨텍스트에 들리면 진단된 실패(어디 봤나 망각·vibe 정지)가 돌아온다. → **코드 하네스가 믿음 상태를 들고**, 그 안에서 모델을 두 역할로 호출.

```
노드에서: 자식들의 싼 프로파일 얻기            ← B1 디렉토리 프로파일 = 루프의 원자
        → 두 prior로 자식별 냄새 점수 + 확신
        → 따뜻한 가지 내려가기 / 차가운 건 쳐내고 "봤음" 기록 (저확신은 보류=defer)
        → 믿음 갱신
        → 한계수익 떨어지면 정지/넓히기/물어보기
```

### 3.2 4단 모델 티어 (비용·내구성)
| 역할 | 누가 | 빈도 | 비고 |
|---|---|---|---|
| 순서·명백 거름 | **로컬 임베더**(~공짜·ms) | 매우 다수 | 유사도≠냄새. **결정권 없음**(가지치기는 committal). 순서매겨 *이른 정지* + flash-lite 호출 줄임 |
| 냄새 채점 | **경량 모델**(flash-lite) | 매 노드(=한 호출, 형제 배치) | 분류기 동형 → [[architecture_meta_routing_durability]] 상속. 호출수=방문 노드수(파일 아님) |
| 재계획 | **본격 모델**(sonnet/opus) | 드물게(전환점) | "여기까지 믿음으로 — 새 가설/전환/정지/물어보기". 외부 믿음을 *읽을* 뿐 들지 않음 |
| 최종 재구성 | **본격 모델** | 1회(끝) | 흩어진 증거 조립 |

핵심: 임베더의 가치는 per-call 비용보다 **정지 경제**(따뜻한 가지부터 → 차가운 가지 보기 전 정지)에 있다. "공짜"는 coarse 층에선 맞고, **결정(특히 commit하는 prune)은 경량모델 이상**.

### 3.3 finder 계약 (위임 능력)
- **입력** = `{목표, 범위, 예산(깊이), 힌트/prior(개인·일화 주입), 완료기준(한 건?/전부?/답하기 충분?)}`
- **출력** = `{결과, 커버리지·믿음 잔여(어디 봤고 *무엇을 못 찾았나*), 확신, 멈춘 이유, (또는) 되묻는 질문}`

출력의 "믿음 잔여"가 결정적 — **"거기 없음" vs "덜 봤음"을 구별**해야 호출자가 다음(넓히기/묻기/진행)을 정한다(여행 트레이스의 그 문제). `완료기준` 한 필드가 seek와 gather를 가른다.

## 4. 선행연구 — 분야별 keep / absorb

렌즈: 각 분야의 **forager가 무엇이었나** → "일반지능 부재를 보상한 기계장치"(=**흡수**) vs "지능이 있어도 남는 문제"(=**지킬 것**).

### 4.1 디지털 포렌식 triage — *문자 그대로 우리 문제*
("내가 안 만든 낯선 디스크에서 관련 증거 찾기." 단일 디스크 = 30만 파일 / 312만 타임라인 이벤트.)

- **forager였던 것**: 인간 수사관 + 해시DB(NSRL) + 과거사건 라벨 학습 분류기(SVM/결정트리) + 손튜닝 정규식. 자백: *"컴퓨터가 정보를 이해할 리 없으니 출력 문자열만 긁어 인간이 의미를 만든다"*(bulk_extractor).
- **흡수(죽은 무게)**: 검색히트 SVM 랭커, 메타→ML 관련성 분류기(과거 라벨 의존이 치명 — 첫 낯선 디스크엔 무력), 손튜닝 정규식 룰북, 사용자 5유형 휴리스틱.
- **지킬 것**: ① **해시 선필터**(NSRL = "이건 OS 기본, 토큰 쓰지 마" = 가장 싼 prior). ② **수퍼-타임라인**($MFT·레지스트리·로그를 한 줄로 = LLM이 추론할 *구조화 기질*, 날바이트 아님). ③ **포렌식 패스 provenance**(바이트오프셋+디코드체인 재현). ④ **타임라인 상위추상**(수백만 이벤트 요약 = LLM 네이티브 일).
- **★측정된 보상곡선**: 메타데이터만으로 **상위 10%에서 recall 75~89%**(Du et al). 우리 "싼 메타 펀넬"의 payoff가 이미 숫자로 존재.
- **대조 깃발**: "메타데이터만으로 충분"이란 주장은 *내용읽기가 비쌌던 제약의 미화*. Du 본인이 future work로 "파일 내용을 입력특징으로, NLP 적용"을 꼽음 = **그들은 LLM을 원했으나 없었다.**

### 4.2 eDiscovery TAR / CAL — *이미 작동하는 우리 루프의 상용 인스턴스*
(거대 문서집합에서 관련 문서를 싼 분류기+인간 루프로. 가장 정독 가치.)

- **forager였던 것**: 선형 분류기(로지스틱/SVM, tf-idf·byte 4-gram — 신경망 0) + 인간 검토자.
- **CAL 루프(AutoTAR, 2015)**: seed 1건(또는 NL 토픽서 합성) → 매 회 임시 무작위 음성 100개 주입 → SVM 학습 → **최고확신(uncertainty 아님) 상위 B개** 인간 코딩 → 학습셋 추가 → B←B+⌈B/10⌉ → 반복. TAR 1.0(seed+train/freeze 분리)→2.0(seed 없이 연속학습, control set 폐지)이 구조적 단절.
- **흡수**: seed set·수천 인간 라벨(LLM은 *NL 목표서 콜드스타트*), 가짜 음성 주입, tf-idf/스테밍/불용어 공학, 배치 재학습, uncertainty-vs-relevance 논쟁(약한 선형경계의 부산물).
- **지킬 것**: ① **최고확신-우선 랭킹** — 저빈도(personal disk 기본=대부분 무관)에선 relevance sampling 압승. ② **정지/recall 수학 전부**(랭커 IQ 독립). ③ **인간이 '관련'을 정박**. ④ **defensibility**(안 본 것에 대한 *통계 주장*).
- **★킬러 임포트 둘**:
  - **(a) Knee(무릎) 정지** (Cormack-Grossman): 누적수확 vs 노력 곡선에서 한계수익 붕괴점. 기울기비 ρ(s)≥156−min(Rel(s),150) AND s≥1000, 목표 recall 0.7. = 우리 "한계수익 정지"의 *집행 가능 공식*.
  - **(b) Elusion test**: 버린 더미를 무작위 표본추출→코딩→elusion(버린 것 중 관련 비율) 추정. 낮으면 "깨끗한 폐기" → 정지. = 우리 **"거기 없음 vs 덜 봄"을 신뢰 가능하게** 만드는 바로 그 도구. (여행 트레이스의 잔여 문제.)
- **prevalence 경제학**: 희귀할수록 recall 경계에 큰 표본 필요(2% 빈도면 383표본의 75%recall CI가 35~97%로 붕괴) → 저빈도에선 직접 recall보다 elusion test. *랭커와 무관한 수학.*
- **법적 정박**: Da Silva Moore(2012, Peck) — TAR은 *비례적 비용*에 합당하면 수용, "magic Easy-Button 아님". recall 70~80%가 통상 수용대.

### 4.3 고전 탐색 이론 — *빌려올 수학* (전부 인간/알고리즘 서술, 에이전트 아님)

셋은 경쟁이 아니라 **한 컨트롤러의 세 층**:

| | 정보 포식(Pirolli/Card) | 베이즈 탐색(美 해군) | Berrypicking(Bates 1989) |
|---|---|---|---|
| 지배 | **언제 멈출까**(patch exit) | **어디 볼까**(cell 선택) | **질의가 어떻게 진화** |
| 내구 수학 | MVT: 한계수익 `g'(t) ≤ R`이면 떠나라 | argmax `p·a/c` + 음성갱신 `p'=p(1−q)/(1−pq)` | 재계획 루프 + 6 탐색수 |
| 외부 상태 | 세션 수익률 R | 사후확률 지도 p | 진화하는 정보요구 |

- **정보 포식**: *냄새*=근접단서로 distal 가치를 추정(Bayesian relevance). Holling 원반식 `R=λg/(1+λt_W)`. **MVT 정지**(인간은 직감·satisfice, LLM은 *R을 literally 들고 계산* → 서술이론을 집행 컨트롤러로 승격). **diet 모델**: 고가치 풍부하면 *식단 좁힘*(쓰레기 무시 가능).
- **베이즈 탐색**: `p`(여기 있을 확률)×`q`(있다면 탐지). **음성갱신** = "뒤졌는데 없음"이 그 칸 낮추고 나머지 올림. argmax `p·a/c`로 다음 칸. 실사례 USS Scorpion·AF447(이전 수색 실패 후 베이즈로 회수). 고전은 *사람이 정한 격자+탐지함수* 필요 → LLM은 가설공간 즉석 생성(격자 가정 absorb).
- **Berrypicking**: 한 방 질의 거부, 매 berry가 다음 질의·*요구 자체*를 고쳐씀 = 재계획 루프 정당화. 6수 툴박스(footnote/citation/journal-run/area-scan/subject/author chasing). 인간은 인지부하로 berry 망각·직렬 → LLM은 진화요구 명시 보유 + 6수 병렬.

**통합 통찰**: 셋 다 인간의 *satisfice/직감*을 **LLM이 literally 들고 계산할 외부 수치**(R / 사후확률 p / 진화요구)로 대체. LLM이 제거하는 공통 한계 = **기억 + 병렬 + 음성결과 추적**.

### 4.4 AI 시대 에이전트 — *대조군: 지능은 있는데 규율을 버렸다*

현대 시스템(LlamaIndex fs-explorer, Azure agentic retrieval, Agentless, deep-research, grep-backbone)이 **새로 가능케 한 것**: NL 질의 재구성·세계모델 관습 prior·내용 읽어 판단·계층 localization·동적 도구선택.

**그런데 빠진 것 = 정확히 옛 분야가 갖고 우리가 더할 것:**

| 갭 | 현대 시스템 상태 |
|---|---|
| 디렉토리 **구조를 prior로** | 거의 미해결 (fs-explorer는 *평탄* metadata.jsonl로 확장) |
| 세션 넘는 **믿음/냄새 지도** | 미해결 ("belief drift" 인정, 메모리 연구 별개) |
| 명시적 **음성정보 추적** | 미해결 (기본 에이전트는 막다른 길 재탐색) |
| 비용비대칭 = **싼/비싼 모델 티어** | 부분·거침 (effort tier만, 원리적 2모델 forager 없음) |
| **하나의 forager, 여러 몸** | 미해결 (디스크/코드/웹 각자 별도 스택) |

**★규모 증거(LlamaIndex 2026)**: 5문서는 fs-도구 정확도 압승(correctness 8.4 vs RAG 6.4, relevance 9.6 vs 8.0), 100+는 RAG 역전. *그런데 둘 다 평탄 비교, 아무도 계층 가지치기를 안 테스트.* 계층 coarse-to-fine은 **코드(Agentless 27.3% SWE-bench·$0.34/이슈)·영상·일반검색(ReTreever)에선 입증된 규모해법**이지만 **일반 파일시스템엔 미검증** = 우리 베팅이 열린 자리.

## 5. 어떤 지능에도 살아남는 세 가지 (= 반드시 지킬 척추)

1. **비용 펀넬** (해시필터→메타/타임라인 랭크→상위만 읽기). 규모·비용은 *물리*지 지능 아님. 포렌식이 payoff까지 측정.
2. **정지·음성결과 수학** (MVT·베이즈 음성갱신·elusion test·knee). "안 본 게 얼마나 되나"는 *미관측 잔여*의 성질 → 랭커 IQ 독립. **가장 과소평가된 수입품.**
3. **Provenance/감사가능성** (포렌식 패스·elusion 증거). LLM이 환각 위험으로 *더 악화*시키는 문제 — 단언마다 "어디·어떤 변환·확신"을 붙여야.

**흡수할 것(공통)**: 학습된 대리 분류기·seed set·특징공학·키워드 분류체계·고정 격자 — 전부 "이해 못 함"의 보상물.

## 6. 우리 고유로 남는 델타 (벤더·선행연구 모두 비어 있음)

선행 장기들은 기성품(재발명 말 것). 합쳐도 빠지는 넷:
1. **구조=자서전적 prior** (포렌식=적의 디스크, TAR=평탄, agentic=구조 뭉갬). "내 디스크=내 과거, 일화기억 조인"은 누구에게도 없음.
2. **포식 간 지속되는 냄새 지도** (어느 시스템도 세션 넘어 믿음 누적 안 함).
3. **싼모델/코드믿음 티어를 *내구성 전략*으로** ([[architecture_meta_routing_durability]]).
4. **하나의 forager, 여러 몸** (디스크→책→코드→웹, 탐침 계약).

이 다섯(구조prior+지속믿음+음성추적+티어모델+다몸)의 조합은 **어디에도 조립돼 있지 않다.** + 사적·협업 맥락([[project_augmentation_over_autonomy]]) = [[architecture_avoid_vendor_layer]]와 같은 자리.

## 7. 열린 연구 질문 (이게 왜 "완성품"이 아니라 "연구"인가)

1. **냄새 채점기 calibration** (유일한 *경험적* 미지수): 임베더/경량모델이 관련 가지를 짚는가 + **애매한 걸 저확신으로 정직히** 내는가(과신=committal prune 재앙). 실측 필요.
2. **계층 가지치기가 규모를 깨는가**: Agentless가 코드에서 입증, *일반 FS엔 미검증*. 우리 기여 후보지만 검증 숙제.
3. **두 prior 합성**: 구조prior(공짜)+개인prior(일화조인) 점수를 어떻게 한 냄새로? 가중·게이팅?
4. **믿음 상태 표현·지속**: 사후확률 지도를 어디에(코드 구조? 파생 캐시?), 세션 넘어 어떻게 무효화하며 누적?
5. **knee/elusion의 디스크 이식**: 법조 수치(ρ(s), recall 0.7)가 개인 디스크 gather에 그대로 맞나? 재보정?
6. **forager 거처**: 코그니티브 층 행동 vs IBL 동사([?:seek]) vs 둘 다? IBL이면 어휘·통화 정합.
7. **탐침 계약의 기질 부재 대응**: 여행 트레이스가 보인 "싼 메타(Spotlight/EXIF) 자체가 없을 수 있음" → 계약에 기질 가용성 점검+폴백을 1급으로.
8. **provenance 통화**: 단언마다 붙일 "어디·변환·확신"을 어떤 통화/스키마로?

## 8. 구체 쇼핑리스트 (설계 단계에서 바로 검토)

해시 선필터(NSRL식 "무시 prior") · 타임라인을 추론 기질로 · relevance-우선 랭킹 · seed 없는 콜드스타트 · **knee 정지 + elusion test**(음성결과 신뢰화) · MVT 컨트롤러(R 추적) · 베이즈 음성갱신 믿음지도 · berrypicking 6수 툴박스 · 계층 가지치기(일반 FS 미검증=기여) · forensic-path provenance · diet 모델(고가치 풍부하면 식단 좁힘).

## 9. 후속 연구 (우선순위 — §0.5 재정초 반영)

- **[진행중·1순위] AI-혼자 베이스라인 실패점 찾기.** 기계장치 선구축 대신, 다양한 케이스로 AI-혼자 포식을 돌려 *실제로 깨지는 지점*을 카탈로그화 → 그 실패가 더할 최소 시스템을 정의. 1차 결과 §11.
- **[완료] TAR knee+elusion 정지 수학 정독** (arXiv:2106.09871). 결론: 카운트 상수(1000/156/0.75C)는 법조 규모 보정이라 이식 불가; 무릎=MVT 집행형이라 *형태만* 수입(상수 재측정); Quant 추정기는 보정확률 의존이라 이식 불가 → **정지는 AI 판단, elusion만 "음성 측정 행위"로 얇게.**
- **[격하] 냄새 채점기 calibration** — Quant apparatus를 안 지을 거면 우선순위 하락. 단 §11에서 AI 자기-정지 판단의 신뢰성이 실패로 드러나면 부활.
- **[조건부] 포렌식 메타 자동분류 정독** (arXiv:2012.01972 등) — *몸(싼 신호)* 보강이 필요하다고 §11이 가리킬 때.
- **두 prior 합성·믿음 지속 구조** 설계 스케치 (더할 것 ②③).

| **F1** | 몸 갭(싼 탐침 기질 부재) | 여행: EXIF/Spotlight null → 비싼 내용층 강제 | ✗ | **① 몸**(probe 바인딩+가용성 점검·폴백) |

| **F5** | 사적 prior 부재 | 낯선 디스크서 현행성·저자 복원에 내용 읽어야 | ✗ | **③ 일화기억·정리습관 주입** |
| ~~F6~~ | ~~스케일~~ | 111k 파일을 ~6개 열어 해결 | — | **불필요**(AI가 트리 탐 — LlamaIndex 실패는 *평탄* 검색에 한정) |

**결과**: 다섯 실제 실패 = §0.5의 **네 추가와 정확히 1:1.** 정지 apparatus·루프 컨트롤러·다단 티어는 실패 목록에 없음 → *짓지 않는다.*

**함의(하네스 축소)**: "finder=프로그램"의 프로그램은 이제 ⓐ 탐침 바인딩(몸) ⓑ 지속 저장소 ⓒ provenance/측정 헬퍼 *셋뿐*. **루프는 AI 에이전트 자체.** §3의 4단 티어·세션내 믿음 부기는 척추에서 빠짐(얇은 비용 최적화로만 잔존).

**미해결(이 분류의 한계)**: 트레이스가 3건뿐 — 더 다양한 케이스(고recall 망라·여러 몸[코드·웹]·진짜 콜드 2세션)가 새 실패를 낼 수 있음. F3 측정행위와 F2 지속기억은 *단일 세션으론 실패를 끝까지 못 봄* → 다음 검증 대상.

## 12. 2-패스 누적 실험 (1차 결과, 2026-06-20) — F2의 *무엇을 증류하나*

방법: 패스1(여행·한국강의 콜드 포식)을 명시 *디스크 경험 노트*로 증류 → 새 질의("RIKEN 일본 시절 자료")를 노트로 포식, 리프트와 *전이된 성분* 관찰. (단일 세션 프록시 — 진짜 교차세션 아님.)

**노트(패스1 증류)**: `total`=계산신경과학자(POSTECH→NYU→RIKEN) 2002~14 아카이브. 폴더정체(RIKEN PC etc=리켄기 / work2=연구·발표 / journal*·안젤루치=논문=죽은가지 / My Pictures=개인사진) + 관습(발표=장소+날짜, 사진=카메라코드) + 기질(EXIF null).

**결과**: 노트가 패스2를 **1탐침 직행**(루트 재프로파일·후보 재triage 스킵)시킴. "journal=논문" 규칙이 *새 가지(RIKEN PC etc/journal)에도 일반화*되어 재확인 없이 prune. 노트가 못 준 새 내용(`내글`·`희정 글`·월별 xls·RIKEN talk/korean)은 탐침이 발견.

| 증류 성분 | 전이 | 비고 |
|---|---|---|
| 폴더 정체 | 높음 | 즉시 라우팅 |
| 주인 관습(명명·정리) | 높음 | *새 가지에도 일반화* |
| 죽은 가지 | 높음 | 재탐색 안 함(음성지식 전이) |
| 주인 정체/약력 | 높음 | 새 가지 즉시 의미화 |
| 이력 join | 보너스 | 별개 포식과 교차연결(korean) |
| 날 내용/로그 | **낮음** | 새 내용은 여전히 탐침 |

**결론**: 재사용 증류물 = **지도(폴더정체)+적응prior(관습)+음성지식(죽은가지)+정체/이력** — *내용·로그 아님.* §0.6의 "(b)=지도+적응prior"를 실증. **싸게 증류·큰 레버리지(오리엔테이션 스킵)** → F2 실현가능·고ROI. = 냄새지도 스키마의 1차 설계 근거.

**한계**: ① 지도≠전지(새 내용은 탐침). ② 교차세션 지속 미검증. ③ 부패(디스크 변경) 미검증.

### 12.1 교차-몸 전이 실험 (§0.6 핵심 주장 검증)

디스크서 누적한 *주인 모델*만 들고 *웹*으로: 강국진/Kukjin Kang(POSTECH→NYU→RIKEN, 신경부호화·tuning curve). 3단:
- **콜드**("강국진") → 엉뚱한 유명인(화가 1939–1992)에 침수.
- **웜**(+도메인 prior) → 화가 사라지고 *올바른 동네*(RIKEN 신경부호화·tuning curve arxiv), 본인은 아직.
- **웜++**(디스크 신호 `FIPopCod`/`chernoff`로 Fisher·population·Chernoff) → **본인 논문 핀포인트**: Kang-Shapley-Sompolinsky, *J.Neurosci* 2004, NYU Scholars 호스팅(=postdoc 확증), Chernoff 사용(=디스크 파일 확증).

**결과**: ① 전이 성공 — 주인 모델이 *검색불가 질의(흔한 이름→유명인)*를 핀포인트로. ② 건넌 건 디스크-지도 아닌 **주인 모델**(정체·로마자·분야·소속·내용신호) = 보편서치가 결여한 중의성 해소 prior. ③ 해상도 기울기(prior 날카로울수록 핀포인트). ④ **복리 루프**: 웹 결과가 디스크 모델 역확증·풍부화 → 양쪽 다음 포식 개선.

**설계 수확**: **F2 지속저장소 = 최소 2층 — (a) 몸별 지도(디스크 전용) + (b) 몸 독립 주인 모델(정체·분야·신호, 모든 몸 공유).** §2 디스크 실험이 (a)를, 이 실험이 (b)를 채움.

## 13. 필터버블 위험 — 논제를 *깨려는* 실험 (1차 결과)

방법: prior가 *죽은가지/차가움*으로 친 곳을 일부러 전수 조사 — 가치 있는 게 숨었나.

**실증된 near-miss**: `RIKEN PC etc`를 prior가 "리켄기 *연구*"로 틀 지웠으나 실제론 — `내글/`(개인 에세이 `나는 누구인가`·`독서노트`·**`주식`·`투자` 폴더**), `희정 글/`(**타인**의 출판 프로젝트 `창비기획서`·`출판사`). 연구-프레임 prior라면 *통째로 prune*했을 곳에 *가장 개인적인 내용*이 숨음. (단 `journal*`/`안젤루치`="논문" 규칙은 정확 — 거짓 prune 0.)

**결론 — 위험은 진짜지만 prior는 균일하게 위험하지 않다:**

| prior 종류 | 필터버블 위험 |
|---|---|
| 구조적(폴더=전부 PDF→논문; 싸게 재검증) | **낮음** |
| 의미/정체("이 폴더=연구기"/"주인=신경과학자") | **높음**(이질적 내용 억압·committal) |

★위험한 건 *의미적 과일반화* prior. 그리고 **가장 개인적인 내용일수록 지배 프레임을 안 따른다** → 개인 비서엔 치명적.

**안전판 4 (설계 요건)**: ① **defeasible prior**(committal prune 금지 — 새 질의 목표가 prune 이유와 안 겹치면 재오픈) ② **surface 카운터-패스**(주인 모델 *위반* 내용을 들어올림 — 이질=놀라움=가치) ③ **이질성이 라벨을 깬다**(동질성 붕괴→의미 라벨 무효화) ④ **prior provenance+확신**(저확신 의미 prior엔 committal prune 권한 없음).

**가장 깊은 함의**: 필터버블=*해자의 그림자*. 누적을 강력하게 만드는 빠른 가지치기 prior가 맹점을 만든다 → 개인 forager는 prior를 *폐기가능하게* 들고 누적에 *반대힘(surface)*을 짝지어야([[project_augmentation_over_autonomy]]).
