# 포식 기억(Forager Memory) — 시스템 설명서

*작성: 2026-06-20 / 성격: **완성된 시스템의 설명서.** 무엇을·왜 만들었고, 어떻게 동작하며, 그 의미가 무엇인지. 깊은 연구 기록은 `FILE_FORAGING_RESEARCH.md`, field 수준 스키마는 `FORAGER_MEMORY_SCHEMA.md`.*

> **한 문장**: AI가 디스크·파일을 *뒤져 찾을 때마다* 그 공간에 대해 배운 것을 세션 너머로 누적해, 다음 탐색을 더 싸고 정확하게 만드는 기억 시스템. "찾는 일이 곧 배우는 일"이 되도록.

---

## 1. 어떤 문제를 풀었나

### 1.1 AI는 매번 첫 방문자다

LLM은 구조적으로 **상태가 없다(stateless)**. 어제 당신의 하드디스크를 한 시간 뒤져 "아, `journal` 폴더는 옛 논문이고 여행 사진은 `My Pictures`에 카메라 코드로 정리돼 있구나"를 알아냈어도, 오늘 같은 질문을 하면 **그 깨달음이 전부 사라진 채 처음부터 다시 뒤진다.** 매 검색이 콜드 스타트다.

이건 단순한 캐시 문제가 아니다. 하드디스크는 **세 번째 기억**이다:

| | 하네스 메모리 | 하드디스크 |
|---|---|---|
| 누가 정리했나 | AI 자신 | **당신이 수년에 걸쳐** (남의 정리 논리를 읽어야) |
| 스키마 | 엄밀 | 느슨 — 폴더·이름·날짜가 *서술적 관습* |
| 꺼내기 | 정확 조회 | **점진적 좁히기**(가설→가지치기) |

문제는 저장이 아니라 **검색**이다. 가치는 거기 있는데 "있는 걸 못 찾는" 상태. 그리고 그 디스크를 *어떻게 뒤지는지*는 — 어느 폴더가 죽은 가지인지, 주인이 무엇을 어떻게 정리하는지는 — **당신만의 것이라 AI의 가중치에 없다.**

### 1.2 웹은 왜 이 문제가 덜한가

같은 "찾기"인데 웹 검색은 AI가 꽤 잘한다. 왜? **"어떻게 뒤지나"가 이미 공동체에 누적**돼 있기 때문이다 — 구글의 색인, 수많은 사람의 클릭, 그리고 그게 녹아든 모델 가중치. AI는 *웹 서치 베테랑으로 도착*한다.

내 디스크엔 그런 공동체 축적이 없다. 그러니 **사적 공간에선 "어떻게 뒤지는지 배우는 일 = 뒤지는 일"이고, 그 누적 기제가 척추**다. 이 시스템이 채우는 게 바로 그 빈자리다. (그리고 이렇게 누적한 *개인의 서치 방식*은 웹에도 얹혀 웹 검색마저 개인화한다 — §5.3.)

### 1.3 그래서 진짜 문제는

> **개인이 자기 공간을 서치하는 *방식* 자체를 어떻게 누적하는가.**

이걸 연구 단계에서 **F2(콜드 시작 — 세션 넘는 기억 없음)**라 불렀고, 여러 실증을 거쳐 "이게 핵심 문제"라고 못박았다. 이번에 그 F2를 실제로 구현했다.

## 2. 핵심 설계 결정 — forager는 짓지 않는다

가장 중요한 통찰부터: **forager(뒤지는 주체)는 이미 AI다. 우리는 그걸 짓지 않는다.**

옛 분야들(디지털 포렌식·eDiscovery·탐색이론)은 *지능이 없던 시대*에 "기계가 정보를 이해할 리 없으니" 학습 분류기·정규식 룰북·정지 공식 같은 **보상 장치**를 잔뜩 지었다. 강한 AI에겐 그 장치 대부분이 **흡수될 죽은 무게**다 — AI는 이미 싼 것부터 뒤지고, 내용을 읽어 판단하고, 언제 멈출지 자연어로 추론한다.

**침식 테스트**: 어떤 부품이든 "모델이 더 좋아지면 이게 풀리나?"를 물어라. 풀리면 짓지 마라. 그래서 정지 공식·루프 컨트롤러·다단 모델 티어는 **안 만들었다.**

안 풀리는 것 — 지능과 무관하게 남는 것 — 은 단 하나로 좁혀졌다: **지속 기억.** AI가 아무리 똑똑해져도 세션이 끝나면 잊는다는 사실은 변하지 않는다. 그래서 우리가 더한 건 forager가 아니라 **forager가 결여한 기억**이다.

## 3. 어떻게 풀었나 — 구조

### 3.1 2층 기억

누적하는 지식을 수명과 범위가 다른 두 층으로 나눴다:

**(a) 몸별 지도 (`forage_map`)** — *이 디스크 전속*
이 디스크 위에서만 의미를 갖는 것. 폴더의 정체, 주인의 정리 관습, 죽은 가지(여기엔 없음), 기질 가용성(이 볼륨은 색인이 없음).
> 예: `journal/ = 출판 논문(죽은 가지) — 여행 자료 없음` / `사진은 카메라 모델 코드로 정리(관습)` / `외장 볼륨이라 EXIF 색인 없음(기질)`

**(b) 몸독립 주인 모델 (`owner_model`)** — *모든 몸 공유*
나라는 개인의 정체·분야·소속·내용 지문·어휘 매핑. **디스크·웹·코드·책을 가로지른다.**
> 예: `domain = 계산신경과학(POSTECH→NYU→RIKEN)` / `habit = 연구와 개인 미디어를 디렉토리로 엄격 분리`

이 분리가 왜 중요한가: 몸별 지도는 디스크가 바뀌면 무의미하지만, **주인 모델은 몸을 가로질러 레버리지**된다. 실제로 디스크에서 쌓은 주인 모델만 들고 웹에서 "강국진"(흔한 이름)을 검색하면 — 엉뚱한 유명 화가에 빠지지 않고 — *본인의 신경과학 논문을 핀포인트*한다(§5.3).

### 3.2 닫힌 루프

```
   ┌─────────────────────────────────────────────────────────────┐
   │                  forage_memory.db (세션 너머 지속)            │
   └─────────────────────────────────────────────────────────────┘
        ↑ 증류                                          ↓ 주입
   ④ 포식 후 *지도 델타만*              ③ 포식 시작 시 <forage_memory>로
      뽑아 누적(surprise/교정)              관련 지도·주인모델 회상
        ↑                                          ↓
        └──────── ② 포식 (AI 루프) ───────────────┘
              fs_query·grep·read 로 싼 것부터 뒤지기
                         │
                ⑤ surface: 기존 라벨을 위반하는
                   이질 내용 발견 → 의심 표식 (반대힘)
```

네 동작이 한 바퀴를 이룬다:

1. **주입** — 실행기억(해마)처럼 *항상-on*. 그 공간에 대해 *이미 아는 것*을 프롬프트에 넣어준다(AI가 첫 방문자가 아니게 된다). 회상은 싸고(LLM 0), 무관하면 query 필터가 빈 결과로 자기-억제(비용~0). **map은 query 필터(관련 위치만), owner(주인모델)는 상시 노출=냄새(scent)** — 명시 명령 없이도 "이 주제로 내 자료 있을 수 있다"를 상기시켜 능동 포식을 촉발. (2026-06-20 DESIGN §10: 키워드→THINK→항상-on 으로 최종 수렴.)
2. **포식** — 루프 자체는 그냥 AI다. 메타데이터(싸다)부터 보고 좁힌 뒤에만 내용(비싸다)을 연다. 이미 IBL 동사(`fs_query`/`grep`/`read`)가 그 손이다.
3. **증류** — 포식이 끝나면 경량 AI가 *다음에도 쓸 일반화 가능한 지식만* 뽑아 지도에 누적한다. "이번에 찾은 특정 파일"은 버리고 "이 폴더는 무엇인가"만 남긴다.
4. **surface** — 누적이 만드는 그림자(필터버블)를 막는 반대힘. 아래 §3.4.

### 3.3 무엇을 누적하고 무엇을 버리나

핵심 규율: **지도를 저장하지 내용을 저장하지 않는다.**

2-패스 실증에서 "재사용되는 증류물"이 무엇인지 측정했더니 — 폴더 정체 / 주인 관습 / 죽은 가지 / 정체·이력. **날 내용과 로그는 전이되지 않았다.** 그래서 증류기는 "여행 사진을 못 찾았습니다" 같은 *이번 결과*는 버리고, "`journal`은 논문 폴더라 사진이 없다(이유: 연구/개인 격리)" 같은 *다음에도 참인 지도*만 남긴다.

이게 싸고 강력한 이유: 한 번의 포식이 11만 개 파일을 색인하는 게 아니라 ~5개의 지도 항목을 남긴다. 그리고 그 항목 하나가 다음 포식의 오리엔테이션을 통째로 건너뛰게 한다.

### 3.4 안전판 — 누적의 그림자를 막는 반대힘

빠른 가지치기를 가능케 하는 prior가 곧 **맹점**이 된다. 실증된 위험: "주인 = 신경과학자"라는 prior는 `내글/`(개인 에세이·**주식·투자** 메모)·`희정 글/`(타인의 출판 프로젝트)을 *통째로 prune*할 뻔했다. **가장 개인적인 내용일수록 지배 프레임을 안 따른다** — 개인 비서엔 치명적이다.

그래서 모든 prior에 네 안전판을 *구조적으로* 박았다:

1. **폐기가능(defeasible)** — "죽음"이 아니라 "~이유로 *아마* 죽음"을 저장한다. 새 질문의 목표가 그 이유와 안 겹치면 자동으로 그 가지를 재오픈한다.
2. **prior_class 게이팅** — *구조적* prior(폴더=전부 PDF, 싸게 재검증)만 단호하게 prune할 권한이 있다. *의미적* prior("이 폴더=연구기")는 건너뛰되 삭제표시는 못 한다.
3. **surface 카운터-패스** — 포식 중 기존 라벨을 *위반하는* 이질 내용을 만나면 그 라벨에 의심 표식을 단다. 이질성이 곧 라벨을 깬다. (실제로 "RIKEN PC etc = 연구"라는 라벨에 개인 투자 메모가 나오자 surface 표식이 찍혔다.)
4. **provenance + confidence** — 모든 항목은 "왜·어느 포식에서 형성됐나"와 확신도를 달고 다닌다. 저확신 의미 prior는 자동으로 prune 권한을 잃는다.

> 필터버블은 *해자의 그림자*다. 누적을 강하게 만드는 그 prior가 맹점을 만든다. 그래서 강한 누적 기제엔 항상 반대힘(surface)을 짝지었다.

## 4. 무엇으로 만들어졌나 (구현 지도)

코드 하네스는 의도적으로 얇다 — **저장소 + 탐침(이미 있음) + provenance 헬퍼, 그뿐.** 루프는 AI다.

| 조각 | 파일 | 역할 |
|---|---|---|
| **저장소** | `backend/forage_memory.py` | 2 테이블(`forage_map`/`owner_model`) + note/recall/forget + provenance 병합(복리) + lazy 부패 체크 + `recall_xml`. 임베딩 없이 키워드/경로 부분일치(항목이 적어서). |
| **IBL 액세서** | `[self:forage]{recall\|note\|forget}` (`self.yaml`·pc-manager) | 기억 CRUD 동사(augmentation — 사람이 직접 조회·정정). 루프 동사 `[?:seek]`는 *안 만들었다*. |
| **회상 주입** | `agent_cognitive._search_forage_memory` (0단계 항상-on) | 해마처럼 매 메시지 회상 → `<forage_memory>` XML을 연상기억에 합침. map=query 필터, owner=상시 노출(냄새, `filter_owner=False`). 무관하면 빈 결과(비용~0). 게이트는 tier가 아니라 *관련성*(query). |
| **자동 증류** | `agent_cognitive._distill_forage_memory` | 포식 종료 훅. 경량 LLM이 기존 지도를 "이미 아는 것"으로 받아 *델타만* 추출. 2단 게이트(질문 단서 + 응답의 디스크 흔적)로 LLM 낭비 차단. |
| **부패 무효화** | `forage_memory._stale_of` | 회상 시 폴더 mtime을 비교해 변했으면 `stale`, 사라졌으면 `missing` 노출 — *삭제하지 않고* AI가 판단하게. (감쇠 곡선 같은 손튜닝 장치 안 씀.) |

이 시스템은 기존 자산 위에 올라탔다 — 이미 있던 `[self:folder_note]`(원시 폴더 주석)를 부유화하고, 해마·심층메모리의 증류·정리 배관을 *공간*에 재사용했다. 새 종(種)을 0부터 짓지 않았다.

## 5. 그 의미

### 5.1 복리되는 개인 자산 (비침식)

이건 *데이터*지 지능이 아니다. 모델이 좋아져도 이 기억은 흡수되지 않는다 — 오히려 좋은 모델이 더 잘 누적한다. 쓸수록 디스크가 *나를 통해 보면* 더 빨리·정확히 탐색된다. **시간이 이 시스템의 편**이다.

### 5.2 벤더가 못 닿는 층

구글·군중이 *공개적으로* 축적한 "어떻게 뒤지나"를, 우리는 *사적·사용자별로* 다시 짓는다. 벤더는 당신의 디스크에 접근하지 못하고, 당신의 정리 습관을 모른다. 분야의 빈자리는 정확히 여기다 — 학계·업계는 *보편 forager* + 얕은 결과 개인화(PIM)에 머물렀고, **누적되는 *개인 forager 정책*은 아무도 조립하지 않았다.**

### 5.3 하나의 forager, 여러 몸 — 코드 몸 추가(2026-06-20)

주인 모델(2층 b)은 디스크에 갇히지 않는다. 디스크에서 배운 "이 사람은 계산신경과학자, `FIPopCod`·`chernoff` 같은 신호를 쓴다"가 **웹 검색의 중의성을 해소**한다. 검증 실험에서 흔한 이름 "강국진"이 콜드 상태에선 엉뚱한 유명인에 빠졌지만, 주인 모델을 얹자 본인의 *J. Neurosci* 2004 논문을 핀포인트했다. 그리고 웹 결과가 다시 디스크 모델을 역확증한다 — **복리 루프.** 같은 코퍼스가 디스크·웹·코드·책을 가로지른다.

**핵심 재정의(다중 몸의 출발점)**: `body` 컬럼이 *하드웨어 자아*(mac/phone)가 아니라 **포식 공간**을 키잉한다. 같은 맥 자아가 디스크도·코드레포도 포식하면 `forage_map`에 body가 둘 — `mac`(홈 디스크) / `code:<repo>`(코드레포). 하드웨어 자아(누가 포식)는 *게이트*(폰 자아는 미디어-한정 skip), 포식 공간(무엇을 포식)은 *body 키*로 깨끗이 분리됐다. owner_model은 여전히 1명, 모든 공간 공유.

**첫 몸 = 코드베이스** — probe(`grep`/`glob`/`read`)가 이미 다 있어 가장 싸게 검증됐다. 인지층이 메시지+응답 단서로 공간을 자동 추론(`code:<repo>` vs `mac`), 증류 프롬프트가 공간-인식(폴더 정체→모듈 정체, 명명 관습→코드 관습), residual은 코드 모집단을 `code_candidate_paths`(repo walk)로 열거한다. 실LLM 종단 검증: 코드 포식 대화가 `code:indiebizOS` body에 "IBL 액션은 ibl_nodes_src/에 정의→build로 생성", "핸들러 op 분기는 `_OP_DISPATCHERS`", "src→build→--check 삼각검증 습관" 같은 *일반화 가능 코드 지식*을 증류했고, 별도 프로세스가 그 지도를 회상했다.

**둘째 몸 = 웹** — 웹의 진짜 가치는 *망라*가 아니라 **owner_model 교차-몸 전이**(§5.3 첫 문단·연구 §12.1). 공간 추론(URL·온라인 의도→`web`, 우선순위 code→web→disk), 증류가 web map은 가볍게(예: "내 논문=NYU Scholars", "arxiv=신경과학 preprint")·**owner_model을 무겁게**(정체·로마자·분야·소속·신호어휘). 실LLM 종단: "강국진 논문 찾아줘" 웹 포식이 owner_model을 강화(신경과학자·Neural coding/tuning curve 신호·POSTECH→NYU→RIKEN)했고, 그 주인모델이 *전 공간 공유*(별도 프로세스의 `mac`·`code` 회상에 그대로 나타남)되어 디스크/코드 포식까지 풍부화한다 — **복리 루프**. **★웹 음성단언은 짓지 않는다**: 모집단이 무한·비열거라 residual 균일표본이 정의 불가 → "충분히 찾았나"는 AI의 berrypicking 포화 판단(연구 §9 "정지는 AI", `FORAGER_MULTIBODY_DESIGN.md` §8). residual은 열거 가능 공간(디스크·코드) 전용.

설계 결정 = `FORAGER_MULTIBODY_DESIGN.md`.

**★매체별 분기를 불변 2축으로 접음**(2026-06-20, DESIGN §9): 코드·웹을 키워드-cue + per-medium 분기로 짰더니 *매체마다 코드가 느는 냄새*가 났다. 매체는 가짜 축 — 진짜는 **① body=문자열(AI가 명명) ② 열거 가능한가(이진)** 둘뿐. 공간 추론 cue 리스트·증류 per-medium 분기를 *삭제*하고, 증류기 LLM 이 포식한 공간을 *명명*(space)하게, 회상은 전-body 자기-스코핑, residual은 이진(code vs path-walk)으로 통합했다. **결과: 책·외장볼륨·미래 매체 = 새 코드 0**("책 추가"=AI가 `book:<제목>`이라 부르고 장을 walk). forager=AI를 끝까지 지킴. 남은 몸 = 실사용 시 *검증만*(구현 불필요).

### 5.4 일곱 번째 기억

indiebizOS는 이미 여섯 종의 기억(의미·작업·일화·절차·관계·자기상태)을 갖고 있다. 포식 기억은 그 일곱 번째 — **공간 기억**이다. 그리고 해마(절차 기억)가 "어떻게 하는가"를 증류하듯, 포식 기억은 "어디에 무엇이 사는가"를 증류한다. 둘 다 *쓰면서 배우는* 자기 학습 루프이고, 둘 다 증류(입력)와 정리(위생)의 대칭을 따른다.

### 5.5 augmentation — AI와 사람의 분업

자율이 아니라 외골격이다. AI가 자동으로 누적·가지치기하되, **사람은 AI가 prune한 것을 재오픈하고(`forget`/`note`), surface는 AI가 숨길 뻔한 것을 들어올린다.** 빠른 누적과 그 반대힘이 항상 짝을 이룬다.

## 6. 현재 상태와 경계

- **동작 검증 완료(실제 LLM)**: 주입 → 포식 → 증류 → surface 한 바퀴가 실LLM으로 돌았다. 증류가 죽은가지(+폐기 이유)·관습·기질·주인 분야를 뽑고 날 결과는 버렸다(연구 실증과 일치). surface가 필터버블 near-miss를 재현해 라벨에 의심을 찍었다.
- **F2 핵심 검증**: *교차세션 지속*(별도 프로세스가 디스크에서 회상) + *부패*(폴더 변경→`stale`, 부재→`missing`) 둘 다 확인.
- **맥 자아 전용**: 주관적 기억이라 자아별 사적이다. 폰은 미디어-한정이라 후속.
- **재시작 후 라이브**: 인지층(주입·증류) 변경은 코어 모듈이라 백엔드 재시작 후 켜진다.
