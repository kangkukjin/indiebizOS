# 모델 기어 시스템 (Model Gear)

자동차 기어처럼, 계기판 레버 하나(절약/균형/최대)로 시스템 전체의 모델 등급을 바꾼다.
"하네스의 가치 = 최고 모델 없이도 좋은 결과" + "대부분의 일은 심사숙고 불필요, 속도가 중요"
라는 명제의 운영 장치. 자율변속기(무의식 분류기 = 작업마다 티어 자동선택) 위에 얹는
**수동 변속 레버**.

## 핵심 구조 (3층 + 오버라이드)

1. **3 티어** = 모델 슬롯. 사용자가 *한 번만* 설정. 기존 config 재사용:
   - `경량` = `lightweight_ai_config.json`
   - `중급` = `midtier_ai_config.json`
   - `고급` = `system_ai_config.json` (← 옛 '시스템' 라벨 교체)
2. **4 축** = 텍스트 역할 종류. 각각 독립적으로 티어 배정:
   - `분류` (라우팅) · `평가` (채점) · `실행` (일) · `의식` (심사숙고)
   - ★평가≠분류, 의식≠시스템AI — 옛날엔 용접돼 있던 걸 분리.
3. **기어** = 축→티어 매핑. 프리셋 3단 + 사용자지정:

   | 기어 | 분류 | 평가 | 실행 | 의식 |
   |---|---|---|---|---|
   | 절약 | 경량 | 경량 | 경량 | 경량 |
   | 균형 | 경량 | 경량 | 중급 | 중급 |
   | 최대 | 경량 | 경량 | 고급 | 고급 |

   분류·평가는 원샷 판단이라 어느 기어서도 경량 권장(큰 모델=지연만↑). 편집 가능.
4. **오버라이드(핀)** = 특정 에이전트/역할만 기어 무시하고 티어/모델 고정.
   우선순위: `agent_id 핀 > role 핀 > 기어`.

## 역할 → 축 (전수)

| 축 | 역할 |
|---|---|
| 분류 | 무의식 분류 / 백그라운드 정리(경험증류·포식정리·recall압축·심층메모리) |
| 평가 | GoalEval 평가자 |
| 의식 | consciousness |
| 실행 | 프로젝트 에이전트 / 시스템AI / Reflex / 수동번역 / android / auto_response / **임베디드 텍스트 생성(슬라이드·정기보고·신문 본문)** |

## 모달리티는 기어 밖

이미지·동영상·임베딩(해마)은 모달리티 고정 모델 — 이미지 모델을 opus로 변속할 수 없다.
기어와 무관한 **별도 설정**(현재는 핸들러 패스스루: Gemini 이미지 키, 해마 fine-tuned 모델).
계기판 설정창에서 별도 구역으로 노출.

## 해소 흐름 (resolve)

```
resolve(role, agent_id?) →
  1. 오버라이드(agent_id, 그다음 role) 있으면 → 그 티어/모델
  2. role → axis      (model_gear.json: role_axis)
  3. axis → tier      (현재 기어 preset)
  4. tier → 모델       (티어 config; api_key 비면 고급 키 폴백)
```

config를 매 호출 읽으므로 기어 변경이 **즉시 반영**(핫리로드 — 재시작 불요).
provider 객체는 `provider|model|key` 로 캐시 → 기어 바뀌면 키가 달라져 자동 교체.
`set_gear()` 가 캐시를 비운다.

## 파일

- `data/model_gear.json` — 단일 진실원 (현재기어·tiers·presets·role_axis·overrides·modality)
- `backend/model_resolver.py` — `resolve()` / `get_gear()` / `set_gear()` / `get_provider_for()`

## 빌드 단계

1. ✅ **리졸버 + 스키마** (이 문서 + 위 2파일) — 동작 변화 없이 중앙화·검증 완료
2. ⏳ **15개 호출처 이관** — tier-named 접근자(lightweight_ai_call/_get_midtier_provider/
   system_ai_call/ConsciousnessAgent/_init_ai)를 role 기반으로. 핵심=의식·시스템AI 분리,
   프로젝트 에이전트가 실행 축 상속(개별설정 불요).
3. ⏳ **핫리로드 마무리** — 옛 캐시(`_midtier_provider_initialized` 등) 제거, 리졸버 캐시로.
4. ⏳ **계기판 UI** — 설정창(3티어 + 모든 에이전트 목록/핀 + 모달리티 구역) + 레버(절약/균형/최대).

## 검증 (1단계)

resolve가 절약/균형/최대에서 4축을 올바른 티어로(분류·평가 항상 경량 / 실행·의식만 변속),
오버라이드 우선순위(agent>role>gear), set_gear 핫리로드, 미등록 역할→실행 기본 전부 통과.
